# 毛绒 ERP 初始化范围

## 当前定位

`plush-toy-erp` 已从“初始化壳层”推进到“基于真实资料收口”的阶段。

当前阶段目标：

- 桌面后台继续保持一个入口
- 通过角色工作台、菜单权限、首页配置、帮助中心区分老板、跟单、采购、生产、仓库、财务
- 移动端按角色拆成多入口、多端口，但仍在同一个项目里共享 common / ui / api / 文档层
- 在不误建 schema 的前提下，先把流程、字段真源、数据模型和导入映射做实

## 已落地

- 管理员登录后默认进入桌面 ERP 后台
- 新增角色切换、角色默认工作台、角色过滤菜单、角色化帮助中心
- 新增统一模板打印中心和 5 套首批固定打印模板预览
- 新增 6 个角色移动端入口
- 基于真实 PDF / Excel / 截图重写流程文档与数据模型文档
- 基于真实资料收口字段真源和导入映射草案

## 本轮不做

- 扩展硬件链路、PDA、条码枪、图片识别
- 正式 Excel 导入落库
- PDF 坐标填充
- Excel 母版回写
- 业务保存链路自动带值打印
- 在字段关系未稳定前直接改 Ent schema

## 端口口径

| 入口 | 端口 | 说明 |
| --- | --- | --- |
| 桌面后台统一入口 | `5175` | 只有一个桌面后台入口，不拆多个桌面站点 |
| 老板移动端 | `5186` | 交期风险、异常、待结算、本周重点 |
| 跟单移动端 | `5187` | 客户 / 款式 / 缺资料 / 催料 / 催合同 / 交期预警 |
| 采购移动端 | `5188` | 缺料、到料、单价确认、回签、辅材包材确认 |
| 生产移动端 | `5189` | 今日排产、进度回填、延期原因、返工、异常 |
| 仓库移动端 | `5190` | 收货、备料、成品入库、待出货、异常件处理 |
| 财务移动端 | `5191` | 待对账、待付款、异常费用、结算提醒 |
| 后端 HTTP | `8200` | 桌面端和所有移动端统一复用 |
| 后端 gRPC | `9200` | 保持不变 |

## 当前数据库边界

- 默认开发数据库：`192.168.0.106:5432/plush_erp`
- 当前 `plush_erp` 已存在，但业务表仍未开始，只能视为账号基线
- 正式业务 schema 仍待字段稳定后再落
