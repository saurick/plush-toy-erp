# ERP 打印模板字段与编辑行为清单

## 目的

说明当前毛绒 ERP 打印模板里“左侧字段怎么改”“右侧纸面怎么编辑”“PDF / 打印到底吃哪一份内容”这些稳定行为，避免把普通路由页、旧草稿恢复和当前窗口级编辑链路混为一谈。

本文只收口当前可见编辑行为，不解释完整实现分层；实现真源见 `docs/erp-print-template-implementation.md`。

## 当前稳定行为

- 当前正式模板只有 `采购合同`、`加工合同` 两套，统一从 `打印模板中心` 打开独立弹窗编辑。
- 弹窗打开时会先经过 `/print-window-shell.html` 的准备页，再进入实际模板工作台；工作台首屏也会保持一次短暂准备态，避免刚打开时看到工具栏和纸面先抖一下再稳定。工作台稳定后会持续回写整窗 HTML 快照；刷新时壳页优先直接恢复该快照。
- 每个弹窗都会生成独立 `state`，并把草稿保存到当前窗口专属 key；同模板同时开多个窗口时，不再默认共用同一份草稿。
- 已有老草稿仍保留一次兼容回读：若当前窗口级草稿为空，会先尝试读旧的模板级 localStorage 草稿，避免本轮切换后把已有编辑内容直接丢掉。
- 左侧字段面板继续支持按“字段名或字段值”搜索过滤；字段列和值列使用固定宽度，长字段名在本列内换行，不再把值列挤窄。
- 左侧字段编辑与右侧模板继续双向同步：左侧 `input/textarea` 改值会立即回写右侧纸面；右侧 `contenteditable` 失焦后会把结果收回当前草稿。
- 两套合同都保留“分行编辑 + 右侧纸面原地编辑”双入口：
  - 左侧分行编辑表格适合连续改数量、单价、备注和条款。
  - 右侧纸面适合直接按打印观感微调单元格内容。
- 采购合同和加工合同主表当前都按“整格输入”理解：点击表格里的实际内容块即可直接改当前格；选行模式开启后，点击明细行会只切换当前操作目标，不会顺手改动其它行。
- `上插一行 / 下插一行 / 删除当前行 / 选择明细行 / 选择单元格 / 合并选区 / 拆分当前 / 计算规则` 继续只作用于当前弹窗里的草稿，不会静默回写业务真源。
- `在线预览 PDF / 下载 PDF / 打印` 都以当前弹窗里的纸面 DOM 为输出来源，不再走第二套单独模板：
  - PDF 预览 / 下载前会先收口当前焦点输入，再把同一份 DOM 快照交给后端 `/templates/render-pdf`。
  - 浏览器打印直接打印当前弹窗里的同一份纸面，不输出左侧字段面板。
- 两套合同在内容仍落在单页 A4 内时，继续保持 `0` 物理页边距；一旦内容跨到第 `2` 页及后续页，浏览器打印与在线 PDF / 下载 PDF 会统一切到 `@page margin: 5mm 0 5mm`，给续页顶部和底部留出固定缓冲，避免正文贴页顶开始排。
- 右侧纸面舞台继续统一使用灰色背景；打印态会隐藏左侧字段面板、顶部工具栏、公式面板和附件上传条，只保留 A4 纸面。

## 当前模板差异

### 采购合同

- 中间采购明细区保留表格，合同头、条款和签字区按普通纸质合同排版。
- `采购金额` 默认按 `数量 × 单价` 重算，但仍允许按合同快照手工改写金额列。
- 行选择、插行、删行会同时维护当前行选择、合并块和合计，不再复制相邻行旧值生成新行。
- 采购明细当前支持原地矩形 merge / split：合并后只保留左上角内容，被覆盖格会清空；删除命中纵向 merge 时会按整块一起删除。

### 加工合同

- 明细金额继续按 `委托加工数量 × 单价` 自动重算，右侧纸面金额列保持只读展示。
- 纸样 / 图样附件通过左侧上传条独立上传后，会同步进入右侧页底附件位，并随 PDF / 浏览器打印一起输出。
- 条款区、签字区和表格区都允许在右侧纸面直接编辑，再回写当前草稿。
- 加工明细当前和采购合同一样支持原地矩形 merge / split；金额列仍保持只读，但可以参与合并展示。

## 当前边界

- 当前窗口级恢复已经升级为“优先恢复整份弹窗 HTML 快照，再回退到对应模板和对应草稿”；但正文仍是 React 工作台，不是完全静态的 builder/runtime。
- 当前只有采购合同、加工合同两套模板接入这套编辑方式；后续若新增其他模板，应继续复用同一套 `state / shell / draft key` 主链，不要再回退到每个模板各自管理弹窗和草稿。
