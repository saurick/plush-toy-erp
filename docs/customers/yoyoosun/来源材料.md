# 来源材料 / Source Materials

本文件只记录永绅 yoyoosun 客户资料的脱敏用途分类，不列真实文件 inventory，也不把资料直接升级为通用产品真源。

| 资料类型 | 用途分类 | 说明 |
| --- | --- | --- |
| Excel | Customer Material / Data Import Source / Industry Template Candidate | 可作为字段、只读提取、dry-run 和行业模板候选；需评审后再沉淀 |
| PDF | Customer Material / Print Template Input | 可作为合同、报表或交付样式输入；不直接决定 schema，也不直接抽成产品模板 |
| 图片 / 截图 | Customer Material / QA Debug | 可帮助识别页面、字段和流程线索；不作为唯一真源，不做 OCR 自动入库 |
| seed / demo 数据 | Demo Seed / QA Debug | 只用于开发验收和演示，不替代真实业务事实 |
| print template | Print Template Input | 通用模板能力已经存在；客户样本只用于版式、抬头、条款和字段来源核对，待多客户重复后再评审行业变体 |
| 正式页面当前数据 | Data Import Target Preview | 只作为 future import preview 的目标模型参照；当前 dry-run 不写数据库 |

使用规则：

1. 资料先归档用途，再决定是否进入 Product Core、Industry Template、Customer Config 或 Import Adapter。
2. 当前客户特殊字段、特殊流程和特殊报表不能直接进入核心 usecase。
3. 涉及库存、质检、出货、预留和财务事实的资料必须进入领域评审。
4. yoyoosun 数据导入必须先校验私有 manifest、dry-run、字段分类、unresolved queue 和人工确认；当前不执行真实导入。

## 原始文件存储口径 / Raw Source Storage

真实 Excel / PDF / 图片不进入 Product Core 是长期边界：

- 永绅客户专属 Private 仓库保存原件、manifest、来源评审和私有验证入口。
- Product Core 只保存通用工具、合成 fixture 和本文件这类脱敏分类。

永绅客户专属 Private 仓库已推送提交 `75423d0`，17 个来源完成远端回读和本地校验；Product Core 当前工作树已移除原件、真实 manifest 和客户工程指导图片。Product Core 正式版本 pin 与既有 Git 历史治理仍未闭环。具体状态、权限和停止条件见 `原始客户文件归档评审.md`。
