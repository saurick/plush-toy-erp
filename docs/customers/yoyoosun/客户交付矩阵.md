# yoyoosun 客户交付矩阵 / yoyoosun Delivery Matrix

本文只回答一个问题：永绅 yoyoosun 当前能看到、试用、验收或暂不承诺哪些能力。

它是客户交付台账，不是 Product Core 真源。产品通用能力成熟度看 `../../product/产品能力进度台账.md`，客户差异看 `客户差异台账.md`。后续新增客户时，新建 `docs/customers/<customer-key>/客户交付矩阵.md`，不要追加到本文。

## 状态口径

| 状态 | 含义 |
| --- | --- |
| 未开始 | 当前不对该客户承诺，也没有可用入口 |
| 内部可用 | 代码或本地验证已有基础，但不直接对客户承诺 |
| 可演示 | 可用于演示或内部说明，但仍需明确不包含内容 |
| 可试用 | 可交给客户试用，必须配套边界说明和本地 / 试用环境验收证据；不等同于目标环境发布、真实客户导入或客户签收 |
| 已交付 | 已完成客户侧验收或交付确认 |

## 试用 / 手动回归数据口径

手动回归先使用 `scripts/qa/manual-regression-data-plan.mjs` 汇总 Product Core 中性 seed、永绅 preview fixture、试用模拟数据、业务事实模拟和岗位任务模拟入口。当前永绅 preview fixture 覆盖客户、供应商、材料、产品、仓库、BOM、销售订单、采购订单、委外订单、采购入库、质检、库存批次、出货、财务草稿和 Workflow 任务，并包含 draft / active / cancelled、pending / passed / rejected、ready / blocked / done 等手动检查状态。

这些数据只服务本地 / 试用环境回归和培训演示；它们不是客户真实导入、不是目标环境发布证据，也不能替代真实 import approval、备份、恢复计划或客户签收。

## 维护规则

- 客户是否能试用、是否能交付承诺、是否被阻塞发生变化时，更新本文。
- 更新时只写业务模块名，不使用 CAP、DELTA、MATRIX、EVIDENCE 等内部编号作为主索引。
- “已交付”必须有客户侧验收或交付确认；内部模拟、目标环境 smoke、本地测试只能支撑“内部可用 / 可演示 / 可试用”。
- 客户交付状态不能反向定义 Product Core；客户差异是否进入产品内核，仍看 `客户差异台账.md` 和产品评审。
