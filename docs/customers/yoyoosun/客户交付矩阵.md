# yoyoosun 客户能力、交付与差异矩阵 / yoyoosun Capability, Delivery and Delta Matrix

本文是永绅 yoyoosun 唯一的客户能力状态真源，回答两类问题：当前能看到、试用、验收或暂不承诺哪些能力；已确认或待确认的客户差异如何处理。

它不反向定义 Product Core。产品通用能力成熟度看 `../../product/产品能力进度台账.md`，差异分类规则看 `../../product/客户差异策略.md`；已确认决策和未决问题分别回到 `决策日志.md` 与 `问题待办.md`。后续新增客户时，新建 `docs/customers/<customer-key>/客户交付矩阵.md`，不要追加到本文。

## 当前环境基线

- Product Core 当前 HEAD 已包含后续审批办理、异常处置、真实收付款和多单核销；这些首先是本地能力。
- 133 `customer-trial-133` 的较早固定 V5 证据绑定 release `80b77faeab566660c77fc23cc66c272096692f16`、migration `20260722000505` 和 active revision `yoyoosun-customer-trial-133-package-v5.runtime-manifest-v1`，包含 11 个测试账号登录、50 页浏览器和 5 份 PDF 技术试用记录。
- 当前 HEAD 后续切片及最新 migration `20260722174820` 尚未整体重新发布到 133；本文凡涉及新异常、客户退货 / RMA 或收付款的目标状态均按“当前版本未发布”处理。
- 客户 UAT / 签收仍未完成。技术试用、页面可见和本地绿色都不能写成“已交付”。

## 状态口径

| 状态 | 含义 |
| --- | --- |
| 未开始 | 当前不对该客户承诺，也没有可用入口 |
| 内部可用 | 代码或本地验证已有基础，但不直接对客户承诺 |
| 可演示 | 可用于演示或内部说明，但仍需明确不包含内容 |
| 可试用 | 可交给客户试用，必须配套边界说明和本地 / 试用环境验收证据；不等同于目标环境发布、真实客户导入或客户签收 |
| 已交付 | 已完成客户侧验收或交付确认 |

## 试用 / 手动回归数据口径

手动回归先使用 `scripts/qa/manual-regression-data-plan.mjs` 汇总 Product Core 中性 seed、永绅 preview fixture、仍受支持的试用模拟入口、旧业务事实只读计划和岗位任务模拟入口。当前永绅 preview fixture 只证明候选数据覆盖；旧业务事实通用 apply 已退役，不能据此声称生产、委外、库存或财务事实已写入。

这些数据只服务本地 / 试用环境回归和培训演示；它们不是客户真实导入、不是目标环境发布证据，也不能替代真实 import approval、备份、恢复计划或客户签收。

## 维护规则

- 客户是否能试用、是否能交付承诺、是否被阻塞，或客户差异的分类、处理方式、触发条件发生变化时，更新本文对应一行。
- 已确认事项同步 `决策日志.md`，未决事项同步 `问题待办.md`；本矩阵只保留当前结论，不堆讨论过程。
- 更新时只写业务模块或差异名称，不使用 CAP、DELTA、MATRIX、EVIDENCE 等内部编号作为主索引。
- “已交付”必须有客户侧验收或交付确认；内部模拟、目标环境 smoke、本地测试只能支撑“内部可用 / 可演示 / 可试用”。
- 客户状态不能反向定义 Product Core；通用能力成熟度只更新全局产品能力台账，差异是否进入内核仍按产品评审和客户差异策略决定。
