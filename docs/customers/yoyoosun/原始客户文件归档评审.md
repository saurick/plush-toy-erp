# 永绅 yoyoosun 原始客户文件归档评审 / Yoyoosun Raw Source File Archive Review

本文只评审永绅真实 Excel、PDF、图片、私密 manifest 和提取输出的存储、访问与验证边界。它不列出真实文件名、人员、hash、大小或来源统计，也不是导入批准。

## 结论 / Decision

- Product Core 当前工作树不保存真实原件、私密 manifest 或未脱敏提取输出；普通构建、测试、CI、源码包和镜像必须在没有客户私有资料与客户访问凭据时运行。
- 客户明确指定永绅专属 Private Git 仓库保存原件、私有 manifest、来源评审与私有验证入口；仓库必须持续保持 Private 和最小授权。
- 客户私有仓库固定 Product Core 精确版本，通过兄弟目录或 CI multi-checkout 调用通用校验工具；不得成为 Product Core 的 submodule / subtree，也不得要求产品 CI 持有客户访问凭据。
- `config/customers/yoyoosun/` 与 `deployments/yoyoosun/` 当前继续保留非原件配置和脱敏部署资料；本轮不把原件隔离扩大为运行时客户配置或部署体系迁移。
- 原件和来源清单不能直接生成 Product Core、Workflow、库存、质检、出货、预留或财务事实。真实导入仍需另行实现通用批次 usecase、RBAC、审计、幂等、事务、失败恢复与对账。

## 存储与依赖方向 / Storage And Dependency Direction

```text
Product Core
  通用校验 / 提取工具
  合成 fixture
  脱敏类别映射
        ^
        | 固定产品版本，兄弟目录或 CI multi-checkout 调用
        |
Customer Private Repository
  tracked sources / private manifest / review / validate entry
```

Product Core 不反向引用客户仓库。新增客户采用相同方向，每客户单独授权，避免一个凭据或仓库可读取所有客户资料。

## 私有 Manifest 边界 / Private Manifest Boundary

私有 manifest 至少记录：

- 稳定 `customerKey` 与 `sourceId`。
- Private 仓库内相对路径、media type、hash 和 size。
- 资料分类、结构化提取策略、敏感复查要求。
- `canExecuteRealImport=false` 等 no-real-import 边界。

Product Core 文档只保留资料类别到领域 / 表单的脱敏映射，不复制真实文件清单、人员信息、hash、大小、对象地址或客户业务统计。

## Private Git 版本规则 / Private Git Version Rules

1. 原件与对应 manifest 在同一提交中更新，不让清单和文件分离。
2. 每次提交前校验所有来源的 hash、size、重复路径和未登记文件。
3. 推送后从远端新 clone 或 archive 回读，再次运行完整性校验。
4. 原件发生变化时通过新 commit 保留审计链，不改写既有提交掩盖变化。
5. 仓库必须保持 Private；访问变化、误公开或凭据泄露按安全事件处理。
6. 普通删除提交不会从 Git 历史清除敏感内容；历史清理必须单独评审维护窗口。

## 允许落点 / Placement Rules

| 内容 | 允许落点 | 边界 |
| --- | --- | --- |
| 真实原件、私密 manifest、来源评审、未决项 | 永绅客户专属 Private Git 仓库 | 不进入 Product Core、公共仓库、源码包、镜像或普通 CI artifact |
| 未脱敏提取输出 | 客户私有仓库 ignored `output/` | 不提交；job 结束后按资料治理要求清理 |
| 通用校验、提取、freeze、dry-run 工具 | Product Core `scripts/import/` | 不硬编码客户，不连接 DB，不执行真实导入 |
| 合成测试输入 | Product Core 合成 fixture | 不包含真实名称、文件内容、价格、地址或订单资料 |
| 脱敏类别 / 字段归属评审 | Product Core 客户文档与配置草案 | 不复制私有 inventory、hash、大小或原始行 |
| 非原件客户配置与部署资料 | 现有 `config/customers/yoyoosun/`、`deployments/yoyoosun/` | 当前暂不迁出，不因此新增 tenant 或代码分叉 |

## 权限、幂等与回滚 / Access, Idempotency And Rollback

- Product Core CI 不拥有客户私有仓库权限；客户私有验证只向授权实施与维护人员开放。
- `customerKey + sourceId + hash` 作为来源身份；相同输入重复校验和提取应得到等价业务内容，不写数据库事实。
- 原件与 manifest 并发通过 Private 仓库 Git review / merge 收口；本轮不引入分布式锁或同步服务。
- 回滚恢复上一个 Private 仓库 commit；禁止用改写历史掩盖普通资料版本变化。
- Git 历史清理由独立维护窗口评审 `git filter-repo`、远端 force push、旧 clone 失效、CI artifact 与备份范围；本轮不自动改写历史。

## 人工复查清单 / Manual Review Checklist

1. 原件是否含客户名、供应商、联系人、电话、地址、价格、结算、图片或其他敏感信息。
2. 私有 manifest 是否覆盖全部输入，且没有绝对本机路径、凭据或长期 URL。
3. Product Core 普通测试是否只使用合成 fixture，并在没有私有仓库时通过。
4. PDF / 图片是否只作人工参考，未被 OCR 后自动生成事实。
5. Excel 字段是否先进入 Product Core / Customer Config / Import Adapter / Deferred 分类，而不是从样本直接升级 schema。
6. Private 仓库远端回读、hash / size、访问策略、备份与恢复是否形成证据。
7. 日志和 CI artifact 是否只输出 sourceId、hash 与脱敏统计，不保存原件或未脱敏行。
8. 产品树清理、Private 仓库存储、Git 历史清理和客户签收是否分别表述，没有互相替代。

## 边界审计 / Boundary Audit

- Product Core：当前提交树与索引已移除原件目录、真实 manifest 和客户工程指导图片；客户源仓库边界测试持续守住当前树。
- SaaS tenant：否。`yoyoosun` 是稳定客户 key，不是 runtime tenant。
- Workflow / Fact：否。来源材料不得直接写 Workflow、库存、质检、出货或财务事实。
- 真实导入：否。私有 manifest、来源校验和 dry-run evidence 均不等于 import approval。
- Private 仓库存储：已建立专属存储路径；当前来源完整性、远端回读和产品版本锁定只以私有仓 manifest、README、`product.lock.json` 与 formal validate 证据为准，历史治理和客户签收不因此自动完成。
