# 永绅客户闭环交付验收清单

> 本清单用于把产品核心能力、永绅客户配置、角色流程、打印字段、来源材料、测试数据和发布证据对齐到同一套验收口径。当前仍以试用 / dry-run / preview 为主，未获得 authenticated active revision readback 前不得标记客户签收。

## 1. 产品核心与永绅客户边界

| 项目 | 当前证据 | 验收判断 | 边界 |
| --- | --- | --- | --- |
| Product Core | 当前代码、Ent schema、Atlas migration、QA 测试 | 可继续本地拉测 | 不因客户配置改变 schema / migration |
| yoyoosun 客户配置 | `customerPackage.mjs` / runtime manifest / projection matrix | 可试用配置资产 | `businessFlows / stateMachines / processPolicies` 仍是 preview-only |
| 角色矩阵 | `roleFlowMatrix.mjs` / `角色能力与流程矩阵.md` | 角色职责已对齐 | 不创建后端 RBAC，不写任务或事实 |
| raw-source 映射 | `rawSourceFormMap.mjs` / `raw-source-form-map.md` | 来源文件已全量映射 | 不直接写 runtime，不做 OCR 自动导入 |
| 试用数据 | `trialDataFixture.mjs` | 可用于本地 / 试用 UI 和打印验证 | 不污染真实客户数据 |
| 配置投影 | `projectionMatrix.mjs` / `配置投影覆盖矩阵.md` | runtime/planned 已分层 | planned field surface 不当作已生效 runtime |

## 2. 角色验收

| 角色 | 必须具备 | 不允许 |
| --- | --- | --- |
| 销售 | 客户、销售源单、销售协同任务 | 直接写库存、质检、出货、应收、发票 |
| 老板 | 审批、退回、经营查看 | 绕过业务 usecase 状态和权限校验 |
| 工程 | 产品、SKU、材料、工序、BOM | 直接生成生产、库存、采购、质检、出货、财务事实 |
| PMC | 生产评审、计划、风险 | 把计划写成生产完工或成品入库 |
| 采购 | 供应商、采购订单、采购合同打印 | 把采购承诺写成入库、库存或应付事实 |
| 仓库 | 入库准备、库存查看、出货执行 | 用任务完成直接增减库存 |
| 品质 | 来料质检、成品质检、异常处理 | 把质检任务完成直接写库存或出货事实 |
| 财务 | 财务放行、应收 / 应付草稿线索 | 冒充税控、总账、收付款核销或完整财务系统 |
| 生产 / 委外 | 委外订单、加工合同、生产进度线索 | 直接写成品入库、质检通过、结算或付款事实 |

## 3. 打印字段验收

### 采购订单 → 采购合同

必须能带出：

- 采购订单号、下单日期、预计回货日期。
- 供应商名称、联系人、电话、地址。
- 买方公司、联系人、电话、地址、签字人。
- 产品订单编号、产品编号、产品名称、材料品名、厂商料号、规格、单位、数量、单价、金额、备注。

边界：

- 客户配置默认值只覆盖买方 / 订货方字段。
- 供应商字段必须来自业务快照。
- 明细缺失时不能用模板样例补造。
- 空字段不能 fallback 成内部 ID。

### 委外订单 → 加工合同

必须能带出：

- 委外单号、下单日期、预计回货日期。
- 加工方名称、联系人、电话、地址。
- 委托单位、委托人、电话、地址、签字人。
- 产品订单编号、产品编号、产品名称、工序名称、加工厂商、工序类别、单位、数量、单价、金额、备注。

边界：

- 客户配置默认值只覆盖委托方字段。
- 加工方字段必须来自业务快照。
- 无来源明细不能补造样例行。

## 4. raw-source-files 验收

| 来源类型 | 当前处理 | 不允许 |
| --- | --- | --- |
| Excel 采购 / 加工 / BOM | 字段映射、dry-run、人工 review | 自动真实导入、直接写库存 / 财务 / 出货 |
| 合同模板 Excel | 打印字段和版式参考 | 作为运行时 Excel 母版 |
| PDF / 图片 / 截图 | 人工核对线索 | OCR 自动入库、作为唯一真源 |
| 岗位流程图 | 角色与流程边界线索 | 自动升级为 Product Core runtime 流程 |
