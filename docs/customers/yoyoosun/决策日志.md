## 新增决策规则

1. 分配唯一决策 ID，不改写历史行；变更时新增决策并写清替代关系。
2. “甲方已确认”必须有 Private 仓库中的会议纪要、签字版、邮件或等价受控来源。
3. 产品实现、客户配置、目标发布和 UAT 是不同类型的决策 / 证据，不合并成一条“已完成”。
4. 决策涉及系统变化、客户差异或交付状态时，同步[客户交付矩阵](客户交付矩阵.md)的对应分区和技术真源。
