# 永绅流程编排闭环矩阵

> 本文把永绅客户的 Workflow、ProcessRuntime、业务流、状态机、流程策略、UI 入口和验收门禁统一到一张矩阵。它的作用是防止把 preview-only 配置误判成 runtime，也防止把任务完成误判成业务结果完成。

## 1. 分层矩阵

| 层 | 当前状态 | 真源 | UI / 使用入口 | 结论 |
| --- | --- | --- | --- | --- |
| Workflow task | runtime_enabled | Workflow task 表和任务事件 | 桌面任务看板、Workflow V1、业务协同 Drawer、岗位任务端 | 可试用；只表达协同进度 |
| ProcessRuntime | runtime_enabled_partial | ProcessRuntimeUsecase 和流程实例 / 节点实例 | 客户配置 runtime manifest、JSON-RPC 受控入口 | 三条主流程已有局部 runtime，需要目标证据继续确认 |
| Production route / WIP | runtime_enabled_local | 生产订单路线快照、WIP 批次 / 事件、Quality Fact 和正式领域 usecase | 生产订单页、工序办理入口、质量检验页 | Product Core 本地源码已有正式入口；对应 WIP migrations 未 apply 到共享开发库或目标客户库，也未形成目标环境发布或客户 UAT 证据 |
| businessFlows | preview_only | `customerPackage.businessFlows` | 客户配置预览 / 文档 | 只表达跨模块意图，不作为执行真源 |
| stateMachines | preview_only | `customerPackage.stateMachines` | 客户配置预览 / 文档 | 不覆盖 Product Core usecase 生命周期 |
| processPolicies | preview_only | `customerPackage.processPolicies` | 客户配置预览 / 文档 | 不跳过权限、状态、引用和幂等校验 |

## 2. runtime 主流程覆盖

| 流程 | 状态 | 业务引用 | 节点类型 | 边界 |
| --- | --- | --- | --- | --- |
| `sales_order_acceptance` | runtime_enabled_partial | sales_order | domain_command / approval / human_task / end | 只处理销售源单受理与协同，不自动形成库存、质检、出货或财务结果 |
| `material_supply` | runtime_enabled_partial | purchase_order | domain_command / end | 采购、质检、入库必须分别经过受控领域入口 |
| `finished_goods_delivery` | runtime_enabled_partial_target_evidence_required | shipment | domain_command / end | 成品质检、财务放行、出货执行、应收草稿必须分开验证 |

### 2.1 客户已确认且已由 Product Core 本地正式入口承接的生产主线

客户已确认 `布料加工 → 裁片检验 → 车缝内外发决策 → 皮套检验 → 手工内外发决策 → 成品检验 → 针检 / 抽检 / 条件性客户验货 → 包装入库 → 放行发货`。详细业务基线见 `需求线索.md`。

Product Core 本地源码已用固定 `PLUSH_SEW_HAND_V1` v1 承接这条路线：生产订单发布冻结工序快照，WIP 批次 / 事件记录本厂车间移交或外发回仓，裁片合格转入车缝后才允许守恒拆批，车缝与手工逐道独立决定本厂或外发，Quality Fact 分别记录各质量关口。正式执行只走生产订单、WIP、委外、质检和库存领域 usecase。

`customerPackage.businessFlows.production_to_inventory` 继续保持 `preview_only` 是正确边界：它只解释永绅客户包的跨模块意图，不重复执行、不覆盖 Product Core 正式合同，也不证明客户配置已经发布或激活。`processes.sort_order` 仍只控制工序档案展示，不是路线真源。对应 WIP migrations 尚未 apply 到共享开发库或目标客户库，也没有目标环境发布、岗位 smoke 或客户 UAT 证据。

## 7. 当前结论

当前已经形成一张可测试的流程编排覆盖矩阵：Product Core 本地正式入口与客户配置预览边界明确，UI 入口、客户配置、角色职责、打印字段和签收门禁可以一起验证。固定 `PLUSH_SEW_HAND_V1` v1 已在本地源码承接 `车缝 → 手工`、逐工序内外发决策、WIP 转移和分段质检；`production_to_inventory` 保持 `preview_only`，不能据此生成第二套执行链。发布结论仍为 `internal-only`，WIP migrations、目标环境 readback、角色 smoke、PDF evidence 和客户 UAT 补齐前不得写成已发布或已交付；目标签收证据仍以 `deployments/yoyoosun/evidence/releases/2026-07-03/customer-signoff-readiness.md` 为准。
