# 永绅角色能力与流程矩阵

> 本矩阵用于对齐 Product Core 角色、永绅客户责任池、页面入口、动作能力和 Workflow / Fact 边界。它不是后端 RBAC 迁移脚本，也不直接创建账号或权限。

## 总原则

- 产品核心角色负责定义稳定权限码和菜单边界。
- 永绅客户角色通过 active customer config 的 role profile、entitlement、work pool membership 做投影。
- 任务可见性必须同时满足角色 / assignee / work pool / capability / customer scope。
- `workflow.task.complete` 只完成任务和推进 ProcessNode，不直接写库存、质检、出货或财务 Fact。
- 只有 domain command handler 能进入具体业务 usecase。

## 角色表

| 永绅角色 | Product Core 角色 | 责任池 | 主要页面 | 关键动作 | 流程职责 | 边界 |
| --- | --- | --- | --- | --- | --- | --- |
| 销售 | sales | sales | 客户、销售订单、出货、任务 | sales_order.* / workflow.task.complete | 销售订单提交、销售到生产线索 | 不写库存、质检、出货、应收或发票事实 |
| 老板 / 管理审批 | boss | boss | 任务、经营看板 | workflow.task.approve / reject | 销售、采购、付款审批 | 不绕过业务 usecase 校验 |
| 工程 | engineering | engineering | 产品、SKU、材料、工序、BOM、任务 | product / sku / bom / workflow | 工程资料补齐 | 不生成生产、库存、采购、质检、出货或财务事实 |
| PMC | pmc | pmc | BOM、生产计划、任务 | pmc.plan / pmc.risk / workflow | PMC 评审、排期、风险 | 计划状态不等于生产完工或成品入库 |
| 采购 | purchase | purchase | 供应商、采购订单、采购入库、任务 | purchase.order / purchase.receipt / workflow | 采购订单评审、采购源单准备、采购合同打印 | 采购承诺不等于采购入库、库存或应付事实 |
| 仓库 | warehouse | warehouse | 库存、入库、出货、任务 | warehouse.* / shipment.ship / workflow | 入库准备、出货执行 | 必须走入库 / 出货 usecase，不能用任务完成直接增减库存 |
| 品质 | quality | quality | 质检、入库、任务 | quality.inspection.* / workflow | 来料质检、成品质检 | 质检任务完成不等于库存或出货事实 |
| 财务 | finance | finance | 财务、采购入库只读、出货、任务 | finance.* / purchase.receipt.read / workflow | 财务放行、应收 / 应付草稿线索、采购入库来源核对 | 不创建、调整或确认入库，不办理采购退货；不等于税控、总账、核销或完整财务系统 |
| 生产 / 委外 | production | production | 委外订单、生产、任务 | outsourcing.order.* / pmc / workflow | 委外合同、生产进度线索 | 不直接写成品入库、质检通过、结算或付款事实 |

## 测试锚点

- `config/customers/yoyoosun/roleFlowMatrix.mjs`
- `scripts/qa/yoyoosun-customer-closure.test.mjs`

测试必须确保：客户包 workflow 中出现的 owner pool 都能在角色矩阵中找到；每个角色都有 `workflow.task.read`；角色 guardrail 不允许直接写库存、财务、出货或自动过账。
