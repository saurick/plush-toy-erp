# 永绅导入准备验收清单 / Yoyoosun Import Preparation Checklist

本清单验收的是来源与 dry-run 质量，不代表真实导入已经批准或执行。

| 检查项 | 通过标准 |
| --- | --- |
| 来源登记 | manifest 覆盖全部输入，hash / size 与原件一致 |
| 隐私边界 | 原始客户行、员工姓名、凭据和 DSN 未进入公开前端配置与提交产物 |
| 数据性质 | fixture / seed / 手工样本明确标记 simulated，不写成客户事实 |
| 字段归属 | 每个字段已归到 Product Core、行业候选、客户配置、客户资料、打印输入或禁止自动导入 |
| 主体匹配 | 客户、供应商、产品、材料、单位、仓库无法唯一匹配时进入未决队列 |
| 重复与冲突 | duplicates / conflicts 有来源行和处理决定，不静默覆盖 |
| 事实边界 | 未生成库存、质检、出货、预留、应收应付、发票或收付款事实 |
| dry-run 标记 | `canExecuteRealImport=false`、`noRealImport=true`、`executesImport=false` |
| 人工复核 | 业务人员能按来源行复核候选、跳过项和未决项 |
| 自动化 | manifest、extract、freeze、dry-run 与 test-data-isolation 测试通过 |

## 验证命令

```bash
node --test scripts/import/customerSourceManifestCheck.test.mjs \
  scripts/import/customerSourceExtract.test.mjs \
  scripts/import/customerSourceSnapshotFreezeCheck.test.mjs \
  scripts/import/customerImportDryRun.test.mjs \
  scripts/qa/test-data-isolation-boundary.test.mjs
```

## 停止条件

只要存在未确认来源、主体歧义、冲突未处理、需新增 schema 的字段、客户未确认或目标环境备份恢复未验证，就只能停在 dry-run。真实写入必须另开明确授权的数据治理任务，并先提供通用批次 usecase、RBAC、审计、幂等、失败恢复和对账能力。
