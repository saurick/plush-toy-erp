# 永绅配置投影覆盖矩阵

> 本文用于标清永绅客户配置投影的 runtime-enabled 与 planned 边界，防止把规划字段当成已经由后端 effective session 执行。

## 投影层级

| 投影层 | 当前真源 | 普通账号规则 | super admin 规则 | 说明 |
| --- | --- | --- | --- | --- |
| 页面投影 | `customer_config.get_effective_session.pages` | RBAC 菜单路径 AND active revision pages | 产品核心审阅账号不被 active revision 收窄 | 页面不可见不等于后端安全边界，后端仍要 RBAC |
| 动作投影 | `customer_config.get_effective_session.actions` | 后端 RBAC AND active entitlement AND module enabled | 产品核心动作 + RBAC + active actions | 正式写按钮不能回退本地 wrapper |
| 字段投影 | `field_policies` | 仅三个 runtime visibility surface 生效 | super admin 可见全部字段 | 只控制列表 / CSV 导出列 `visible`；正式字段合同不是配置 surface |
| 打印默认值 | `print_template_defaults` | 只允许买方 / 委托方默认字段 | 同样不覆盖业务快照 | 供应商 / 加工方字段必须来自业务单据快照 |

## 字段 surface

当前 demo 与 yoyoosun manifest 都从 Product Core catalog 生成相同的 `visible=true`，因此这里只证明投影链路已接通，不代表永绅已经配置了专属字段隐藏。后端 validate / publish 只接受 boolean `visible`，会拒绝 label、editable、required、空策略和非布尔值；表单及领域校验仍由页面 / usecase 合同负责。

| Surface / 合同 key | 状态 | 字段 | 结论 |
| --- | --- | --- | --- |
| `customers.default` | runtime_visibility_consumed | customer_code / display_name | 客户列表和 CSV 导出消费 `visible` |
| `suppliers.default` | runtime_visibility_consumed | supplier_code / supplier_type | 供应商列表和 CSV 导出消费 `visible` |
| `sales_orders.default` | runtime_visibility_consumed | order_no / source_no / expected_ship_date | 销售订单列表和 CSV 导出消费 `visible` |
| `sales_order_items.default` | formal_field_contract | product_no / product_name / sku_no / quantity / unit_price / amount | 正式源单字段；后端不允许作为 active field policy |
| `purchase_orders.default` | formal_field_contract | purchase_order_no / supplier_snapshot / expected_arrival_date / lifecycle_status | 正式源单字段；不是客户配置 surface |
| `purchase_order_items.default` | formal_field_contract | product_order_no / product_no / material_name / spec / unit / quantity / unit_price / amount | 正式源单字段；不是客户配置 surface |
| `purchase_receipts.default` | formal_field_contract | receipt_no / purchase_order_no / warehouse / status | 正式事实字段；不是客户配置 surface |
| `quality_inspections.default` | formal_field_contract | inspection_no / source_no / result / decision_note | 正式事实字段；不是客户配置 surface |
| `inventory_lots.default` | formal_field_contract | lot_no / warehouse / material / quantity / status | 正式事实 / read model 字段；不是客户配置 surface |
| `inventory_txns.default` | formal_field_contract | txn_no / txn_type / warehouse / material / quantity | 正式事实字段；不可编辑，也不是客户配置 surface |
| `bom_versions.default` | formal_field_contract | product / version / source_order_no / quantity_text / spare_text / print_date / designer / maker / auditor / hair_direction / status | BOM 表单 / 打印合同；不是 effective-session 字段策略 |
| `bom_items.default` | formal_field_contract | material / quantity / unit / loss_rate / position / piece_count / total_usage_snapshot / process_base / process_method / note | BOM 明细合同；不是 effective-session 字段策略 |
| `shipments.default` | formal_field_contract | shipment_no / sales_order_no / warehouse / status / shipped_at | 正式事实字段；不是客户配置 surface |
| `outsourcing_orders.default` | formal_field_contract | outsourcing_order_no / processor_snapshot / expected_return_date / lifecycle_status | 正式源单字段；不是客户配置 surface |
| `finance_facts.default` | formal_field_contract | finance_fact_no / source_no / fact_type / amount / status | 正式业务财务字段；不是客户配置 surface |

`fieldNumberingConfig.mjs` 中的 `product_id / style_no / color_size` 仍是 preview-only 客户评审候选，不属于上面的正式字段策略，也不会自动创建 SKU。

## 打印默认值边界

| 模板 | 状态 | 可默认字段 | 受保护业务快照 |
| --- | --- | --- | --- |
| `material-purchase-contract` | runtime_enabled | buyerCompany / buyerContact / buyerPhone / buyerAddress / buyerSigner | supplierName / supplierContact / supplierPhone / supplierAddress / lines |
| `processing-contract` | runtime_enabled | buyerCompany / buyerContact / buyerPhone / buyerAddress / buyerSigner | supplierName / supplierContact / supplierPhone / supplierAddress / lines |
