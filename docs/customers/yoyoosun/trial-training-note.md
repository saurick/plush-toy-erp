# 永绅 yoyoosun 试用培训说明 / Yoyoosun Trial Training Note

本文用于给永绅 yoyoosun 试用前讲清当前系统入口、已能试用的能力和明确不能误读的边界。本文不是 runtime、schema、migration、API、RBAC 或数据导入真源；当前行为仍以代码、测试和 `docs/current-source-of-truth.md` 为准。

## 1. 当前试用入口

当前桌面后台正式入口集中在左侧菜单：

| 菜单 | 当前用途 | 试用边界 |
| --- | --- | --- |
| `客户档案` | 维护客户交易主体 | 不在这里维护订单、出货、库存或财务 |
| `供应商档案` | 维护供应商 / 加工厂交易主体 | 不在这里维护采购入库、应付或付款 |
| `销售订单` | 维护客户订单承诺和订单明细 | 只表示客户订单和明细承诺，不自动生成出货、库存扣减、应收、发票或收款 |
| `产品`、`材料 BOM`、采购 / 仓储 / 生产 / 财务相关入口 | 当前已有业务页或后续试用线索入口 | 仍按各自能力成熟度和页面说明试用，不代表全部事实闭环已经完成 |

岗位任务端通过 `/m/<role>/tasks` 进入，按账号已有岗位权限自动进入对应任务页。用户不在登录前手选岗位；设备只影响默认入口，不决定权限。

## 2. 已退出的旧入口

以下旧入口不再作为产品试用入口：

| 旧路径 / 旧入口 | 当前状态 | 试用说明 |
| --- | --- | --- |
| `/erp/master/partners` / 旧 `partners` | 已退出产品内路由、菜单、重定向和权限别名 | 不再用于新增客户或供应商 |
| `/erp/sales/project-orders` / 旧 `project-orders` | 已退出产品内路由、菜单、重定向和权限别名；普通 `business` API 写入已冻结 | 不再用于新增销售订单或款式订单 |

旧 `business_records` 表仍存在，用于兼容层、source snapshot、debug/demo 和其他尚未退出的通用模块。当前不要把“旧入口已退出”理解成 `business_records` 表、所有历史记录或所有通用模块已经删除。

## 3. 菜单配置边界

永绅 yoyoosun 已有前端桌面菜单配置：`config/customers/yoyoosun/menuConfig.mjs`。

菜单配置只影响：

- 菜单分组。
- 菜单排序。
- 入口显隐。
- 入口显示文案。

菜单配置不影响：

- 后端动作权限。
- 用户能否执行保存、启停、提交、取消等动作。
- Workflow 任务状态机。
- 库存、出货、财务、采购、质检等事实规则。
- 数据库表结构。
- 真实数据导入。

因此，隐藏菜单不是安全边界；真正的权限仍由账号角色、后端 RBAC action permission 和业务 usecase 校验。

## 4. 销售订单的试用边界

`销售订单` 当前是 Source Document / Business Commitment。

可以试用：

- 新建、编辑、查看客户销售订单。
- 维护销售订单明细。
- 维护订单生命周期中的草稿、提交、激活、关闭、取消等当前状态。

不能误读为：

- 已生成出货单。
- 已扣减库存。
- 已做库存预留。
- 已生成应收。
- 已生成发票。
- 已生成收款或财务凭证。

真实出货、库存预留 / 出库、应收、发票和收付款仍需要后续 Shipment / Inventory / Finance usecase 单独评审和实现。

## 5. 试用前培训口径

试用前建议按下面顺序讲解：

1. 登录后先看桌面左侧正式菜单，不使用旧书签或旧路径。
2. 客户、供应商、销售订单从正式 V1 入口维护。
3. 岗位任务端只处理当前账号有权限的任务，不是另一套桌面菜单。
4. `销售订单` 只表示订单承诺；出货、库存和财务闭环还没有全部完成。
5. 菜单隐藏只影响看不看得到入口，不代表后端规则不存在，也不代表可以绕过权限。
6. 客户真实数据导入尚未批准；当前 dry-run evidence 只用于核对字段和风险。

## 6. 待确认问题

| 问题 | 当前处理 |
| --- | --- |
| 客户编码、客户简称、税号是否必填 | 进入客户字段 / 编号规则配置评审 |
| 销售订单编号和客户订单号分别由谁维护 | 进入导入 / 编号规则评审 |
| 销售订单激活后是否必须走出货计划 | 后续 ShipmentUsecase 评审 |
| 出货后何时生成应收和发票 | 后续 FinanceUsecase 评审 |
| 哪些角色能看到哪些菜单 | 先按当前角色权限和 yoyoosun 菜单配置试用，后续再做角色模板配置评审 |

## 7. 培训验收清单

| 检查项 | 通过标准 |
| --- | --- |
| 正式入口 | 试用人员能找到 `客户档案`、`供应商档案`、`销售订单` |
| 旧入口 | 试用人员知道旧 `partners / project-orders` 不再使用 |
| 销售订单边界 | 试用人员明确销售订单不等于出货、库存或财务 |
| 菜单配置边界 | 试用人员明确菜单隐藏不是权限或事实规则 |
| 导入边界 | 试用人员明确当前没有真实导入落库批准 |
