# 永绅 yoyoosun 试用培训说明 / Yoyoosun Trial Training Note

本文用于给永绅 yoyoosun 试用前讲清当前系统入口、已能试用的能力和明确不能误读的边界。本文不是 runtime、schema、migration、API、RBAC 或数据导入真源；当前行为仍以代码、测试和 `docs/当前真源与交接顺序.md` 为准。

当前没有可直接执行的客户真实数据。试用模拟不拆 A/B/C/D 或任何字母子任务，试用培训只能一次性按模拟数据演练：使用 seed、fixture 或手工构造的模拟客户、供应商、联系人和销售订单数据验证入口、账号、菜单、V1 页面和岗位任务端，不执行真实导入，不代表客户字段已经确认，也不生成出货、库存或财务事实。

## 1. 当前试用入口

当前桌面后台正式入口集中在左侧菜单：

| 菜单 | 当前用途 | 试用边界 |
| --- | --- | --- |
| `客户档案` | 维护客户交易主体 | 不在这里维护订单、出货、库存或财务 |
| `供应商档案` | 维护供应商 / 加工厂交易主体 | 不在这里维护采购入库、应付或付款 |
| `销售订单` | 维护客户订单承诺和订单明细 | 只表示客户订单和明细承诺，不自动生成出货、库存扣减、应收、发票或收款 |
| `工作台`、`任务看板` | 处理当前待办、阻塞和岗位协同任务 | 任务完成只代表 Workflow 协同状态，不等于库存、出货或财务事实已过账 |
| `产品档案`、`物料清单（BOM）`、采购 / 仓储 / 生产 / 财务相关入口 | 当前已有业务页或后续试用线索入口 | 仍按各自能力成熟度和页面说明试用，不代表全部事实闭环已经完成 |

岗位任务端通过 `/m/<role>/tasks` 进入，按账号已有岗位权限自动进入对应任务页。用户不在登录前手选岗位；设备只影响默认入口，不决定权限。

## 2. 已退出的旧入口

以下旧入口不再作为产品试用入口：

| 旧路径 / 旧入口 | 当前状态 | 试用说明 |
| --- | --- | --- |
| `/erp/master/partners` / 旧 `partners` | 已退出产品内路由、菜单、重定向和权限别名 | 不再用于新增客户或供应商 |
| `/erp/sales/project-orders` / 旧 `project-orders` | 已退出产品内路由、菜单、重定向和权限别名；旧普通 `business` API 方法已退出运行时 | 不再用于新增销售订单或款式订单 |

旧 `business_records / business_record_items / business_record_events` 表族已从当前 schema 和运行时删除。当前不要把“旧入口已退出”理解成还有只读旧页面、旧 API 或可继续试用的历史记录查询入口；如未来需要客户可见历史归档，必须单独评审归档模型，不能恢复旧三表。

## 3. 菜单配置边界

永绅 yoyoosun 已有前端桌面菜单配置源：`config/customers/yoyoosun/menuConfig.mjs`。试用部署时应通过静态根路径 `customer-config.js` 注入客户配置；默认产品前端包不静态打包 yoyoosun 配置。

菜单配置只影响：

- 菜单分组。
- 菜单排序。
- 入口显隐。
- 入口显示文案。

当前 yoyoosun 试用侧已显示 `业务看板`、`异常处理`、`生产排程`、`生产异常` 和 `出货放行`，并继续保留 `工作台`、`任务看板` 和岗位任务端。各入口只展示当前账号获授权且客户配置启用的页面；`出货放行` 仍只表示 Workflow / Shipment Release 口径，不等于出货单已 `SHIPPED`，也不会自动扣库存或生成应收。

菜单配置不影响：

- 后端动作权限。
- 用户能否执行保存、启停、提交、取消等动作。
- Workflow 任务状态机。
- 库存、出货、财务、采购、质检等事实规则。
- 数据库表结构。
- 真实数据导入。

因此，隐藏菜单不是安全边界；真正的权限仍由账号角色、后端 RBAC action permission 和业务 usecase 校验。

## 4. 销售订单的试用边界

`销售订单` 当前是 Source Document / Business Commitment。

可以试用：

- 新建、编辑、查看客户销售订单。
- 维护销售订单明细。
- 维护订单生命周期中的草稿、提交、激活、关闭、取消等当前状态。

不能误读为：

- 已生成出货单。
- 已扣减库存。
- 已做库存预留。
- 已生成应收。
- 已生成发票。
- 已生成收款或财务凭证。

销售订单生命周期本身不会自动生成出货、库存预留 / 出库、应收或发票；这些能力已分别通过销售订单或已出货单上的显式来源动作办理，并继续由 Shipment / Inventory / Finance usecase 校验。收付款、银行流水、完整核销、总账和税控仍未实现，不能从订单状态推断。

## 5. 试用前培训口径

试用前建议按下面顺序讲解：

1. 登录后先看桌面左侧正式菜单，不使用旧书签或旧路径。
2. 客户、供应商、销售订单从正式 V1 入口维护。
3. 岗位任务端只处理当前账号有权限的任务，不是另一套桌面菜单。
4. `销售订单` 只表示订单承诺；出货、库存和财务闭环还没有全部完成。
5. 菜单隐藏只影响看不看得到入口，不代表后端规则不存在，也不代表可以绕过权限。
6. 当前没有客户真实数据可导入，只能用模拟数据做试用演练；dry-run evidence 和 execution report 只用于核对字段、风险和门禁。
7. 模拟数据只服务培训和环境验证，不代表客户真实数据已导入或客户字段已确认。

## 6. 待确认问题

| 问题 | 当前处理 |
| --- | --- |
| 客户编码、客户简称、税号是否必填 | 进入客户字段 / 编号规则配置评审 |
| 销售订单编号和客户订单号分别由谁维护 | 进入导入 / 编号规则评审 |
| 销售订单激活后是否必须先做出货计划 | ShipmentUsecase 已支持显式出货；是否把出货计划设为客户操作前置仍需客户确认，不能由订单激活自动推断 |
| 出货后何时生成应收和发票 | 已支持在 `SHIPPED` 后显式生成；仍需客户确认由哪个岗位、在何时发起，不能自动等同银行收款或开票完成 |
| 哪些角色能看到哪些菜单 | 先按当前角色权限和 yoyoosun 菜单配置试用，后续再做角色模板配置评审 |

| 导入边界 | 试用人员明确当前没有真实导入落库批准 |
