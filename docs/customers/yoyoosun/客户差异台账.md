# yoyoosun 客户差异台账 / yoyoosun Delta Ledger

本文记录永绅 yoyoosun 客户的差异需求分类和处理规则。它只代表该客户差异，不替代全局产品能力台账，也不把客户差异自动升级为 Product Core。

## 1. 文档边界

- 全局产品能力成熟度见 `../../product/产品能力进度台账.md`。
- 本客户交付矩阵见 `客户交付矩阵.md`。
- 总入口索引见 `../../product/产品台账索引.md`。
- 新客户必须新建 `docs/customers/<customer-key>/客户差异台账.md`，不要继续追加到本文。

## 2. 客户差异台账说明

客户差异台账用于记录每个客户提出或隐含的差异需求，并对其分类。

客户差异必须先分类，再决定是否实现。

### 2.1 差异分类

| 分类                     | 含义                         | 是否进入 Product Core |
| ------------------------ | ---------------------------- | --------------------- |
| Product Core             | 所有客户都应该共享的核心能力 | 是                    |
| Industry Template        | 毛绒玩具行业常见默认能力     | 可能                  |
| Customer Config          | 客户配置项                   | 否                    |
| Customer Extension       | 客户专属扩展                 | 否，除非多客户验证    |
| Data Import Adapter      | 数据导入适配                 | 否                    |
| Print Template Candidate | 打印 / 导出格式样本          | 否，当前默认 Deferred |
| Reporting                | 报表展示差异                 | 视情况                |
| Customer Material        | 客户资料线索                 | 否                    |
| Deferred                 | 延后评审                     | 否                    |
| Forbidden                | 明确禁止                     | 否                    |

## 4. 客户差异处理规则

### 4.1 可以直接进入 Product Core 的条件

满足以下条件之一，才考虑进入 Product Core：

- 多个客户都会用。
- 属于 ERP 核心事实正确性。
- 属于库存、出货、财务、审计、权限等不可客户自由配置的规则。
- 不依赖 yoyoosun 单一客户特殊字段。
- 有明确业务口径和测试口径。

### 4.2 不得直接进入 Product Core 的内容

以下内容不得直接进入 Product Core：

- yoyoosun Excel 中单独出现的字段。
- yoyoosun PDF 中单独出现的合同条款。
- 当前客户口头说法但未确认的问题。
- 打印样本字段和单客户打印格式。
- 导入适配字段。
- demo / seed 数据字段。
- 仅由单客户样本提出、尚未经过领域评审的 SKU / 采购 / 出货 / 财务扩展字段。

### 4.3 必须进入 Customer Config 的内容

适合 Customer Config：

- 公司名称。
- Logo。
- 菜单显示。
- 字段显示 / 必填。
- 编号规则。
- 打印抬头、页脚、logo、公司信息等低风险展示参数。
- 默认仓库。
- 默认单位。
- 角色模板。
- 权限模板。
- 初始化数据。

当前实现没有开放上述全部项目：active field policy 只接通客户、供应商和销售订单列表 / CSV 列的 `visible`，且 yoyoosun 尚未声明专属隐藏规则；表单必填、可编辑和字段命名仍由 Product Core 页面 / usecase 合同负责。其他字段差异必须先留在 `fieldNumberingConfig.mjs` 评审，不得写成已生效配置。

### 4.4 必须进入 Data Import Adapter 的内容

适合 Data Import Adapter：

- yoyoosun Excel 字段映射。
- 删除前导出、归档 evidence 或旧样本的离线字段映射；目标只能是正式 V1 / 领域模型，不恢复 `business_records` 表、查询、双写或兼容层。
- 不同客户导入表头。
- 数据清洗规则。
- unresolved queue。
- dry-run preview。
- duplicate candidates。
- conflict candidates。

### 4.5 客户打印差异不自动升级为核心规则

Product Core 已有采购合同、加工合同和工程打印模板，但下列客户版式内容仍只作为 Customer Material / Customer Config / Print Template Candidate：

- 加工合同格式。
- 客户送货单样式。
- 采购单打印样式。
- 对账单导出样式。
- 页眉页脚。
- 公司名称、地址、电话、税号。
- 客户专属条款。

这些内容不得改变 schema、Fact、Workflow、权限或业务字段来源。只有至少 2-3 个真实客户的同类结构重复，且差异主要是抬头、字段显示、排序或版式微调时，才评审是否升级为行业模板变体。

### 4.6 仍需独立评审的能力

已有 Product Core 不能被单客户样本直接扩张。采购需求自动生成、完整生产订单 / MES、完整财务核销 / 总账 / 税控、真实客户导入执行、扫码和客户专属 extension 仍需独立评审；现有 SKU、采购、预留、出货、财务、生产和委外能力继续按各自 usecase 与事实边界演进，不再写成“未开始”。

## 5. 维护流程

### 5.1 什么时候更新客户差异台账

以下情况必须更新客户差异台账：

- 甲方提出新需求。
- 从 Excel / PDF / 截图中发现字段差异。
- 从试用反馈中发现特殊流程。
- 从导入 dry-run 中发现 unresolved 字段。
- 从客户打印样本中发现客户专属字段。
- 从权限配置中发现职责差异。
- 从业务讨论中发现行业共性。
- 某客户差异被提升为行业模板候选。
- 某客户差异被拒绝进入产品内核。
- 某客户差异被标记为 deferred。

更新时必须分类：

- Product Core
- Industry Template
- Customer Config
- Customer Extension
- Data Import Adapter
- Print Template Candidate
- Reporting
- Customer Material
- Deferred
- Forbidden

### 5.2 三类台账之间的关系

```text
客户差异台账
  -> 判断某个需求属于什么分类

产品能力进度台账
  -> 判断产品内核是否已经具备该能力

客户交付矩阵
  -> 判断某个客户是否可以看到、试用、验收该能力
```

关系示例：

```text
yoyoosun 提出“订单里要颜色”
  -> 客户差异台账：颜色字段来自 永绅 yoyoosun 样本，分类为 Industry Template Candidate / Deferred
  -> 产品能力进度台账：product_skus 已接维护页面和销售订单行 SKU 选择；显式 SKU 已贯通批次、库存流水 / 余额、生产 / 委外、出货和预留，未分规格库存保持独立 grain
  -> 客户交付矩阵：yoyoosun 可以试用订单行从已维护 SKU 选择带值并查看规格库存；暂不能承诺订单颜色自动转 SKU、BOM SKU 粒度、旧库存自动归类或真实数据导入
```

### 5.3 防止信息差规则

#### 5.3.1 禁止写法

禁止在没有证据时写：

- 已完成。
- 已上线。
- 已支持。
- 已交付。
- 可销售。
- 事实真源。
- 客户已确认。
- 已迁移。

#### 5.3.2 推荐写法

应写：

- 已评审。
- 已形成草案。
- 已完成 schema。
- 已生成 migration。
- 已完成 repo/usecase。
- 已完成 API/RBAC。
- 已完成 UI。
- 可内部联调。
- 可客户试用。
- 待客户确认。
- 待真实导入。
- 待交付验收。

#### 5.3.3 “已完成”必须附证据

任何“已完成”都必须附：

- 变更文件或当前真源路径。
- 实际执行的验证命令与结果。
- 明确未覆盖的内容、剩余风险和后续条件。
