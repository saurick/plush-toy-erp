## 使用边界

- `yoyoosun` 客户资料只能作为 Customer Material、Demo Seed、Industry Template Candidate、Print Template Input 或 Data Import Source。
- 原始 Excel 必须先由客户私有仓库 manifest 和 `scripts/import/customerSourceManifestCheck.mjs` 校验，再由 `scripts/import/customerSourceExtract.mjs` 按 `structuredExtract.enabled=true` 的来源生成客户私有 ignored `output/` evidence；该输出不纳入 Git，不是真实导入批准。
- `config/customers/yoyoosun/importConfig.mjs` 是从脱敏字段分类、产品核心边界和客户台账人工收口后的配置草案；只记录字段分组、review queue 和 forbidden targets，不嵌入私有 inventory、真实统计或 raw rows，不接 loader、不执行真实导入。
- Product Core 普通构建 / CI 只使用合成 fixture，不 checkout 客户私有仓库；客户私有验证采用兄弟目录或 CI multi-checkout，不使用 submodule。
- 运行时历史实现和删除前 evidence 不属于客户数据导入来源；真实导入只从已登记客户原件、确认后的映射和通用批次 usecase 进入。
- 本轮 import source inventory 不是 migration plan，也不是 loader spec。
- 任何涉及 shipment、inventory 或 finance facts 的来源只能进入 deferred / forbidden review，不得自动导入。
