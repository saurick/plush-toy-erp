# 需求线索 / Requirement Clues

本文件只记录需求线索，不等于已确认需求或 Product Core 规则。

## 客户 / 订单 / 款式

- 客户主数据、联系人、订单编号、客户订单号、款式编号、产品编号需要分层确认。
- 当前截图和样本可作为字段线索，不能直接决定唯一 schema。

## 产品 / SKU / BOM

- 成品、SKU、材料、BOM 需要区分主数据和采购 / 生产执行快照。
- BOM 是工程真源，采购页是执行快照入口。

## 采购

- 采购订单、采购入库、采购退货、采购入库调整需要区分源单据和事实单据。
- 采购合同和打印格式先进入客户打印样本记录，默认 Deferred；不直接进入行业模板候选。

## 委外

- 委外下单、发料、回货、返工、结算需确认是否属于行业通用能力。

## 入库 / 质检

- 来料质检已有最小事实表；后续需确认质检明细、缺陷字典和报告。

## 生产

- 生产排单、进度、延期、返工、异常先作为源单据和协同任务线索。

## 出货

- `shipment_release` 只代表放行，真实出货、库存扣减和 shipped 必须单独设计。

## 财务

- 应收、应付、发票、收付款和对账不能从 workflow done 直接生成。

## 手机端

- 角色移动端入口由权限控制，客户岗位名不同不应改核心流程。

## 权限

- 流程节点绑定职责 / 权限，不绑定岗位。

## 打印模板

- 合同和报表格式可作为客户样本记录，但数据来源和事实规则必须统一；暂不做通用打印模板内核。

## 部署交付

- current 是私有化部署实例，未来配置、env、发布清单和备份恢复进入 `deployments/current`。
