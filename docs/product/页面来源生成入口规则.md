# 页面来源生成入口规则 / Source-Generated Page Entry Policy

本文定义 ERP 业务页面的“新建 / 生成 / 登记”入口口径，避免来源生成单据、事实页和只读台账继续出现无来源的页面级新建按钮。

## 当前结论

由其他页面、来源单据、事实行或后续 usecase 生成的表单，不建议保留无来源的页面级“新建”按钮。页面可以提供来源选择 / 来源导入控件或“从 X 生成 / 生成 X 草稿”动作，但必须让用户看到来源、余额、状态和不可生成原因；如果真实后端路径还没完成，前端不放占位按钮。

| 页面类型 | 页面级入口口径 | 允许条件 | 禁止口径 |
| --- | --- | --- | --- |
| MasterData / 主数据 | 可以“新建” | 当前页面就是对象 owner，后端已完成 create / update / RBAC / 引用约束 | 不能用新建绕过启停、引用检查或客户配置 |
| Source Document / 源单据 | 可以“新建”或“从来源导入明细” | 当前页面拥有草稿生命周期；来源导入只填明细，不直接写事实 | 不能把销售订单、采购订单、委外订单直接说成库存、质检或财务事实 |
| 来源生成单据 | 用“生成 X 草稿 / 从 X 生成” | 来源对象、来源行、剩余量、状态和后端校验都可见 | 不使用无来源“新建 X” |
| Fact Action / 事实动作 | 用“过账 / 取消 / 冲正 / 判定 / 释放 / 消耗”等动作 | 后端 usecase、RBAC、幂等和状态机已完成 | 不用页面级“登记事实”代替来源生成 |
| Read Model / Ledger | 不显示创建入口 | 只读查询、筛选、导出或列配置 | 不放“新建 / 登记 / 生成”占位按钮 |
| Future / 未闭环能力 | 不显示按钮或降级为只读说明 | 后端合同、来源和测试未完成时先隐藏 | 不用前端本地表单伪造后端事实 |

## 23 个正式业务页面血缘

`web/src/erp/config/businessPageLineage.mjs` 是页面血缘的可执行注册表；每个 `formal-v1` 页面必须且只能登记一次。`producerActions` 只登记当前真实 RPC / 聚合保存动作，直接外键关系登记为上游页面，不能把“相关”误写成“已自动生成”。同一注册表还用 typed flow 区分目录带值、来源关联、领域生成、事实过账、只读投影、来源导航和打印快照；“能跳转”“有外键”“生成了下游事实”不能混写成同一种流。

主数据批量带值统一使用草稿级 `catalog_fill` 合同：调用页面必须显式声明 `append / replace` 和 `allow / skip / reject` 重复策略，选择器不替业务域全局去重。同一 SKU 或材料可因交期、价格、批次或部位拆成多行；带值只复制主档字段，不自动建立来源关系或过账事实。真实 Excel / 跨系统导入只有在后端具备校验、幂等、审计和错误报告后才能登记为 `external_import`；当前注册表保持为空。

| 页面 | 页面角色与入口 | 数据从哪里来 | 继续流向哪里 | 当前状态 |
| --- | --- | --- | --- | --- |
| 客户档案 | Owner；可新建 | 本页维护客户主数据 | 销售订单 | 已接入 |
| 供应商档案 | Owner；可新建 | 本页维护供应商 / 加工厂主数据 | 采购订单、委外订单 | 已接入 |
| 产品档案 | Owner；可新建产品与 SKU | 本页维护产品与规格主数据 | 销售订单、BOM、委外、生产、出货 | 已接入 |
| 材料档案 | Owner；可新建 | 本页维护材料主数据 | BOM、采购、委外、入库、库存 | 已接入 |
| 销售订单 | Source Document owner；可新建草稿 | 客户、产品 / SKU | 生产订单、库存预留、出货单 | 已接入；不会自动成为出货或财务事实 |
| 物料清单（BOM） | Owner；可新建版本草稿 | 产品、材料 | 生产订单 | 已接入；不会自动生成采购或库存 |
| 加工环节 | Owner；可新建 | 本页维护标准加工环节 | 委外订单 | 已接入；不承担完整工艺路线或排程 |
| 采购订单 | Source Document owner；可新建草稿 | 供应商、材料 | 采购入库 | 已接入；由已审核订单生成入库草稿 |
| 入库管理 | Source-generated；无通用新建或手工加行 | 已审核采购订单；退货 / 调整继续引用采购入库或质检来源 | 来料质检、库存、应付 | 已接入；创建入库草稿时逐行生成待检，全部通过后才允许入库确认并更新库存 |
| 质量检验 | Source-generated；只允许从正式来源生成或补建 | 采购入库草稿创建时逐行自动生成；委外回货从委外页面发起；出货前成品检验从草稿出货单按产品 / SKU、仓库、批次送检粒度发起 | 批次状态、采购退货、委外应付、出货来源追溯 | 三类来源入口已接入；检验判定不增减库存总量，也不启动或推进 Workflow |
| 库存台账 | Read model；无创建入口 | 已过账采购入库 / 退货 / 调整、生产事实、委外事实、出货以及库存预留投影 | 只读查询与来源追溯 | 已接入；当前无盘点、调拨或通用调整入口 |
| 委外订单 | Source Document owner；可新建草稿 | 供应商、产品 / 材料、加工环节 | 委外发料 / 回货、质检、库存、应付 | 已接入；发料 / 回货从已确认合同行生成 |
| 生产订单 | Source Document owner；可新建草稿 | 产品 / SKU，可关联销售订单与当前 BOM | 发布时原子生成排程任务；另可生成生产领料 / 完工事实 | 已接入；发布只生成协同排程，不等于已领料或已完工 |
| 生产排程 | Workflow inbox；无通用新建 | 生产订单 `DRAFT -> RELEASED` 原子生成 `production_scheduling` | 任务状态与事件；完成后订单才可关闭 | 已接入；责任岗位 PMC，任务完成不写生产事实 |
| 生产进度 | Fact processing；无通用新建 | 已发布生产订单生成领料 / 完工草稿，完工事实可生成返工草稿 | 库存变动；REWORK 过账原子生成生产异常任务 | 已接入；确认后才写库存事实 |
| 生产异常 | Workflow inbox；无通用新建 | REWORK 生产事实 `DRAFT -> POSTED` 原子生成 `production_exception` | 任务状态与事件；返工 / 报废 / 库存调整仍在业务页办理 | 已接入；责任岗位生产，任务完成不代替 Fact posted 或冲销 |
| 出货放行 | Workflow inbox；无通用新建 | 草稿出货单在来源页“提交出货放行”生成 `shipment_release` | 完成只写 `shipping_released`；实际出货仍回出货单办理 | 已接入；责任岗位仓库，放行不等于出货或扣库存 |
| 出库管理 | Fact processing；无通用新建 | 销售订单生成 / 释放库存预留；出货单确认消耗预留并写出库事实 | 出货单、库存 | 已接入；预留本身不是库存流水 |
| 出货单 | Fact owner；可新建草稿 | 产品 / SKU，可关联销售订单 | 出货前成品检验、出货放行任务、库存出库、应收、开票 | 已接入；成品检验必须在提交放行前完成，放行任务完成后才可确认出货，确认出货后才允许生成应收 / 开票 |
| 对账管理 | Fact processing；无通用新建 | 已过账应收、应付或发票 | 核对完成 / 取消 | 已接入；不等于付款或会计凭证 |
| 应付管理 | Fact processing；无通用新建 | 已过账采购入库或合格 / 让步委外回货 | 对账 | 已接入 |
| 应收管理 | Fact processing；无通用新建 | 已出货出货单 | 对账 | 已接入 |
| 发票管理 | Fact processing；无通用新建 | 已出货出货单 | 对账 | 已接入；系统记录不等于税控开票 |

## 单据流、事实流与任务流

1. 销售主链：客户 / 产品 → 销售订单 → 生产订单、库存预留或出货单；草稿出货单可按送检批次发起出货前成品检验，已有检验必须合格或让步接收。检验集合确定后从出货单提交放行，仓库完成 `shipment_release` 只形成 `shipping_released` 协同状态；随后仍须回出货单确认出货，才写库存流水，再按已出货来源生成应收 / 发票 → 对账。
2. 采购主链：供应商 / 材料 → 采购订单 → 采购入库草稿与逐行 IQC → 全部通过后入库确认 → 库存；采购入库还可继续办理退货 / 调整和应付。
3. 委外主链：供应商 / 产品 / 材料 / 加工环节 → 委外订单 → 发料 / 回货事实 → 质检、库存与应付。
4. 生产主链：产品 / 销售订单 / BOM → 生产订单发布 → PMC 排程任务 → 领料 / 完工 / 返工事实 → 库存；REWORK 过账同时生成生产异常任务。生产事实写入库存时，库存来源必须使用事实自身的 `PRODUCTION_FACT + fact.id`，不能回退成生产订单 ID；排程或异常任务完成也不能代替事实过账。
5. 三类来源任务统一保存 `workflow.source-task/v1 + source_task_producer + source_type + source_id + intent_hash`，使用确定性 task code；同意图重放返回原任务，不同意图冲突并回滚。公开 `workflow.create_task` 拒绝 `production_scheduling / production_exception / shipment_release`，试用任务只能使用 `trial_*` 组。普通协同任务无法证明 `source_id` 对应哪个业务表时，只能按可读单号筛选，不能猜 ID。

## 当前未闭环与停止线

- 三类来源任务已接入正式业务事件；来源取消不会把活动任务伪造成 `done / rejected`。生产订单关闭 / 取消、已过账 REWORK 冲销和实际出货均按各自终态规则 fail closed；若未来需要无条件撤销任务，应正式增加 `cancelled` 状态、migration、事件和看板语义，不能偷用现有终态。
- 入库页面已经退出无来源手工加行，公开 `add_purchase_receipt_item` 也要求采购订单行来源；但公开 `create_purchase_receipt_draft / create_purchase_receipt_with_items` 仍可创建无采购订单来源的记录。后两者退出、改为受控导入来源或补齐独立来源真源前，不能宣称采购入库后端全链只允许采购订单来源。
- 自动生成的初始逐行 IQC 与已过账入库后的追加检验必须分开：初始 IQC 的 `REJECTED` 只阻断入库，不形成库存退货；现有质检来源“退供应商”动作要求来源采购入库已 `POSTED`，不能把两者描述成同一步。
- `SHIPMENT / FINISHED_GOODS` 是出货前检验侧链，不是“生产完工 → 成品质检 → 成品入库”主链。当前来源粒度是同一出货单内的产品 / SKU、仓库和批次组合；同组合的重复出货明细聚合为一个送检批次，不声称逐出货行留痕。若未来必须逐行审计，需要新增正式来源行锚点和 migration，不能用备注或前端映射伪造。
- `OUTSOURCING_FACT`、`FINANCE_FACT`、`PURCHASE_RETURN`、`PRODUCTION_ORDER_MATERIAL_REQUIREMENT` 等间接来源，要精确跳回最初单据时仍需后端返回可解析的来源投影；页面不得猜 ID 或用同名列表替代精确追溯。
- 没有真实 producer、状态规则、RBAC、幂等、审计和测试时，停止在只读 / deferred 口径；不能用前端表单补造业务来源。

## 新增按钮检查清单

新增或恢复业务页主按钮前，先回答：

1. 当前页面是不是该对象的唯一 owner。
2. 如果不是 owner，来源对象、来源行、剩余量、状态和不可生成原因是否可见。
3. 后端 usecase、JSON-RPC、RBAC、审计、幂等和状态机是否已经完成。
4. 按钮文案是否准确区分“新建”“生成草稿”“过账 / 冲正 / 取消 / 判定”“发起协同”。
5. 表单、表格、筛选、导出列是否只展示业务字段，不展示幂等键、内部主键、内部引用、trace / request ID、裸数据库 ID、`#123` 兜底、ID 选择控件或手填 `source_id` / `source_line_id`；这些字段如需提交，只能隐藏携带或由后端生成。
6. L1 页面回归或静态测试是否断言了按钮存在 / 不存在，以及技术字段不会重新出现在用户可见层。
