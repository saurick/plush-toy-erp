# 页面来源生成入口规则 / Source-Generated Page Entry Policy

本文定义 ERP 业务页面的“新建 / 生成 / 登记”入口口径，避免来源生成单据、事实页和只读台账继续出现无来源的页面级新建按钮。

## 当前结论

由其他页面、来源单据、事实行或后续 usecase 生成的表单，不建议保留无来源的页面级“新建”按钮。页面可以提供来源选择器或“从 X 生成 / 生成 X 草稿”动作，但必须让用户看到来源、余额、状态和不可生成原因；如果真实后端路径还没完成，前端不放占位按钮。

| 页面类型 | 页面级入口口径 | 允许条件 | 禁止口径 |
| --- | --- | --- | --- |
| MasterData / 主数据 | 可以“新建” | 当前页面就是对象 owner，后端已完成 create / update / RBAC / 引用约束 | 不能用新建绕过启停、引用检查或客户配置 |
| Source Document / 源单据 | 可以“新建”或“从来源导入明细” | 当前页面拥有草稿生命周期；来源导入只填明细，不直接写事实 | 不能把销售订单、采购订单、委外订单直接说成库存、质检或财务事实 |
| 来源生成单据 | 用“生成 X 草稿 / 从 X 生成” | 来源对象、来源行、剩余量、状态和后端校验都可见 | 不使用无来源“新建 X” |
| Fact Action / 事实动作 | 用“过账 / 取消 / 冲正 / 判定 / 释放 / 消耗”等动作 | 后端 usecase、RBAC、幂等和状态机已完成 | 不用页面级“登记事实”代替来源生成 |
| Read Model / Ledger | 不显示创建入口 | 只读查询、筛选、导出或列配置 | 不放“新建 / 登记 / 生成”占位按钮 |
| Future / 未闭环能力 | 不显示按钮或降级为只读说明 | 后端合同、来源和测试未完成时先隐藏 | 不用前端本地表单伪造后端事实 |

## 当前页面分类

| 页面 | 当前入口 | 处理口径 | 说明 |
| --- | --- | --- | --- |
| 客户、供应商、联系人、材料、产品、仓库、BOM、工序 | 保留“新建” | 主数据 / owner 页面 | 退出默认走启停或版本 / 草稿规则，不能补通用回收站 |
| 销售订单、采购订单、委外订单、出货单 | 保留“新建”或“从来源导入明细” | Source Document / owner 页面 | 出货明细可从销售订单行导入，但出货单仍是出货事实 owner |
| 采购入库 | 不显示页面级“新建入库单” | 从采购订单生成入库草稿 | 入库页面只处理已生成草稿的加行、过账、取消和列表 |
| 来料质检 | 显示“生成质检草稿” | 从采购入库行生成 | 表单必须选择采购入库、入库行和批次；本次送检数量等候后端合同 |
| 库存流水 | 不显示创建入口 | Ledger / read model | 库存变动来自采购入库、出货、调整或冲正等事实动作 |
| 出库管理 / 库存预留 | 不显示页面级“登记库存预留” | 来源生成 / 事实处理页 | 当前页只处理释放、消耗和查询；来源自动预留或手工来源生成另走后端合同 |
| 应收、应付、发票、对账 | 不显示页面级“登记事实” | Finance fact read / processing page | 应来自真实出货、采购、委外、应收应付或对账来源；没有来源生成 usecase 前不放按钮 |
| 生产发料 / 入库事实 | 暂保留“登记生产事实” | 当前页面是生产事实登记 owner | 后续若接入生产任务来源，再按来源生成规则收口 |
| Workflow 协同页 | 用“发起 X 协同” | Workflow task / business status | 协同完成不等于库存、出货、质检或财务 fact posted |

## 新增按钮检查清单

新增或恢复业务页主按钮前，先回答：

1. 当前页面是不是该对象的唯一 owner。
2. 如果不是 owner，来源对象、来源行、剩余量、状态和不可生成原因是否可见。
3. 后端 usecase、JSON-RPC、RBAC、审计、幂等和状态机是否已经完成。
4. 按钮文案是否准确区分“新建”“生成草稿”“过账 / 冲正 / 取消 / 判定”“发起协同”。
5. L1 页面回归是否断言了按钮存在或不存在，避免未来回退。
