# 正式产品入口与菜单配置计划

- 文档类型：产品路线配套计划
- 状态：规划输入，不是 runtime 真源
- 对应 roadmap 阶段：第 12 阶段 `建立正式产品入口`
- 对应建议路线：`013 v1-formal-menu-and-legacy-entry-exit`

## 0. 文档目的

本文只承接正式产品入口、行业默认菜单候选、客户菜单配置和旧入口退出的规划细节。

本文不是产品总路线图，不替代 `docs/product/product-completion-roadmap.md`。本文也不是当前菜单 runtime、seedData、docs registry、权限码、路由或前端实现真源；任何实际修改仍必须拆到具体 `docs/codex-goals/*.md`。

## 1. 产品目标

第 12 阶段的产品目标是把 V1 正式页面从旧通用入口、直链或兼容入口切到正式产品入口，并让不同客户通过配置控制可见菜单，而不是复制菜单代码。

必须满足：

- 客户、供应商、联系人、销售订单等 V1 正式能力有稳定入口。
- 旧 `business_records` 重叠入口退出正式业务写路径。
- 菜单隐藏只影响入口可见性，不替代后端动作权限、状态机和事实校验。
- 移动端任务入口按任务和职责投影，不复制桌面菜单树。
- 帮助文档入口跟随产品能力和客户启用状态；打印入口只在已有客户交付样本范围内展示，不代表当前要做通用打印模板内核。

## 2. 菜单目标结构

| 层级 | 目标 | 说明 |
| --- | --- | --- |
| Product capability baseline | 定义产品内核真实能力 | 只描述产品具备哪些通用能力，不写客户差异 |
| Default industry menu | 行业默认菜单 | 从产品能力形成毛绒玩具行业默认菜单候选 |
| Customer menu config | 客户菜单配置 | 控制每客户启用 / 隐藏 / 排序 / 文案 / 默认入口 |
| RBAC action permission | 后端动作权限 | 决定用户能否执行动作，不能被菜单隐藏替代 |
| Mobile task entry | 手机端岗位入口 | 从任务和职责投影，不直接复制桌面菜单 |
| Docs / print entry | 帮助文档和客户打印样本入口 | 帮助文档跟随产品能力展示；打印样本入口只按客户交付范围展示 |
| Legacy removal marker | 旧入口退出标记 | 只说明旧入口已退出正式产品入口，不承诺客户可见归档页 |

菜单调整后的结果应是：

- 同一套产品代码支持不同客户菜单。
- 不同客户通过配置隐藏、启用、排序和改显示文案。
- 默认行业菜单可以作为初始模板，但不是每个客户必须完全一致。
- 菜单配置不改变库存、出货、财务、审计等核心事实规则。
- 菜单隐藏只影响可见入口，不替代后端权限校验。
- 旧 `business_records` 重叠入口不再作为正式业务菜单；如需保留历史数据，只作为迁移、审计或回滚参考。

## 3. 入口颗粒度规则

菜单命名和入口颗粒度必须稳定，不应把多个流程节点用 `/` 绑成一个长期菜单名。这里不要求把左侧菜单改成可展开的多级树；当前“分组标题 + 功能入口”的形态可以继续存在，但功能入口本身必须表达稳定业务对象或管理页。流程步骤应放在页面内 Tab、按钮、待办、移动端任务或工作流节点里。

| 层级 | 应表达什么 | 不应表达什么 |
| --- | --- | --- |
| 菜单分组 | 稳定业务域，如销售、采购、仓储、质检、生产、委外、出货、财务、系统 | 多个流程节点拼接名 |
| 功能入口 | 稳定业务对象或管理页，如销售订单、入库管理、库存台账、出货单、应收管理 | 一个名字塞入库通知 / 检验 / 入库等多个动作 |
| 页面 Tab / 分段 | 同一对象下的状态视图或流程段，如待质检、待入库、已入库 | 替代后端状态机或事实判断 |
| 按钮 / 动作 | 提交、审核、放行、确认入库、确认出库、作废、冲正 | 作为菜单权限或事实真源 |
| 任务 / 移动端入口 | 按职责投影待办，如质检确认、仓库入库、出货放行 | 复制桌面菜单树 |

## 5. 当前组合入口拆分输入

当前 UI 中把多个业务对象绑在一起的组合入口只作为“待拆依据”，不作为正式菜单名，也不作为后续客户交付入口。后续应拆成稳定产品能力，再由默认行业菜单和客户菜单配置组合出可见菜单。

| 现有组合入口（待拆，不作为正式菜单名） | 拆分后的正式入口方向 | 能力归类 | 产品默认 | 客户菜单配置 | 业务约束 |
| --- | --- | --- | --- | --- | --- |
| 任务看板 | 任务中心 | Work Queue / Workflow | 必须有可达入口 | 可按角色换位置或改名，不应全局关闭 | 如果有任务流转，必须至少有任务入口或移动端任务入口 |
| 业务看板 | 经营看板 | Dashboard / Reporting | 默认可见 | 可隐藏 | 不能作为业务事实或状态机真源 |
| 客户与供应商组合入口 | 客户档案 | MasterData | current MVP 必须 | 可对非相关角色隐藏 | 订单、导入依赖时不能禁用对应后端能力 |
| 客户与供应商组合入口 | 供应商档案 | MasterData | current MVP 必须 | 可对非相关角色隐藏 | 采购、导入依赖时不能禁用对应后端能力 |
| 产品 | 产品档案 | MasterData | current MVP 必须 | 可对非相关角色隐藏 | 订单、BOM、库存依赖时不能禁用对应后端能力 |
| 订单与款式立项组合入口 | 销售订单 | Source Document | current MVP 必须 | 无销售链路客户可隐藏 | 只是源单据，不自动生成出货、库存、财务事实 |
| 订单与款式立项组合入口 | 款式立项 | Product Planning | 后续评审 | 未启用产品工程客户可隐藏 | 款式资料、打样和评审状态不和销售订单共用正式菜单名 |

| 加工合同与委外下单组合入口 | 委外订单 | Outsourcing Source Document | 行业默认可选 | 无委外客户可隐藏 | 隐藏后不得把任务派给不可见且无权限的委外入口 |

| 待出货与出货放行组合入口 | 出货单 | Shipment Fact | 后续评审 | 未启用出货链路客户可隐藏 | shipped 后才是出货事实 |
| 出库 | 出库管理 | Shipment / Inventory Outbound | 后续评审 | 可在出库事实未启用时隐藏 | 启用后必须走正式 Shipment / Inventory usecase，不从 workflow 直接扣库存 |
| 生产排单 | 生产排程 | Production Planning | 后续评审 | 无生产排程客户可隐藏 | 隐藏后不得创建只能由生产排程入口处理的任务 |
| 生产进度 | 生产订单 | Production Execution | 后续评审 | 无生产进度客户可隐藏 | 启用后生产事实与 workflow 协同状态必须分开 |

## 6. 菜单隐藏与权限关闭后的业务保证

菜单隐藏、客户未启用能力、用户无权限是三种不同状态，不能混用。

| 状态 | 含义 | 后端要求 | Workflow / 业务保证 |
| --- | --- | --- | --- |
| 菜单隐藏 | 客户或角色看不到入口 | 不改变后端事实规则和动作权限 | 任务仍可通过任务看板、移动端或其他被授权入口处理 |
| 功能未启用 | 客户配置未启用某能力 | 对应 API / usecase 不应作为正常业务入口开放 | 上游流程不得生成只能由该能力处理的任务；必须停在已定义的 deferred / disabled / blocked 状态并给出原因 |
| 用户无权限 | 当前用户没有动作权限 | 后端拒绝动作，前端隐藏按钮只是体验 | 任务不能派给无职责角色；已派错时应进入可审计的改派 / 阻塞 / 退回路径 |
| 旧入口移除 / 隐藏 | 旧入口退出产品入口 | 禁止写入正式业务事实 | 不再作为客户可见能力；历史数据如需保留，只用于迁移、审计或回滚 |

核心规则：

- 客户可以组合菜单，但不能组合出破坏事实层的业务规则。
- 隐藏某个菜单前，必须确认没有 active workflow task 只能通过该入口处理。
- 关闭某个能力前，必须确认上游状态机不会继续派生该能力的待办任务。
- 如果任务已存在但入口被隐藏，系统必须提供任务看板、移动端或管理员改派 / 退回 / 阻塞路径。
- 后端 usecase、RBAC action permission、owner_role_key、assignee_id 和 task_status_key 才是状态机与业务事实的保护层。
- Workflow 可以派生协同任务和业务状态，但不能因为菜单存在或隐藏而直接写库存、出货或财务事实。
