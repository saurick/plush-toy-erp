## 当前硬边界

- `workflow_tasks` 是协同任务真源，不是库存、质检、出货、财务事实真源。
- ProcessRuntime 只通过明确的 domain command handler 进入业务 usecase；payload 里的 `command_key` 不被采信为事实命令。
- 运行时先调用 handler 的无副作用只读预检，再用单条条件 `UPDATE ... RETURNING` 原子 claim `command_key + idempotency_key + JSON payload` 的 SHA-256 fingerprint；确定性坏参数不会先污染 active node。protocol v1 的 6 个写 handler 在各自领域事务内同时落 side effect 与 durable result / effect ref；两个只读 gate 在锁定对象的短事务内记录 `effect_state=none`。runtime 返回后的 `RecordResult` 只做 exact replay。
- 重试先读取、校验并重放持久结果，不再次 Validate 或执行 handler；节点结算、下游激活、WorkflowTask 创建和 `linked_business_refs` CAS 失败可继续推进对账。结果冲突会让同一领域事务回滚，不留下“业务已写但 result 未写”的 v1 窗口。
- 销售订单取消、采购入库取消冲正、已出货取消冲正和财务事实取消会在领域事务内标记 compensation；认证 JSON-RPC 主路径同时记录补偿操作人。active 节点读取 compensated result 会阻塞流程，已完成节点在补偿后重放会返回 `40921` 且不再推进。补偿后若只激活了尚无领域副作用的下游，管理员可用唯一决策 `terminate_and_withdraw_downstream` 撤回 active / blocked 节点、拒绝其关联任务并阻塞流程；任何下游已完成或已有 effect ref 均 fail closed 转人工核对。质量结果只允许在没有已生效下游处置、财务或出货事实时生成新的 `SUBMITTED` 更正单，原判定保留并标记 superseded；客户退货质检因隔离库存可能已有下游，当前不开放结果更正。
- migration `20260710150000` 只给升级前 active 空 fingerprint 写 fail-closed sentinel；`20260710150001` 才增加 durable result / hash / effect-ref / compensation schema，并把旧 fingerprint 标为 protocol 0。protocol 0、已提交销售订单但缺 exact result，以及缺少可证明判定时间的旧成品质检 result-missing 仍需 `40921` 人工核对。本地 PostgreSQL 已覆盖原子回滚、exact recovery、payload conflict、compensation 和 claim / linked-ref 并发；目标环境尚未发布。
- handler payload 只保留会参与领域结果的字段：`inventory.post_inbound / shipment.finance_release / shipment.ship` 只接收各自主引用；成品质检执行人取认证 actor；应收客户取已出货 shipment 的 customer 真源，不接受 caller 自报 customer 或尚未落库的 due date。
- `quality_inspection.aggregate_gate` 只按 `purchase_receipt_id` 聚合读取逐行正式质检结果，不接收质检单 ID 或判定结果，也不替代 `QualityInspection` 逐行判定。全部 `PASS / CONCESSION` 才推进；任一 `REJECT` 阻塞；仍有未判定行时保持当前节点 active。它只更新 ProcessRuntime 的推进或阻塞状态，不写质检或库存 Fact。
- `inventory.post_inbound` 仍会在同一事务内重新校验全部收货行、正式质检结果和 `ACTIVE` lot，流程门禁通过不能替代入库 usecase 自身校验。
- businessFlows / stateMachines / processPolicies 当前仍是客户配置 preview / lint / manifest 层，不是 runtime loader 或 execute 真源。
- 状态机 preview 不覆盖现有 usecase 生命周期状态规则。
- processPolicies 不允许跳过后端权限、状态、幂等、引用和 Fact 写入校验。
- 发布证据若没有 authenticated effective session active revision readback 和客户签收，只能标记 internal-only。
