# 正式菜单候选原型 / Formal Menu Candidate Prototype

- 阶段：待实现 / To Implement
- 类型：HTML 原型样板
- 来源：`docs/reference/第二次20260611/产品核心菜单与页面功能规格.md`、`docs/product/正式产品入口与菜单配置计划.md`、`docs/product/菜单映射评审表.md`、当前运行时菜单截图与已有原型样板
- 边界：Reference Only，不替代当前 `seedData.mjs`、客户菜单配置、路由、RBAC、schema、API、WorkflowUsecase 或 Fact usecase

## 目标

本样板把 `core-menu-coverage-v1` 中的 51 个内部覆盖项压缩成 12 个正式菜单候选主入口，用于评审“哪些入口应该出现在左侧导航，哪些细项应进入页面内 tab、筛选、动作按钮或详情区”。

它不是正式菜单改造实现，也不代表要一次性重构当前侧边栏。

进入真实菜单、路由、seed 或权限实现前，必须先按 `docs/product/菜单映射评审表.md` 完成当前运行时菜单、12 个候选主入口和 51 个内部覆盖细项的映射评审。

## 候选入口

| 正式候选入口 | 吸收范围 | 页面内承载方式 |
| --- | --- | --- |
| 工作台 | ERP 总览、异常提醒 | 总览卡片、风险队列、当前任务关联记录 |
| 我的任务 / 任务看板 | 我的任务、我发起的 | 待办、发起记录、处理动作 |
| 客户 / 供应商 / 产品 / BOM | 主数据与产品结构 | tab、分组筛选、详情区 |
| 销售订单 | 销售订单与明细 | 列表、详情、明细表 |
| 采购 / 入库 / 质检 | 采购、收货、退货、质检 | 状态 tab、任务入口、动作浮层 |
| 库存 | 余额、流水、批次、预留、调整、调拨 | 台账 tab、过滤、调整动作 |
| 生产 / 外协 | 生产任务、上报、领料、外协发料 / 回货 | 计划与执行 tab、动作浮层 |
| 出货 | 出货单、装箱 / 发货明细 | 出货状态、装箱明细、放行动作 |
| 财务对账 | 应收、应付、收付款 | 对账视图、收付款记录、详情 |
| 报表 | 订单、库存、交付看板 | 只读看板 |
| 数据导入 | 导入任务、预检、未匹配队列 | 导入工作台、预检报告、人工匹配 |
| 系统管理 | 用户、角色、审计、客户配置、系统配置、备份恢复、历史归档 | 管理页、审计视图、运维入口 |

## 不做内容

- 不把 51 个内部覆盖项直接改成左侧导航。
- 不改当前正式菜单、客户菜单、导航 seed 或后端内置菜单。
- 不新增正式路由、权限码、API、schema、migration 或业务事实写入。
- 不把 Workflow task done 写成库存、出货、财务等事实过账。
- 不把客户配置、备份恢复或历史归档提前提升为 Product Core runtime 能力。

## 验收建议

- 静态原型打开后应显示 12 个主入口和 51 个内部覆盖项。
- 点击每个入口时，右侧详情应展示吸收范围、页面内承载方式和边界。
- 搜索“库存流水”“Legacy”“采购收货”等细项时，应定位到对应主入口。
- 移动端视口不应出现横向溢出。
