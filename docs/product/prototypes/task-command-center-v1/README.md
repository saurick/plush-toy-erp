# 任务中心样板 / Task Command Center Pattern

本目录保存后台“我的任务 / 任务看板”的 To Implement 标准样板，用于重新确认任务入口的意义。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，把任务入口收敛为职责处理台：待我处理、我发起的、阻塞交接、当前任务详情和关联业务对象。 |

## 设计结论

- 任务菜单的意义不是复制业务菜单，也不是给每个流程节点新增入口；它只回答“哪些任务现在需要我处理、我发起的卡在哪里、阻塞该交给谁”。
- 任务中心只处理 Workflow 协同任务、原因、催办、改派和跳转；任务完成不等于库存、出货、财务、发票或收付款事实过账。
- 业务对象只作为任务上下文和跳转入口出现，例如销售订单、入库通知/检验/入库、出货放行、单据模板；它们不替代正式业务菜单。
- 岗位任务端仍按职责投影，不复制桌面菜单树；桌面任务中心用于跨角色管理、跟进和交接。

## 待实现吸收状态

截至 2026-06-13，本原型按 `待实现 / To Implement` 登记。当前运行时已有 `/erp/task-board` 和岗位任务端 `/m/<role>/tasks`；本样板只说明后续可如何让桌面任务入口更有意义，不直接改运行时。

| 原型区域 | 当前运行时参考路径 | 尚待人工确认点 | 边界 |
| --- | --- | --- | --- |
| 职责队列 | `/erp/task-board`、`/m/<role>/tasks` | 是否按“待我处理 / 我发起的 / 阻塞交接”作为桌面首层筛选。 | 只查 Workflow 任务，不写 Fact。 |
| 当前任务详情 | `/erp/task-board` 详情区 | 是否把原因、下一步、关联对象和建议动作做成首屏核心。 | 完成 / 阻塞 / 驳回必须走现有 Workflow API。 |
| 关联业务对象 | 业务模块正式入口 | 是否从任务直接跳转到关联对象页。 | 跳转不是菜单替代，也不绕过 RBAC。 |
| 交接边界 | 异常 / 阻塞闭环、任务事件 | 是否保留催办、改派和交接摘要。 | 交接只写任务事件，不修正业务事实。 |

明确不直接吸收：原型里的固定数字、静态任务、mock 脚本和任务菜单分组不进入正式运行时；真实实现必须重新连接当前 Workflow API、RBAC、owner_role_key、assignee_id 和 task_status_key。

## 验收关注点

- 筛选可切换，当前任务详情随选择变化。
- 当前任务的关联业务对象会高亮，但明确只是跳转上下文。
- 桌面和移动视口不出现页面级横向溢出。
- 文案不能把任务完成、异常关闭或催办写成事实层过账结果。
