# 审计日志页原型 / Audit Log Page Prototype

本目录保存系统管理下 `审计日志` 页的 `待实现 / To Implement` 原型。它用于把当前页面从“空表壳”收敛为可追溯、可筛选、可快速定位风险事件的审计工具。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖审计摘要、筛选分组、日志表格、事件详情、空态和风险事件态。 |

## 选定方向

- 首屏先回答“最近有没有高风险系统控制面事件”，再进入筛选和明细。
- 表格仍是主路径，但不把页面做成单一大空表；右侧事件详情承接 payload、变更前后、操作者和追溯说明。
- 空态不只显示插画，应说明当前筛选条件、可调整的筛选项，以及审计日志不替代业务事实。
- 事件类型按系统控制面归类：账号、角色权限、初始化和安全阻止；业务事实审计仍回到各领域事实表和正式文档。

## 放弃方向

- 放弃“标题说明 + 筛选 + 单表格”的当前壳层方向，因为它在无数据时过空，在有数据时也不能快速判断风险和追溯对象。
- 不把审计日志做成业务看板或操作中心；本页只读，不承接启停账号、重置密码、修改权限等真实操作。
- 不在原型里新增导出审批、审计报表、告警订阅或保留策略配置；这些属于后续系统治理能力。

## 吸收范围

后续进入真实页面时优先吸收：

1. 顶部审计摘要：总事件、高风险、被阻止初始化、最近事件时间。
2. 筛选结构：来源、动作、风险、对象 / 操作者 / payload 关键词、时间范围。
3. 表格结构：时间、风险、来源、动作、操作者、对象、摘要。
4. 事件详情区：事件 key、request id、payload 摘要、before / after 变更、只读边界说明。
5. 空态、风险筛选态和详情关闭 / 恢复态。

## 不吸收范围

- 不新增后端 API、schema、migration、RBAC 权限码或菜单入口。
- 不把 mock 事件、固定数字、request id、IP 地址或时间写入 seed / runtime。
- 不把系统审计日志升级成业务事实真源；库存、出货、财务、应收、发票和收付款仍回到对应 Fact usecase。
- 不把本原型状态改为 `Current`，除非真实页面完成吸收、测试和浏览器回归后由用户明确确认。

## 验收关注点

- 桌面首屏不空洞，日志表和详情区能同时扫描。
- 空态、默认有数据态、风险筛选态都有明确视觉反馈。
- 长 event key、payload 摘要、对象名和 request id 不溢出容器。
- 浅色和暗色都可读，风险色不过度占据页面。
