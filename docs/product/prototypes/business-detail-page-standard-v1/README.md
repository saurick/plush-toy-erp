# 业务详情页样板 / Business Detail Page Standard

本目录保存后台业务详情页的 `待吸收实现 / To Implement` 标准样板。它用于说明同类详情页如何复用统一结构，而不是为每个菜单单独设计。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖基础信息、业务状态、关联单据、操作记录、附件区，以及 Workflow 协同动作和 Fact 事实动作的视觉边界。 |

## 标准样板口径

- 适用页面：销售订单详情、采购入库详情、质检详情、库存批次详情、出货放行详情、财务事实详情等“单个业务对象 + 状态 + 关联链路 + 操作记录”的页面。
- 同类菜单默认参照本样板，不逐菜单重做原型；字段、状态、关联单据和允许动作按各自正式 usecase / API / RBAC 填充。
- Workflow 协同动作和 Fact 事实动作必须分区展示：协同动作只处理任务状态、原因和催办；事实动作只在对应领域 usecase 允许时出现，不由协同任务完成自动触发。
- 附件区只表达图纸、合同、质检照片和交付凭据等资料入口，不新增真实附件服务、OCR、打印留档或反向写事实。

## 边界

- 本样板保持 `To Implement`，不晋级 `Current`。
- 不替代正式菜单、权限、路由、业务语义、Workflow / Fact 真源、schema、migration 或后端 API。
- 不把协同任务完成写成库存、出货、财务、开票或收付款事实。
- 打印 / 导入 / 导出 / 帮助等辅助动作只作为弱入口参照，不在本轮单独做完整页面。

## 验收关注点

- 桌面和移动视口不横向溢出。
- 基础信息、状态、关联单据、操作记录和附件区均可扫描。
- Workflow / Fact 动作区可展开切换，且视觉语义不同。
- 浅色 / 暗色均可读，状态色不过度使用。
