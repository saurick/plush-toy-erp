# 产品原型资产 / Product Prototype Assets

本目录保存可长期追溯的产品原型资产，用于评审和指导后续实现。

## 边界

- 本目录不是运行时页面目录，不进入 `web/src`、菜单、路由或生产构建。
- 本目录不替代 `docs/current-source-of-truth.md`、代码、migration 或测试。
- `index.html` 是本目录的轻量静态查看器，只用于浏览原型资产；它不接后端、不接权限、不进入 ERP 正式菜单，也不作为开发文档 registry。
- 本地前端开发环境额外提供 `/__dev/prototypes` 原型查看器，用于在 `http://localhost:5175` 下按目录浏览、折叠、置顶和预览这些资产；该入口仍是 dev-only，不进入生产构建和正式菜单。
- 原型用于确认信息架构、交互入口和视觉方向；真正实现时仍应回到共享组件、真实数据接口和项目测试。
- 若原型结论改变业务能力、菜单入口、Workflow / Fact 边界或交付状态，必须另行同步正式产品 / 架构文档。

## 格式约定

- 页面和交互原型默认使用 HTML，例如后台业务页、移动端页面、表格分页、筛选、抽屉、展开 / 收起、选中操作条和协同入口。
- 流程、架构、菜单结构、状态关系、Workflow / Fact 边界优先写在 Markdown 中，并按需要使用 Mermaid。
- PNG 用于早期视觉方向、截图证据、方案对比和历史参考；不要为了统一格式把未采纳或仅供参考的 PNG 强行复刻成 HTML。
- 某个 PNG 方向被确认要落地时，应重新做可交互 HTML 原型或直接进入真实页面实现，而不是把图片机械转换为静态 HTML。
- 真实页面落地后，以 `web/src` 运行时代码、自动化测试和浏览器回归结果为准；HTML 原型退为历史参考。

## 资产状态

| 状态 | 维护口径 |
| --- | --- |
| 当前实现对齐版 | 用于说明真实页面或当前推荐形态已经吸收后的 as-built 参考；真实口径仍以运行时代码、测试和浏览器回归为准。 |
| 探索方案 | 用于评审信息架构、交互入口或页面骨架，尚不代表正式实现承诺。 |
| 历史参考 | 用于追溯旧方向、旧视觉或已退为参考的原型，不再作为新实现默认真源。 |
| 截图证据 | 用于保留曾参与评审的视觉或页面证据，不能单独替代代码、测试或正式文档。 |
| 方案对比 | 用于并排比较多个方向，后续采纳结论应写回对应 README 或正式设计文档。 |

## 使用方式

直接用浏览器打开本目录的 `index.html`，或在本地前端开发环境访问 `http://localhost:5175/__dev/prototypes`。两个查看器都只做资产浏览和全屏预览；PNG 资产通过 `index.html`、`/__dev/prototypes` 或所在 `images/` 目录访问。开发态 `/__dev/prototypes` 额外提供目录分组折叠和浏览器本地置顶偏好，用于原型数量增加后的日常查找；这些偏好不写入后端、不改变资产状态，也不进入正式菜单或权限体系。

实现时优先把共性能力收口到业务列表页共享组件，而不是每个模块单独复制原型结构。采购合同、应收 / 开票、发票登记、出库、入库通知等“列表 + 单据操作 + 协同入口”的页面可先套用标准页；库存余额、库存流水、生产排单、生产进度、质检等信息结构明显不同的页面应单独评审变体。

维护 `index.html` 时保持手工登记和相对链接，不新增依赖或构建工具；新增资产后同步补充名称、类型、状态、所属目录、用途和打开入口。全屏预览只能作为查看辅助，不改变资产状态或正式实现口径。
