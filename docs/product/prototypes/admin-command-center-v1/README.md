# 后台工作台样板 / Admin Workbench Pattern

本目录保存后台首页 / 工作台的待实现样板。它只解决一个问题：当前账号今天先处理哪一项协同任务，以及选中后需要核对哪些上下文。

## 阶段与文件

- 阶段：待实现 / To Implement
- 归属：产品内核 / Core
- 原型：`index.html`
- 方向图：`images/workbench-redesign-reference.png`
- 适用：桌面工作台的优先队列与当前任务 master-detail

本资产未晋级为 Current。原型中的编号、数量、岗位和记录均为中性演示数据，不是当前数据库、客户现场或后端返回值。

方向图只用于快速比较信息层级；可点击、键盘、响应式和状态恢复以 `index.html` 为评审主件，运行事实以当前代码与浏览器回归为准。

## 设计结论

- 工作台使用“优先队列 + 当前任务详情”两栏结构。左侧负责判断先后顺序，右侧负责核对影响、责任、来源、到期和关联记录。
- 页面只保留两类真实交互：切换队列、选择当前任务。两者都会更新页面状态并提供明确选中反馈。
- 删除页面级刷新、通用快捷入口、重复菜单入口和没有真实执行结果的“处理 / 催办 / 记录阻塞”演示按钮。
- 关联记录是当前任务的只读上下文，不再做成一组放大的快捷卡。真实实现只有在目标路由、菜单和权限已经确认后，才能提供真实链接。
- “处理提示”是只读的原型规则示例，不是人工填写的计划。正式运行时应由任务状态、当前账号可用操作和来源入口共同派生；来源入口还必须通过当前账号的页面权限投影，无法可靠派生时不应补造提示。
- 完成协同任务不代表采购入库、库存、出货、生产或财务记录已经完成。

## 信息结构

| 区域 | 回答的问题 | 元素语义 | 边界 |
| --- | --- | --- | --- |
| 队列切换 | 当前先看待我处理、风险还是等待交接 | 真实 `button / tab` | 只改变当前列表，不修改业务数据 |
| 任务主列表 | 当前队列里先处理哪一项 | 真实 `button` 选择 | 选择后只更新右侧上下文 |
| 当前任务详情 | 为什么现在处理、由谁处理、去哪里处理 | 只读详情 | 不在工作台保存业务记录 |
| 关联记录 | 当前任务与哪些业务记录有关 | 只读行 | 不复制完整菜单或通用入口 |
| 处理提示 | 当前可以执行什么操作或去哪里核对 | 只读规则示例 | 正式运行时按任务状态、账号可用操作与已授权来源入口派生 |

## 交互与可访问性

- 队列使用 `tablist / tab / tabpanel`，当前项同步 `aria-selected` 和 roving `tabindex`。
- 键盘可用 `Left / Right / Home / End` 切换队列。
- 任务行使用真实按钮和 `aria-pressed` 表达当前项；`Up / Down / Home / End` 可切换任务并同步详情。
- hover、focus 和 selected 三种状态相互独立；焦点不依赖颜色变化判断。
- 980px 以下改为上下 master-detail，660px 以下队列指标、上下文字段和关联记录改为单列。
- 浅色与暗色均使用同一信息层级；`prefers-reduced-motion` 下关闭非必要动效。

## 当前运行时参考

运行时 `/erp/dashboard` 已吸收低密度、待我处理 / 阻塞逾期两类队列、主从详情、只读处理提示和单一“处理任务”入口，但这不把本资产晋级为 Current。原型中的等待交接变体、完整键盘矩阵和状态演示仍是后续评审输入，不证明 API、权限或客户菜单已经完成同等重构。

真实实现时应：

1. 从当前账号可见的任务真源读取队列，不从前端样例推导任务或数量。
2. 复用全局刷新入口，不恢复页面级刷新。
3. 只有存在正式目标页面时才渲染真实链接；无目标时保持只读说明。
4. 保持 Workflow 协同与业务记录办理边界，不由工作台直接写库存、出货或财务事实。

## 验收关注点

- 默认、队列切换、任务切换、键盘焦点和恢复态均有明确视觉反馈。
- 切换队列后不保留不属于当前队列的旧任务详情。
- 长标题、长业务编号和多岗位文本不造成页面级横向溢出。
- 桌面、窄屏和暗色下都能读清任务优先级与当前选择。
- 页面不存在页面级刷新、通用快捷入口或无真实结果的伪按钮。

## 不在本原型内

- 不新增或修改正式菜单、路由、RBAC、API、schema、migration、Workflow usecase 或 Fact usecase。
- 不把静态数据、演示编号或原型脚本复制到运行时。
- 不替代任务看板、业务看板、业务模块标准页或客户配置评审。
