# 后台工作台与看板原型 / Admin Command Center Prototype

本目录保存后台首页、任务看板、业务看板、模板打印中心和异常 / 阻塞闭环的整套交互原型。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖工作台、任务看板、业务看板、模板打印中心和异常 / 阻塞闭环五个视图。 |

## 设计结论

- 后台首页 / 工作台作为登录后第一屏，聚合“今日必须处理、跨角色阻塞、业务状态摘要、打印队列和常用入口”，不替代具体业务模块。
- 任务看板是协同层总览，按角色、状态、到期时间和业务来源聚合 Workflow 任务；它只表达协同任务，不把任务完成写成库存、出货、财务或发票事实。
- 业务看板只展示业务记录和协同状态的运营摘要，不作为 `inventory_txns`、`shipments`、`finance_facts` 等事实层真源。
- 模板打印中心独立成轻量工作台，覆盖模板选择、预览、字段映射、示例记录和打印 / 导出动作；正式实现仍以现有打印模板文档和运行时代码为准。
- 异常 / 阻塞闭环定义阻塞原因、转派、催办、处理、验证和关闭的用户动作；真实状态流转仍应回到 WorkflowUsecase 和 RBAC 边界。

## 边界

- 本原型自身不接后端、不接权限、不进入 ERP 正式菜单、不进入生产构建。
- 本原型不替代 `docs/current-source-of-truth.md`、Workflow / Fact 架构文档、schema、migration、API 或测试。
- 真实实现时应优先复用后台布局、业务列表共享组件、任务详情面板和打印模板组件，不把 HTML 直接复制进 `web/src`。
- 如果后续继续扩展任务看板、业务看板或工作台菜单，需要同步评审菜单配置、RBAC 权限码、seed、帮助文档和浏览器回归。

## 验收关注点

- 五个视图可以在同一页面内切换。
- 任务详情、阻塞原因面板、模板切换和异常步骤切换有基础交互。
- 桌面宽屏和移动窄屏不出现横向溢出。
- 文案避免把 Workflow 任务状态误写成事实层过账结果。
