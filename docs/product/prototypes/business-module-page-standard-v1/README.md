## 当前设计结论

- 采用“业务对象优先 + 独立筛选条 + 结果工具条 + 当前记录操作条 + 主表分页 + 底部轻量协同入口”的标准页骨架。
- 标题、筛选、工具条和当前操作区默认使用紧凑首屏密度，避免在常见低高度屏幕上把表格完全挤出首屏。
- 业务主表交互默认保持“单击行选中、双击行进入编辑 / 主操作弹窗”；业务对象查看、编辑和新建统一进入业务表单弹窗，不再为同一对象额外保留详情抽屉。
- 状态类动作默认收口为“主状态动作 + 更多操作”；`更多操作` 下拉内用 `状态变更` 分组承接关闭、取消、重新提交等低频或高风险动作，不把所有状态变更横向铺成按钮。
- 运行时 formal-shell 页已降级为待接入预览页；字段预览弹窗只表达字段、明细 items 和接入边界，真实保存必须等领域 usecase、API 和 RBAC 接入后再启用，不把预览入口写成真实新建、编辑或状态变更。
- 标准页原型不再额外放置左侧快捷侧栏，避免把菜单示意误读为新的正式菜单；菜单、路由、RBAC 和模块归属仍以运行时代码与正式配置为准。
- 协同入口不再作为独立页面，也不复制完整任务看板；它只显示当前页面相关任务。
- 协同入口收起时只保留待办 / 阻塞 / 到期提示，展开后展示本页任务列表和当前任务详情；顶部灰蓝色粗横杠是高度拖动握柄，拖动高度只在当前页面会话内保留，刷新后恢复默认高度。
- 业务页面默认不展示调试任务。调试任务只属于开发、测试或内部验收能力，不进入生产业务用户主入口。
- 本原型不写库存、出货、财务事实，只表达 Workflow 协同入口和业务页布局。

同类列表页默认参照本样板，不逐菜单重做。客户档案、供应商档案、产品档案、工序档案、销售订单、采购订单、加工合同/委外下单、入库通知/检验/入库、库存、待出货/出货放行、出库、生产排单、生产进度、延期/返工/异常、品质检验、对账/结算、待付款/应付提醒、应收/开票登记和发票登记可先按本骨架评估；若信息结构明显不同，再单独做变体评审。

工序档案属于 MasterData / Engineering Data 标准页变体：页面可以给 `查货 / 手工 / 车缝 / 包装` 等毛绒玩具行业默认候选降低录入成本，但主路径仍是可扩展工序主数据，不把候选项升级成不可改枚举，也不代表生产排程、委外源单、质检事实或库存事实已完成。加工合同标准页变体只承接源单、工序明细、状态和打印，不在当前操作区放质检、库存或应付处理入口；查货结果和质检判定回质检 / 异常链路。

## 实现边界

落地时应改共享业务列表页布局和协同面板组件，不应把本目录 HTML 当成生产页面复制到 `web/src`。真实实现必须重新连接项目现有数据、权限、主题 token、浅色 / 暗色回归和前端测试。
