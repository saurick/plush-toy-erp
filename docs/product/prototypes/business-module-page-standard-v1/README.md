# 业务模块标准页原型 / Business Module Page Standard Prototype

本目录归档业务模块标准页的原型文件和早期方向图。

## 文件说明

| 文件 | 说明 |
| --- | --- |
| `index.html` | 当前推荐整页原型。包含顶部业务统计、筛选区、表格工具、选中记录操作条、主表分页和底部常驻协同入口。 |
| `task-collab-entry-v2.html` | 协同入口独立候选原型。用于对比任务分组、展开 / 收起、本页待办展示方式和桌面端高度拖拽手柄。 |
| `images/direction-1-current-record-sidebar.png` | 方向 1：右侧当前记录协同侧栏。 |
| `images/direction-2-flowbar-task-drawer.png` | 方向 2：顶部流程条 + 协同任务抽屉。 |
| `images/direction-3-bottom-task-table.png` | 方向 3：表格下方协同待办表。 |

## 当前设计结论

- 采用“业务记录优先 + 当前记录操作条 + 主表分页 + 底部常驻协同入口”的标准页骨架。
- 协同入口不放在长表格底部的普通内容流里，避免一页数据过多时必须滚到底才能看到。
- 协同入口展开后桌面端顶部显示横向拖拽手柄，可临时上下拖动调整面板高度；高度不缓存，刷新后回到合理默认值，移动端不启用拖拽以避免和页面滚动冲突。
- 业务页面默认不展示调试任务。调试任务只属于开发、测试或内部验收能力，不进入生产业务用户主入口。
- 本原型不写库存、出货、财务事实，只表达 Workflow 协同入口和业务页布局。

## 实现边界

落地时应改共享业务列表页布局和协同面板组件，不应把本目录 HTML 当成生产页面复制到 `web/src`。真实实现必须重新连接项目现有数据、权限、主题 token、浅色 / 暗色回归和前端测试。
