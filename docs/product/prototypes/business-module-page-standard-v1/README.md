## 当前设计结论

- 采用“业务对象优先 + 当前记录操作条 + 主表分页 + 底部轻量协同入口”的标准页骨架。
- 左侧侧栏只作为常用入口 / 快捷入口示意，不是新的正式菜单，也不改变客户菜单、路由、RBAC 或模块归属。
- 协同入口不再作为独立页面，也不复制完整任务看板；它只显示当前页面相关任务。
- 协同入口收起时只保留待办 / 阻塞 / 到期提示，展开后展示本页任务列表和当前任务详情；不再使用桌面端高度拖拽手柄，避免为一个辅助入口增加操作复杂度。
- 业务页面默认不展示调试任务。调试任务只属于开发、测试或内部验收能力，不进入生产业务用户主入口。
- 本原型不写库存、出货、财务事实，只表达 Workflow 协同入口和业务页布局。

同类列表页默认参照本样板，不逐菜单重做。客户档案、供应商档案、产品、销售订单、辅材/包材采购、加工合同/委外下单、入库通知/检验/入库、库存、待出货/出货放行、出库、生产排单、生产进度、延期/返工/异常、品质检验、对账/结算、待付款/应付提醒、应收/开票登记和发票登记可先按本骨架评估；若信息结构明显不同，再单独做变体评审。

## 实现边界

落地时应改共享业务列表页布局和协同面板组件，不应把本目录 HTML 当成生产页面复制到 `web/src`。真实实现必须重新连接项目现有数据、权限、主题 token、浅色 / 暗色回归和前端测试。
