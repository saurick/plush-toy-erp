- 已吸收变体：`BOM 管理`、`入库通知/检验/入库`、`库存`、`出库` 和生产事实页额外补只读或事实边界区，强调 BOM 明细、质检 / 入库、库存事实、出库事实、生产协同和生产事实边界；这些变体不新增额外事实写入。
- 待人工确认范围：当前运行时差异是否足以接受并晋级 Current；库存、出库、质检 / 入库等事实层专表和动作仍需后续独立 usecase / Fact 评审。

明确不直接吸收：原型里的固定数字、静态任务、假客户、mock 脚本、未评审业务动作和 dev-only 外壳不进入正式运行时；三张 PNG 方向图继续作为起草阶段 + 方案对比资料。

## 当前设计结论

- 采用“业务对象优先 + 业务工作台 + 当前记录操作条 + 主表分页 + 底部轻量协同入口”的标准页骨架。业务工作台收纳筛选、导出、列顺序、新建入口和当前操作，不拆成多个视觉孤岛。
- 标题摘要在桌面右侧横向排列，统计卡参考 trade-erp 的紧凑 tile 尺寸，默认约 `104-130px` 宽、约 `64-70px` 高。窗口宽度逐步缩小时，先让左侧标题说明自然换行；只有页头内容区继续收窄到无法同时容纳说明区和摘要组时，才把摘要整体落到描述 / 标签下方，不把摘要卡提前竖排堆在页头右侧占用 y 轴空间。
- 筛选、工具按钮和当前操作区默认使用紧凑首屏密度，避免在常见低高度屏幕上把表格完全挤出首屏。
- 标题摘要只放少量只读判断信息，默认 2-4 项：对象总量、当前筛选结果、一个最能影响下一步处理的关键状态或只读模式。不要把状态分布全部铺成头部卡片；多状态统计应回到状态筛选、表格状态列、看板或详情。
- 当前选中、来源已选、清空已选、已关联、可执行动作等操作反馈不放进标题摘要；这些信息必须靠近当前记录操作条、来源选择器或业务弹窗，避免头部重复显示 0/1 条造成心智负担。
- 标题摘要卡本身保持只读，不作为按钮或筛选入口；只读语义通过可访问名称、默认光标和视觉层级表达，不在每张卡上重复显示“摘要”徽标。
- 页面工具条只保留真实共享动作，例如导出筛选结果、列顺序和按对象专项评审后的状态说明；不内置通用批量删除、回收站或还原删除。空选中当前操作条统一显示“请选择一条记录”，选中后再显示业务单号、名称或来源摘要。
- 业务主表交互默认保持“单击行选中、双击行进入编辑 / 主操作弹窗”；业务对象查看、编辑和新建统一进入业务表单弹窗，不再为同一对象额外保留详情抽屉。
- 状态类动作默认收口为“主状态动作 + 更多操作”；`更多操作` 下拉内用 `状态变更` 分组承接关闭、取消、重新提交等低频或高风险动作，不把所有状态变更横向铺成按钮。
- 运行时已不再暴露 formal-shell 页；`生产排程`、`生产异常` 和 `出货放行` 以 Workflow V1 协同页承接任务读取、创建、完成、阻塞和催办。真实生产、出货、库存或财务事实写入仍必须等领域 usecase、API 和 RBAC 接入后再启用，不把协同任务处理写成事实过账。
- 标准页原型不再额外放置左侧快捷侧栏，避免把菜单示意误读为新的正式菜单；菜单、路由、RBAC 和模块归属仍以运行时代码与正式配置为准。
- 协同入口不再作为独立页面，也不复制完整任务看板；它只显示当前页面相关任务。
- 协同入口收起时只保留待办 / 阻塞 / 到期提示，展开后展示本页任务列表和当前任务详情；顶部灰蓝色粗横杠是高度拖动握柄，拖动高度只在当前页面会话内保留，刷新后恢复默认高度。
- 业务页面默认不展示调试任务。调试任务只属于开发、测试或内部验收能力，不进入生产业务用户主入口。
- 本原型不写库存、出货、财务事实，只表达 Workflow 协同入口和业务页布局。

同类列表页默认参照本样板，不逐菜单重做。客户档案、供应商档案、产品档案、工序档案、销售订单、采购订单、加工合同/委外下单、入库通知/检验/入库、库存、待出货/出货放行、出库、生产排单、生产进度、延期/返工/异常、品质检验、对账/结算、待付款/应付提醒、应收/开票登记和发票登记可先按本骨架评估；若信息结构明显不同，再单独做变体评审。

工序档案属于 MasterData / Engineering Data 标准页变体：页面可以给 `查货 / 手工 / 车缝 / 包装` 等毛绒玩具行业默认候选降低录入成本，但主路径仍是可扩展工序主数据，不把候选项升级成不可改枚举，也不代表生产排程、委外源单、质检事实或库存事实已完成。加工合同标准页变体只承接源单、工序明细、状态和打印，不在当前操作区放质检、库存或应付处理入口；查货结果和质检判定回质检 / 异常链路。

## 实现边界

落地时应改共享业务列表页布局和协同面板组件，不应把本目录 HTML 当成生产页面复制到 `web/src`。真实实现必须重新连接项目现有数据、权限、主题 token、浅色 / 暗色回归和前端测试。
