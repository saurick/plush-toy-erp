# 业务管理中心样板 / Business Management Center Pattern

本目录保存后台业务管理中心的待实现样板。它把业务看板收敛为三件事：读摘要、扫对象、分开看协同风险与数据失败。

## 阶段与文件

- 阶段：待实现 / To Implement
- 归属：产品内核 / Core
- 原型：`index.html`
- 方向图：`images/business-board-redesign-reference.png`
- 适用：桌面业务看板的只读总览

本资产未晋级为 Current。原型中的数量与异常均为状态演示数据，不是当前数据库、客户现场或发布环境证据。

方向图只作为信息层级参考；四类真源口径、20 个对象统计、失败与零值的交互验收以 `index.html` 为主，运行事实回到当前代码与测试。

## 设计结论

- 顶部只保留基础资料、业务单据、办理结果、需要关注四类只读摘要。四类语义不同，不能相加为“业务对象总量”或健康分。
- 基础资料、业务单据和办理结果只在所属模块全部可用时聚合；任何模块失败时该类显示 `— / 暂不可用`，不能用 0 掩盖失败。
- 对象扫描区使用 8 个对象族、20 个模块的可扫描表格。每个模块独立显示口径、数量、可用状态和正式入口归属。
- “需关注”只表达当前账号可见的阻塞与到期协同任务；任务完成不代表业务记录已经办理。
- “数据状态”单独列出失败来源。业务统计与待办概览独立成功、独立失败，一侧失败不清空另一侧成功结果。
- 页面不提供保存、过账、冲正、导出或模拟业务动作，也不恢复第二套通用业务记录。

## 信息结构

| 区域 | 真源语义 | 失败行为 | 是否可操作 |
| --- | --- | --- | --- |
| 基础资料摘要 | 客户、供应商、产品、BOM 模块投影 | 任一来源失败则整类暂不可用 | 否，真实 `div` |
| 业务单据摘要 | 销售、采购、生产、委外业务单据投影 | 任一来源失败则整类暂不可用 | 否，真实 `div` |
| 办理结果摘要 | 入库、质检、库存、出货、生产与财务记录投影 | 任一来源失败则整类暂不可用 | 否，真实 `div` |
| 需要关注摘要 | 当前账号阻塞与到期任务 | 待办来源失败时独立暂不可用 | 否，真实 `div` |
| 对象扫描表 | 8 个对象族 / 20 个模块逐项状态 | 失败模块显示 `—`，保留其余行 | 只读扫描 |
| 协同风险区 | 阻塞与到期任务摘要 | 不清空模块统计 | 只读提示 |
| 数据状态区 | 失败来源与影响范围 | 不覆盖已成功区域 | 只读诊断 |

## 原型状态演示

顶部“原型数据状态”下拉只用于设计复核，不是拟进入产品的筛选器：

| 演示态 | 需要证明 |
| --- | --- |
| 全部可用 | 摘要、模块、风险和成功空态都可读 |
| 部分来源失败 | 失败模块为 `—`，其余模块与风险继续保留 |
| 成功返回零值 | 0 明确表示来源成功且当前无记录 |
| 全部暂不可用 | 模块清单和入口归属仍保留，失败范围独立说明 |

## 当前运行时参考

运行时 `/erp/business-dashboard` 本轮已吸收四类只读摘要、20 个独立对象行、右侧需要关注区、失败与真实零值分离以及显式业务入口；原型还保留更完整的状态演示和键盘 review，因此仍不自动晋级为 Current。

真实实现时应：

1. 继续从正式业务统计与待办真源读取数据，不在前端合成未定义的事实。
2. 保留模块级 `available / total` 语义；失败与真实零值必须可区分。
3. 使用全局刷新入口，不新增页面级刷新。
4. 模块入口必须服从当前菜单、路由和权限；本原型的“入口归属”文字不反推新菜单。
5. 字段编辑、业务办理、过账、冲正和导出回到对应业务页面。

## 响应式与可访问性

- 摘要卡是只读 `div`，无 pointer、hover 抬升、button 角色或键盘焦点。
- 状态演示使用带 label 的原生 `select`，切换结果通过 `aria-live` 提示。
- 对象扫描表在窄屏内局部横向滚动，不造成页面级横向溢出。
- 1120px 以下风险区与失败区移到表格下方，700px 以下改为单列。
- 浅色、暗色、长模块名、大数字、零值和不可用状态都保持可读。

## 不在本原型内

- 不新增或修改 API、schema、migration、RBAC、正式菜单、路由、Workflow / Fact usecase。
- 不把演示数字、失败样例或原型脚本复制到运行时。
- 不以看板摘要替代业务模块列表、详情、表单、打印或动作浮层。
