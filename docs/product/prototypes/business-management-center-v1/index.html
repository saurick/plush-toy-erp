Business / 对象总控

业务管理菜单只做一件事：帮用户选对业务对象和处理路径。

这里不把所有模块铺成大菜单，也不恢复旧 business_records。业务管理只按链路聚合对象、风险和入口，真实字段、动作和过账仍回到标准业务页。

1
先选链路客户订单、采购入库、库存出货或财务。
2
再选对象进入销售订单、入库、库存、出货等标准页。
3
最后做动作保存、过账、冲正和导出都回到目标页。

销售订单

承接客户订单、交期和明细承诺。 下一步：进入销售订单标准页，查看列表、详情和明细。
页面类型标准列表 + 详情
事实边界不写库存 / 出货 / 财务
能力状态已接 V1
建议入口销售订单

销售订单表达客户承诺，不自动生成出货、库存预留、应收或发票事实。

只读摘要

业务管理只提示风险，不在这里写事实

风险卡只作为跳转和筛选线索，不能替代业务页状态机。
交期风险订单未确认交期时，提示进入销售订单或任务中心。
入库风险质检或入库阻塞时，提示进入入库 / 质检对象。
事实风险库存、出货、财务动作必须由对应 usecase 完成。
边界

业务管理不是第二套业务系统

不恢复 business_records

不查询、不写入旧通用业务记录，也不把它作为对象总控真源。

不提前承诺未接能力

入口壳可以展示方向，但必须标明待接 API / usecase。

不替代正式菜单

是否新增或合并左侧菜单，仍要单独评审 seed、客户菜单和 RBAC。
