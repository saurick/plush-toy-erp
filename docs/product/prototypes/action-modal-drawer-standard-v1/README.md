# 局部动作弹窗样板 / Action Modal Standard

本目录保存后台局部动作弹窗的 `待实现 / To Implement` 标准样板。它统一列表页内的来源导入、回收站、列顺序、删除确认和危险确认等局部动作；业务对象新建 / 编辑不再走抽屉。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖来源明细导入、中型表格弹窗、回收站、列顺序调整、删除确认、底部动作和信息边界。 |

## 标准样板口径

- 适用页面：出货、采购、质检、库存、财务等需要从列表页打开局部动作浮层的场景。
- 同类菜单默认参照本样板，不逐菜单重做原型；字段真源、动作可见性、权限、幂等、来源导入门禁、明细行校验和事实约束仍由正式 API / usecase / RBAC 决定。
- 局部动作弹窗只承接当前列表记录的辅助动作；完整新建 / 编辑必须参照 `business-form-page-standard-v1/index.html` 的同一业务表单弹窗，不另做抽屉路径。
- 来源导入只表达候选行选择、不可导入原因和当前动作所需列；可导入数量、库存批次、状态边界和重复提交仍由后端守卫。
- 回收站、列顺序和删除确认按业务列表页局部工具收口；不恢复旧业务页，也不把列偏好、回收规则或删除策略写成原型真源。
- 危险动作必须明确影响范围并二次确认；Workflow 任务关闭不等于库存、出货、财务、开票或收付款事实过账。

## 边界

- 本样板保持 `To Implement`，不晋级 `Current`。
- 不修改正式运行时代码、WorkflowUsecase、Fact usecase、后端 API、RBAC、schema 或 migration。
- 不替代 `business-form-page-standard-v1` 的业务对象新建 / 编辑弹窗骨架，也不替代字段残值 / 缺值防护规则。
- 原型中的来源导入、明细行、回收站、列顺序、危险确认和按钮状态只用于表达交互规则，真实落地仍需测试锁住后端行为。

## 验收关注点

- 局部补录、来源导入、回收站、列顺序和删除确认可以切换，标题、宽度、底部动作和内容密度清晰区分。
- 来源导入能展示不可导入原因，不把不可导入行伪造成可选项。
- 删除确认必须二次确认，不可静默执行。
- 移动端打开浮层后背景不可滚动，关闭浮层后恢复页面滚动。
- 桌面和移动视口不横向溢出，浅色 / 暗色都可读。
