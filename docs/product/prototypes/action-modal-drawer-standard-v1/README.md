# 弹窗 / 抽屉动作样板 / Action Modal And Drawer Standard

本目录保存后台弹窗 / 抽屉动作的 `待吸收实现 / To Implement` 标准样板。它用于统一审批、驳回、阻塞、冲正、关闭任务和危险确认等动作浮层的语义边界。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖动作抽屉、原因必填、主次按钮、危险确认弹窗、关闭态和 Workflow / Fact 动作差异。 |

## 标准样板口径

- 适用页面：任务处理抽屉、审批 / 驳回弹窗、阻塞原因弹窗、库存 / 出货 / 财务冲正确认、关闭任务确认和危险操作确认。
- 同类菜单默认参照本样板，不逐菜单重做原型；动作可见性、权限、幂等、原因字段和事实约束仍由正式 usecase / RBAC 决定。
- Workflow 动作只处理协同任务状态，reason 必填、空字符串无效等规则在原型中表达，不改真实 WorkflowUsecase。
- Fact 动作必须有更强确认，尤其是冲正、取消过账、关闭事实单据等操作；危险动作必须明确说明影响范围。
- 弹窗 / 抽屉只承接当前动作，不把帮助、导入导出、开发验收入口混在主流程里。

## 边界

- 本样板保持 `To Implement`，不晋级 `Current`。
- 不修改正式运行时代码、WorkflowUsecase、Fact usecase、后端 API、RBAC、schema 或 migration。
- 不把任务关闭等同于库存、出货、财务、开票或收付款事实过账。
- 原型中的 reason 必填、危险确认和按钮状态只用于表达交互规则，真实落地仍需测试锁住后端行为。

## 验收关注点

- 抽屉可打开 / 关闭，动作切换后原因说明和按钮语义清晰。
- 空原因会展示错误，输入有效原因后错误消失。
- 危险确认弹窗必须二次确认，不可静默执行。
- 桌面和移动视口不横向溢出，浅色 / 暗色都可读。
