# 新建 / 编辑业务弹窗样板 / Business Form Modal Standard

本目录保存后台业务对象新建 / 编辑弹窗的 `待实现 / To Implement` 标准样板。它用于表达同一个弹窗内“上方主表字段、下方明细 items”的表单结构、校验、保存动作和字段残值 / 缺值防护原则。

## 文件

| 文件 | 用途 |
| --- | --- |
| `index.html` | 单文件可交互 HTML 原型，覆盖首个落地对象提示、上方主表字段、下方明细 items、item 区来源选择器入口、横向滚动、已录入 / 数量 / 金额统计、必填提示、校验错误、保存 / 取消 / 重置、来源带值、切换来源清值、新增 / 编辑 / 只读状态和缺值 / 残值防护提示。 |

## 标准样板口径

- 适用页面：销售订单新建 / 编辑、采购订单、客户 / 供应商资料、产品 / BOM、质检记录、出货草稿、财务登记等业务对象表单弹窗。
- 同类菜单默认参照本样板，不逐菜单重做原型；具体字段、必填规则、来源对象和保存权限仍由正式 API / usecase / RBAC 决定。
- 新建和编辑必须复用同一个业务弹窗；标题、默认值、只读状态和保存动作按模式变化，不另做一套抽屉或详情表格。
- 桌面 ERP 的业务对象创建、编辑和查看统一使用 Modal。确认 / 删除 / 简单提示使用小弹窗，基础资料使用标准弹窗；采购订单、销售订单、出货单、质检单、BOM、委外等业务单据使用宽弹窗，参考运行时共享尺寸 `min(1720px, calc(100vw - 96px))`。
- 表单应优先按信息来源和业务语义分组，不把所有字段堆成一张长表。
- 弹窗上半部分维护主对象字段；下半部分维护必要明细 items，例如联系人 items、订单行 items、BOM 明细 items、出货明细 items 或其他模块必需子项。item 区必须有独立标题、边界、操作区和统计，视觉上能一眼区别于上方主表字段。
- 订单行、采购明细、BOM 明细、出货明细等 item 区可以在本区内提供“从产品 / SKU / 材料 / 来源单据库导入”的入口；入口打开来源选择器作为第二层弹窗，选择器只负责筛选和选择来源行，导入后回到当前业务弹窗维护数量、价格、仓库、批次等本单字段。
- item 卡片默认不换行堆叠字段；字段较多时保持横向滚动，避免把弹窗撑成多行长表。单行输入统一高度，多行输入按单行高度倍数处理。
- 来源带值必须同时表达“已带出什么”和“来源切换后会清空什么”；缺值时不伪造值，残值风险必须在保存前可见。
- 来源选择器、整张单据级来源导入、回收站、列顺序和危险确认统一参照 `action-modal-drawer-standard-v1/index.html` 的局部动作弹窗；它们不替代本样板的新建 / 编辑业务弹窗主路径。
- 新增、编辑、只读和无权限状态必须显式区分；只读或无权限状态下可以回显字段，但不能继续暴露保存动作。
- 保存 / 取消 / 重置是业务弹窗动作，批量导入、打印、帮助和开发验收入口只作为弱辅助规则，不喧宾夺主。
- 底部动作区只作为表单末尾确认入口，不应覆盖字段分组、错误提示、来源说明或残值 / 缺值防护提示。

## 边界

- 本样板保持 `To Implement`，不晋级 `Current`。
- 不新增真实字段语义，不修改 schema、migration、后端 API、RBAC 或正式菜单。
- 原型里的模拟字段只用于说明布局和校验状态，不作为产品字段真源。
- 字段残值 / 缺值防护只是原型表达；真实落地时必须回到统一映射函数、转换层或共享 helper。
- 不在本样板重复维护回收站、列顺序或危险确认；这些是列表页局部动作弹窗。item 区的“从业务库导入”只保留入口和回填结果，来源选择器本体回到局部动作弹窗样板。
- 原型里的只读、编辑和保存状态只表达页面策略；真实保存入口必须由后端权限、对象状态和 usecase 共同决定。

## 验收关注点

- 必填、错误、来源带值、清值和保存前防护都能被看见。
- 首个落地对象、字段真源、不吸收内容和状态覆盖在首屏可见。
- 新增、编辑、只读状态可以切换；只读状态下保存按钮不可用。
- 表单弹窗在 1280px 和 390px 视口都可操作。
- item 区在弹窗中有明显辨识度，来源选择器入口、添加条目和统计入口位置稳定，横向滚动不会造成整页溢出。
- 底部动作区在默认态、校验态和移动端都不遮挡字段或错误文案。
- 重置动作能恢复示例表单，保存动作能触发错误展示。
- 浅色 / 暗色均可读，错误态和禁用态不与普通态混淆。
