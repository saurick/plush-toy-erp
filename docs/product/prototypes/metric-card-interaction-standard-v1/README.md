# 指标卡交互语义样板 / Metric Card Interaction Standard

## 阶段与归属

- 阶段：待实现 / To Implement
- 归属：产品内核 / Core
- 目标：统一后台统计卡、动作卡和筛选卡的视觉语义，避免新用户分不清“只是看数据”和“可以点击处理 / 筛选”。

## 吸收范围

- 只读统计卡：用于业务摘要、总数、比例、只读投影。默认不使用 pointer 光标，不做 hover 抬升，不声明 button 角色，并露出轻量“摘要”角标。
- 可点击动作卡：用于进入处理、查看详情或打开关联对象。默认必须是真实 button 或链接，露出动作文案、进入箭头、hover / focus / pressed 状态。
- 筛选统计卡：用于改变当前列表筛选。默认必须有选中态和恢复态，不能只把数字卡加上 `onClick`。

## 不吸收范围

- 不新增菜单、路由、RBAC、后端 API、schema、migration 或 Fact 写入。
- 不把示例数字、示例任务、示例模块写成真实数据。
- 不要求所有统计都变成按钮；能点才做成动作卡，不能点就保持只读。
- 不替代业务页、任务看板、业务看板或岗位任务端的字段真源。

## 进入真实实现前的规则

1. 先判断卡片语义：只读、动作还是筛选。
2. 只读卡不得有 `onClick`、`role="button"`、`tabindex="0"` 或 pointer 光标。
3. 动作卡和筛选卡必须是可聚焦控件，并有默认态可见的动作信号。
4. 筛选卡必须把当前选中态显示出来，恢复态必须可见。
5. 浅色、暗色、桌面和移动断点都要检查文字不溢出、卡片不重叠、动作提示不遮挡数字。

## 当前运行时吸收口径

- `/erp/business-dashboard` 顶部业务摘要：按只读统计卡处理。
- 业务看板的“主数据 / 工程、业务源单、事实记录、协同风险”是四类不同业务口径，只能分别阅读，不能相加为对象总量或健康分数。
- 指标来源不可用时显示 `—` 和“暂不可用”，不能显示 0；只有来源成功返回且真实数量为 0 时才显示 0。对象概览中的每个模块也遵循同一 `available / total` 语义。
- `/erp/task-board` 顶部任务指标：按筛选统计卡处理。
- `/erp/dashboard` 不再保留通用业务对象入口区块；工作台里的当前任务关联记录入口按普通按钮 / 链接处理，不作为指标卡候选。
