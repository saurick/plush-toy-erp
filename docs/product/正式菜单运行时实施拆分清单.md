## 1. 当前运行时真源

| 面 | 当前文件 | 作用 | 实施注意 |
| --- | --- | --- | --- |
| 前端默认导航 | `web/src/erp/config/seedData.mjs` | 定义工作台、任务看板、业务看板、打印中心、异常闭环、权限管理等默认入口，并组合业务模块导航 | 改 key / label / path / section 都会影响前端菜单、权限项、测试和截图 |
| 前端业务模块 | `web/src/erp/config/businessModules.mjs` | 定义业务页分组、模块标题、路由、摘要和页面口径 | 改 title / route 会影响侧边栏、权限项、业务页标题和文档 |
| 客户菜单配置 | `web/src/erp/config/customerMenuConfig.mjs`、`config/customers/yoyoosun/menuConfig.mjs` | 控制客户侧显示、隐藏、排序和文案 | 只影响入口可见性，不是 RBAC 或事实层安全边界 |
| 前端菜单权限项 | `web/src/erp/config/menuPermissions.mjs` | 从当前导航生成菜单权限分组和角色预设 | 改菜单会影响 `ERP_MENU_PERMISSION_GROUPS`、默认角色预设和旧路径归一化 |
| 后端内置菜单 | `server/internal/biz/rbac.go` 的 `BuiltinAdminMenus()` | 登录返回 menus、RBAC 菜单可见性和 menu_options | 必须和前端正式入口保持 path / label / required permissions 一致 |
| 后端菜单测试 | `server/internal/biz/rbac_test.go`、`server/internal/service/jsonrpc_*` | 锁定管理员可见菜单、旧入口退出和 JSON-RPC 返回结构 | 前端菜单变更若影响后端内置菜单，必须同步这些测试 |

当前实现已经不是“只有前端菜单”。如果只改 `seedData.mjs` 或客户配置而不改后端 `BuiltinAdminMenus()`，登录返回菜单、权限管理选项和前端导航会漂移。

当前仍是正式入口壳的入口只剩：`生产排程`、`生产异常`、`出货放行`。这些入口只恢复页面结构和交互，不读取或写入旧 `business_records`，也不代表对应领域写入 API 已完成。`产品档案`、`BOM 管理`、`采购订单`、`入库管理`、`来料质检`、`库存台账`、`出货单`、`委外订单`、`生产进度`、`出库管理`、`对账管理`、`应付管理`、`应收管理` 和 `发票管理` 已有对应 V1 或收窄 V1 页面，仍需按各自边界区分主数据、工程资料、采购承诺、采购入库事实、质检事实、含 ACTIVE 预留可用量 read model 的库存只读台账、出货事实、委外加工合同源单、生产 operational facts、库存预留和财务 facts。

## 4. R0 当前实现边界

R0 当前只做后端内置菜单与当前前端默认导航对齐：

1. 后端 `global-dashboard` label 为 `工作台`，path 为 `/erp/dashboard`。
2. 后端 `task-board` path 为 `/erp/task-board`，label 为 `任务看板`，required permission 为 `workflow.task.read`。
3. 后端 `business-dashboard` label 为 `业务看板`，path 为 `/erp/business-dashboard`。
4. 后端 `exception-flow` path 为 `/erp/operations/exceptions`，label 为 `异常 / 阻塞闭环`，required permission 为 `workflow.task.read`。
5. 后端 `receivables` label 为 `应收管理`。
6. 后端 `invoices` label 为 `发票管理`。

R0 不做：

- 不改 `web/src/erp/config/seedData.mjs`、`businessModules.mjs` 或 yoyoosun 客户菜单。
- 不新增路由、页面、权限码、API、schema、migration。
- 不把任务看板或异常闭环写成 Fact 入口。
- 不改变客户菜单显隐；yoyoosun 仍可隐藏任务看板和异常闭环。

## 6. 停止条件

遇到以下情况必须停止并重新评审，不继续顺手扩范围：

- 需要新增或修改权限码。
- 需要改路由、页面路径或旧路径兼容策略。
- 需要新增 API、schema、migration 或 seed。
- 需要改变 yoyoosun 菜单显隐。
- 需要把报表、数据导入、审计日志、客户配置 UI、备份恢复 UI 等候选能力变成正式入口。
- 发现 active workflow task 只能通过被隐藏或将删除的入口处理。
