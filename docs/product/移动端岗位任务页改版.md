## 已采用方向

- 默认态采用“移动优先任务列表”，优先让岗位人员快速看到待处理任务、预警、超时和阻塞原因。
- 点选任务进入详情态，集中展示关键信息、关联单据、最近动态和操作原因输入。
- 底部保留移动端导航感，并固定在岗位任务页视口底部；正文内容独立滚动，避免导航跟随任务列表滚走。
- `待办 / 已办 / 消息 / 我的` 当前只是同一页面内的前端分区：待办展示待处理任务、已办展示终态任务和进度摘要、消息展示预警与通知、我的承载当前账号、快捷入口和退出登录。
- 待办页只保留一组主筛选：`全部 / 风险 / 超时 / 我负责`。顶部四项指标只作为只读摘要，不再承担 tab 或筛选职责；风险筛选统一承接预警、即将超时、阻塞和高优先任务。
- 待办主筛选是视图筛选，不是互斥业务分类；`风险` 是异常聚合视图，`超时` 是其中需要单独快速定位的强风险子集，因此同一任务可以同时出现在 `风险` 和 `超时` 视图中。
- 已办页的进度四项只作为完成度摘要，不再作为跳转待办筛选的二级 tab；“我的”页保留 `待办 / 已办 / 超时 / 风险` 快捷入口，其中可回到待办的入口必须落到可见主筛选，避免跳转后出现隐藏筛选态。
- 当前暂不提供移动端任务搜索；长列表先通过主筛选、风险优先排序和分批展开控制。待办初始显示 12 条、已办初始显示 10 条、预警 / 通知初始显示 8 条；每次展开只增加一个同等批次，最后一批展开后才显示收起入口。
- 列表页滚动超过阈值后显示右下角回到顶部按钮，点击只滚动当前正文容器，不改变筛选、展开数量或任务状态；任务详情页不显示该按钮，避免遮挡底部处理动作。
- 当前不把这些底部导航文案升级为新的权限或路由真源。
- 暗色模式继续遵循 ERP theme token，不单独引入只服务原型图的色系。

## 实现边界

- 当前实现入口：`web/src/erp/mobile/pages/MobileRoleTasksPage.jsx`。
- 移动端外壳入口：`web/src/erp/mobile/MobileAppLayout.jsx`。
- 移动端暗色覆盖：`web/src/erp/styles/app.css` 中 `.mobile-app-layout` 范围内规则。
- 状态更新、阻塞、退回、完成、催办仍复用现有 `moveTask` / `urgeTask` 主路径。
- 本改版不改变后端、schema、migration、RBAC、seedData、Workflow / Fact 语义或事实层。
