# 菜单映射评审表 / Menu Mapping Review v1

本文原用于菜单评审与原型资料收口。2026-06-13 已按本文的运行时门槛把当前可承载的产品核心入口吸收到运行时菜单、路由、RBAC 菜单和测试；schema、migration、旧 `business_records` 运行时和真实领域写入 API 仍未恢复或新增。

## 评审结论

- 当前产品默认桌面菜单来自 `web/src/erp/config/seedData.mjs` 和 `web/src/erp/config/businessModules.mjs`，共 25 个左侧入口。
- 永绅前端菜单源来自 `config/customers/yoyoosun/menuConfig.mjs`，部署时通过 `customer-config.js` 注入，默认产品前端包不静态打包具体客户配置；它按客户配置覆盖品牌、favicon、分组、排序和文案，但不替代 RBAC、安全边界、schema 或 Fact 规则。
- `formal-menu-candidate-v1` 的 12 个主入口只是正式菜单候选评审输入，不能直接升成正式菜单。
- `core-menu-coverage-v1` 的 51 个细项是内部覆盖矩阵，不代表要把 51 项逐个铺进左侧导航。
- 运行时实现已按 `docs/product/正式菜单运行时实施拆分清单.md` 同步前端菜单、客户配置、后端内置菜单、RBAC 菜单和测试影响；真实领域写入 API 仍需后续逐模块接入。
- 本轮不新增 `tenant_id`、多租户、license、客户工单或 SaaS 管理能力，不把 yoyoosun 客户资料写成 Product Core 规则。

## 运行时影响拆分口径

本轮所有行的实际运行时影响均为 `否`。下表只说明若后续采纳对应处理策略，需要评审哪些运行时面。

| 影响项 | 本轮是否修改 | 后续评审口径 |
| --- | --- | --- |
| 文案 | 否 | 改名会影响左侧显示、权限标签、测试快照、用户培训文档和截图证据。 |
| 分组 / 排序 | 否 | 改分组会影响 `seedData` 默认菜单、客户菜单配置、菜单权限分组顺序和导航测试。 |
| 显示 / 隐藏 | 否 | 客户显示配置可隐藏菜单，但不能替代 RBAC；产品默认隐藏需要单独评审。 |
| 路由 | 否 | 路由改变必须同步页面、权限归一化、旧路径兼容策略和浏览器回归。 |
| 权限 | 否 | 菜单权限当前以路由为 key；新增或移除入口必须同步 `menuPermissions`、预设角色和 RBAC 测试。 |
| 后端内置菜单 / seed | 否 | 若后端存在对应内置菜单或 seed，需要同步后端真源和测试；本轮不触碰。 |
| 测试 | 否 | 运行时变更至少更新 `seedData.test.mjs`、`menuPermissions.test.mjs` 和受影响页面回归。 |
| 正式文档 | 是，仅新增评审文档 | 若后续变更正式菜单，需要同步 `正式产品入口与菜单配置计划.md`、`当前真源与交接顺序.md`、`web/README.md` 和产品帮助口径。 |
