# 产品原则 / Product Principles

## 核心原则

| 原则 | 说明 |
| --- | --- |
| 一套标准产品内核 | 核心业务代码尽量不分叉，库存、出货、财务和权限安全边界统一维护 |
| 毛绒玩具行业模板 | 优先服务毛绒玩具和相近轻工制造行业，而不是所有行业通用 ERP |
| 多客户配置包 | 客户差异优先通过配置、初始化数据、菜单和权限表达 |
| 少量客户专属样本 / 数据适配 | 打印样本、导入适配和报表口径可隔离在客户配置或交付包中；打印模板产品内核暂不做 |
| 极少数客户扩展 | 真正客户专属逻辑必须放到 extension 边界，并记录原因和退出条件 |
| 核心事实规则不分叉 | 库存流水、出入库、质检、出货、财务和审计规则不能按客户随意拆分 |

## 不做什么

- 不做所有行业通用 ERP。
- 不做低代码平台。
- 不做复杂 BPMN 或可视化流程编排器。
- 不做泛化 Change 模块。
- 不创建 `change_records` 作为通用需求或业务事实表。
- 不提前实现 SaaS、license server、套餐计费或客户工单系统。

## 设计取舍

当前阶段优先稳定、可维护和易理解：

1. 先把 Product Core、Industry Template、Customer Config 和 Customer Extension 边界写清。
2. 先用文档和目录表达未来落点，不提前做 loader、DSL 或 schema。
3. 客户差异先进入 `docs/customers/yoyoosun` 和 `config/customers/yoyoosun` 草案，不直接进入核心 usecase。
4. 只有库存、出货、财务等事实规则经过架构评审后，才进入 Product Core。
5. 打印格式先按客户样本和交付诉求记录，只有多客户重复性成立后才评审是否产品化。
