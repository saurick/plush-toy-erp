# 产品能力进度台账 / Product Capability Ledger

本文是 `plush-toy-erp` 唯一的全局产品能力状态真源。它只记录产品能力做到哪一层、当前边界和下一步；yoyoosun 是否可见、可试用或已交付，统一看[客户交付矩阵](../customers/yoyoosun/客户交付矩阵.md)。

## 怎么查

1. 先在“能力状态”中定位业务能力和成熟度。
2. 需要确认 Workflow、Fact、字段来源或实现边界时，进入对应专题文档、代码、migration 和测试；本台账不复制详细证据清单。
3. 需要判断客户状态时，进入客户交付矩阵；产品成熟度不能直接推出客户已发布或已验收。

代码、schema、migration、测试和运行态证据与本文冲突时，以前者为准并修正本文。完整阅读顺序见[当前真源与交接顺序](../当前真源与交接顺序.md)。

## 成熟度口径

| 等级 | 名称 | 含义 | 是否可对客户承诺 |
| ---: | --- | --- | --- |
| L0 | Not Started | 未开始，只在路线或待办中 | 否 |
| L1 | Discussed | 已讨论，有初步口径 | 否 |
| L2 | Reviewed | 已完成架构或业务评审 | 否 |
| L3 | Drafted | 有 schema、配置或数据映射草案 | 否 |
| L4 | Schema Ready | schema、migration 和生成物完成 | 否 |
| L5 | Runtime Ready | repo/usecase 完成，后端核心逻辑可测 | 否 |
| L6 | API Ready | API 与 RBAC 完成，可内部联调 | 否 |
| L7 | UI Ready | 正式页面已接，可作为试用候选 | 否，仍需客户矩阵与环境证据 |
| L8 | Delivery Ready | 页面、真源、权限、审计、测试、部署、培训和客户验收均闭环 | 是 |

L0-L8 表示产品技术成熟度，不表示某个客户环境的发布状态。当前没有任何能力仅凭本表即可写成“客户已交付”。

## 流程运行时状态

这张表只保留流程能力的差异状态，避免 preview 配置被误读为已执行 runtime。详细边界见[状态、工作流与事实边界](../architecture/状态工作流事实边界.md)和[当前真源与交接顺序](../当前真源与交接顺序.md)。

| 流类型 | 当前状态 | 领域写入边界 |
| --- | --- | --- |
| Workflow task | runtime 已接 | 只管协同；不直接写库存、质检、出货或财务 Fact。 |
| ProcessRuntime / sales_order_acceptance | runtime 部分接入 | 可调用销售订单领域命令；来源批准与状态推进仍需逐域收口。 |
| ProcessRuntime / material_supply | runtime 已接，目标证据待补 | 可创建收货/质检并经正式门禁过账库存；流程门禁不替代入库 usecase 校验。 |
| ProcessRuntime / finished_goods_delivery | runtime 部分接入 | 可进入质检、出货、库存和财务草稿命令；目标 execute/readback 和客户验收未闭环。 |
| 首次 IQC 退厂 / 补换 | 领域 usecase 已接，目标证据待补 | 按精确行和累计数量登记；补换生成新待收 / 待检链，不写入库事实。 |
| 委外返厂 / 返工 | 领域 usecase 已接，目标证据待补 | 返厂写库存 OUT / REVERSAL；返工写可追溯 WIP 子批并受下游取消门禁。 |
| 生产报废 / 超领 / WIP 让步 | 决定与执行分离，目标证据待补 | 批准只记录决定或额度；明确执行才写 SCRAP / WIP，超领由正式领料消费额度。 |
| 财务红冲 | 领域 usecase 已接，目标证据待补 | 仅正式应收 / 应付可生成红冲；反向红冲保留原记录并支持结果读回。 |
| `businessFlows` / `stateMachines` / `processPolicies` | preview-only | 只作配置预览和 lint，不覆盖 usecase 状态机、权限或 Fact 写入规则。 |
| 打印默认值 | effective session 投影 | 只提供买方/委托方默认展示值，不覆盖供应商、加工方或源单业务快照。 |

## 高风险判定口径

- 库存闭环必须同时能解释 `IN / OUT / REVERSAL` 流水、余额与可用量、ACTIVE 预留、批次/质检状态、来源单据和取消/冲正；页面存在或任务完成不能单独证明闭环。
- 显式 SKU 在批次、流水、余额、生产、委外、出货和预留间保持同一 grain；`product_sku_id = NULL` 是独立的未分规格库存池，不能当作 SKU 不足时的兜底。
- 客户配置 runtime 只消费已发布、激活并经 catalog、依赖与 RBAC 交集校验的 revision；preview 内容不得执行、提升权限或改写业务事实。
- 异常审批只形成决定或额度；报废、让步、返厂、返工、补换和红冲仍由各自领域动作形成 Fact / WIP / 来源链，并可按其下游依赖独立拒绝或冲正。
- 本地代码、测试或较早目标版本绿色不等于当前 HEAD 已发布，也不等于客户 UAT 或签收完成。

## 维护规则

- 只有产品能力成熟度、当前边界或下一步发生实质变化时才更新对应一行；不要把测试输出、每轮补丁历史或完整证据清单复制进来。
- 客户是否启用、是否可试用、目标环境是否发布、客户差异或确认事项，只更新对应客户交付矩阵。
- 代码、migration、测试或运行态事实与本表冲突时，先按正式真源核对，再修正文档；不得反向修改实现来迎合台账。
- 新增客户时复用一份客户交付矩阵，不复制本全局台账。
