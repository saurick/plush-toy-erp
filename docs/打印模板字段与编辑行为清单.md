# 打印模板字段与编辑行为清单 / ERP Print Template Field Behavior

## 目的

说明当前毛绒 ERP 打印模板里“左侧字段怎么改”“右侧纸面怎么编辑”“PDF / 打印到底吃哪一份内容”这些稳定行为，避免把普通路由页、旧草稿恢复和当前窗口级编辑链路混为一谈。

本文只收口当前可见编辑行为，不解释完整实现分层；实现真源见 `docs/打印模板实现原理.md`。

## 当前稳定行为

- 当前正式模板包括 `采购合同`、`加工合同`、`物料分析明细表`、`色卡`、`作业指导书`；当前运行时主入口是 `打印模板中心` 的默认样例、纸面预览和打印窗口入口，以及采购订单 / 委外订单 / BOM 管理业务页按领域模型生成的打印草稿。委外订单带值打开加工合同和外发作业指导书；BOM 管理带值打开物料分析明细表、色卡和内部 / 工程资料作业指导书。旧业务页选中 `business_records` 后带值打开独立弹窗的链路已退出；后续若要从其他领域页带值打印，必须先定义领域模型到打印草稿的字段映射。字段编辑、明细确认、图片槽维护和纸面微调都在独立打印窗口内完成。
- 正式模板都必须在 `printTemplates.mjs` 声明 `fieldRequirements`、`moduleKeys` 和 `factBoundary: read_snapshot_only`：`fieldRequirements` 只说明打印草稿需要哪些合同头、供应商 / 加工方、BOM / 产品头、明细、条款、步骤或附件 / 图片快照；`moduleKeys` 必须和服务端 PDF active module state 门禁一致；`read_snapshot_only` 表示打印不会创建、确认、过账或反写业务事实。
- 打印中心默认样例属于 Product Core 通用样例，可使用 `本公司`、`示例供应商`、`示例加工厂`、`采购负责人`、`公司联系电话` 这类中性展示值，方便编辑页和纸面预览完整呈现；客户公司名、真实联系人、真实电话、真实地址、真实签字人和原始资料路径不得作为核心模板默认值。客户专属信息应来自业务草稿、客户配置包、客户打印模板或客户交付资料边界；从业务页带值打开打印时，真实业务缺失字段保持空白，不用打印中心样例兜底。
- 客户配置包可声明 `printTemplateDefaults` 作为甲方 party defaults 草案：当前先经过客户包 lint、开发态客户配置页和 runtime manifest 编译，再由后端 active revision 的 `customer_config.get_effective_session` 受控投影给正式前端；采购订单页的 `material-purchase-contract` 打印入口和委外订单页的 `processing-contract` 打印入口已消费该投影中的买方 / 委托方默认字段。销售订单打印模板仍未启用，供应商 / 加工方名称、联系人、电话和地址仍必须来自业务记录快照，不允许被客户包默认值覆盖。
- 独立打印窗口提供 `手签留白` 动作。该动作只清空当前窗口草稿里的甲方签名和乙方签名，保留甲乙方日期、签字区版式、合同主体、明细、条款和附件，便于双方打印后手写签字。该动作不代表电子签署、审批通过或业务事实过账，也不会修改领域记录、模板配置、后端事实或其他窗口草稿。
- 独立打印窗口提供 `空白模板` 动作。该动作只清空当前窗口草稿里的字段值、明细行值和合并状态，保留模板结构与合同条款；加工合同会同时清空纸样 / 图样附件快照。该动作不会修改领域记录、模板配置、后端事实或其他窗口草稿。
- 弹窗打开主路径是直接进入 `/erp/print-workspace/:templateKey?...&state=:stateID` 形式的 React 工作台 URL；`state` 用来隔离当前窗口草稿。`/print-window-shell.html` 只保留为已有窗口状态和极端恢复场景的兼容兜底，不再作为可编辑模板的刷新真源。
- 刷新当前弹窗时，React 工作台按当前 `state` 读取结构化草稿并重新渲染右侧纸面；右侧 `contenteditable` 输入期通过 `input` 和当前 editable DOM 文本变化同步写入当前窗口草稿，失焦再提交 React 渲染态；若用户在焦点仍停留时直接刷新，页面退出前会先让当前编辑区失焦再 flush 当前 draft。`恢复样例 / 空白模板` 等外部重置会同步更新纸面 DOM。若同一浏览器持续复现旧行为，先用硬刷新、清站点数据或无痕窗口排除旧前端 bundle / 浏览器缓存，再判断当前 runtime。
- 每个弹窗都会生成独立 `state`，并把草稿保存到当前窗口专属 key；同模板同时开多个窗口时，不再默认共用同一份草稿。
- 已有老草稿仍保留一次兼容回读：若当前窗口级草稿为空，会先尝试读旧的模板级 localStorage 草稿，避免本轮切换后把已有编辑内容直接丢掉。
- 左侧字段面板继续支持按“字段名或字段值”搜索过滤；字段列和值列使用固定宽度，长字段名在本列内换行，不再把值列挤窄。
- 左侧字段编辑与右侧模板继续双向同步：左侧 `input/textarea` 改值会立即回写右侧纸面；右侧 `contenteditable` 输入期会写当前窗口草稿，失焦只是把当前值正常提交回 React state。
- 两套合同都保留“分行编辑 + 右侧纸面原地编辑”双入口：
  - 左侧分行编辑表格适合连续改数量、单价、备注和条款。
  - 右侧纸面适合直接按打印观感微调单元格内容。
- 采购合同和加工合同主表当前都按“整格输入”理解：点击表格里的实际内容块即可直接改当前格；选行模式开启后，点击明细行会只切换当前操作目标，不会顺手改动其它行。
- 右侧纸面默认不显示编辑边框；焦点态统一使用与采购合同 / 加工合同一致的绿色实线内描边和浅绿色底色。主表单元格聚焦时内描边覆盖整个单元格；像物料分析明细表顶部“日期 / 毛向”和底部“审核 / 制表”、色卡顶部“产品编号 / 产品名称”、色卡页脚“制卡 / 日期 / 审核 / 复核”这类标签和值同格的区域，打印视觉按源 Excel 保持非格子排版，标签不进入编辑区；字段名文字和值文字按合同模板口径保持视觉对齐，右侧可编辑值槽保留稳定点击高度和宽度，不能只包住文字或退回过小点击区；值槽里的文字在编辑框内上下居中，长值仍在值槽内换行且不得横向溢出；打印和 PDF 不带出这些焦点样式。
- `上插一行 / 下插一行 / 删除当前行 / 选择明细行 / 选择单元格 / 合并选区 / 拆分当前 / 计算规则` 继续只作用于当前弹窗里的草稿，不会静默回写业务真源。
- `在线预览 PDF / 下载 PDF / 打印` 都以当前弹窗里的纸面 DOM 为输出来源，不再走第二套单独模板：
  - PDF 预览 / 下载前会先收口当前焦点输入，再把同一份右侧纸面主区域 DOM 快照交给后端 `/templates/render-pdf`。
  - 浏览器打印直接打印当前弹窗里的同一份纸面，不输出左侧字段面板。
- 两套合同在内容仍落在单页 A4 内时，继续保持 `0` 物理页边距；一旦内容跨到第 `2` 页及后续页，浏览器打印与在线 PDF / 下载 PDF 会统一切到 `@page margin: 5mm 0 5mm`，给续页顶部和底部留出固定缓冲，避免正文贴页顶开始排。
- 右侧纸面舞台继续统一使用灰色背景；打印态会隐藏左侧字段面板、顶部工具栏、公式面板和附件上传条，只保留 A4 纸面。

## 当前模板差异

### 采购合同

- 中间采购明细区保留表格，合同头、条款和签字区按普通纸质合同排版。
- `采购金额` 默认按 `数量 × 单价` 重算，但仍允许按合同快照手工改写金额列。
- `单位` 是打印草稿快照字段，不回写主数据；采购合同纸面暂按中文单位优先展示。中文来源单位保留原文，例如 `片 / 对 / 米 / 盒` 继续显示中文；带英文括号的单位优先取中文前缀，例如 `米（M）` -> `米`；已知英文 code 会映射回中文：`PCS / PC / EA` -> `件`，`PAIR / PR` -> `对`，`SET` -> `套`，`M / CM / MM / YD / FT / IN` -> `米 / 厘米 / 毫米 / 码 / 尺 / 英寸`，`KG / G / T / LB / OZ` -> `千克 / 克 / 吨 / 磅 / 盎司`，`CTN / BOX / PKG / BAG / ROLL / BTL / DRUM` -> `箱 / 盒 / 包 / 袋 / 卷 / 瓶 / 桶`，`DOZ / GRS / REAM / BDL` -> `打 / 罗 / 令 / 扎`。无法确认映射的单位保留来源文本，不为了统一外观伪造中文单位。
- 行选择、插行、删行会同时维护当前行选择、合并块和合计，不再复制相邻行旧值生成新行。
- 采购明细当前支持原地矩形 merge / split：合并后只保留左上角内容，被覆盖格会清空；删除命中纵向 merge 时会按整块一起删除。

### 加工合同

- 加工明细纸面默认只渲染真实明细行，不再为了凑固定高度额外补两行占位空白行；需要新增空行时，必须通过 `上插一行 / 下插一行` 显式插入。
- 从委外订单带值生成加工合同时，明细行 `产品订单编号` 优先读取 `outsourcing_order_items.product_order_no_snapshot`，旧数据缺失时才回退到委外订单表头 `source_order_no`；该字段只作为合同逐行追溯快照，不新增销售订单外键，也不生成委外事实。
- 明细金额默认按 `委托加工数量 × 单价` 带值；如合同快照已有确认金额，可在左侧分行编辑或右侧纸面直接改写 `委托加工金额`。
- 纸样 / 图样附件通过左侧上传条独立上传后，会同步进入右侧页底附件位，并随 PDF / 浏览器打印一起输出。
- 条款区、签字区和表格区都允许在右侧纸面直接编辑，再回写当前草稿。
- 加工明细当前和采购合同一样支持原地矩形 merge / split；金额列可编辑，也可以参与合并展示。

### 工程资料模板

- `物料分析明细表` 从 BOM 管理选中版本带值打开，服务板房工程部整理材料、部位、用量和工艺方式，打印给仓库对照发料；右上角和页底图片槽只进入当前打印草稿和 PDF / 浏览器打印输出，不替代业务附件归档。
- BOM 管理页当前承接工程资料打印所需业务字段：表头维护 `来源订单号 / 订单数量 / 备品 / 制表日期 / 设计师 / 制表 / 审核 / 毛向`，明细维护 `片数 / 总用量含损耗 / 加工底料 / 加工方式`；这些字段随 BOM 版本和 BOM 明细保存、复制、JSON-RPC 回显和导出，并由 `物料分析明细表`、`色卡`、BOM 来源的 `作业指导书` 生成器读取。它们只服务工程资料、仓库发料对照和打印快照，不写采购、库存、出货、财务或 Workflow 完成事实。
- `物料分析明细表` 的 `毛向` 属于顶部业务信息区字段，在顶部信息区左侧单独换行展示并可编辑，不和 `设计师` 横向挤在同一值槽；源 Excel 把 `毛向` 放在标题右侧靠近图片区的位置按当前口径视为源文件噪点，不照搬为运行时模板结构。
- `物料分析明细表` 支持按源 Excel 结构做当前窗口内的矩形合并 / 拆分：启用 `选择单元格` 后依次点击起点和终点，再执行 `合并选区`；合并后只保留左上角内容，被覆盖格会清空，`拆分当前` 只移除当前合并块。插入 / 删除物料行会维护合并块位置；删除命中纵向合并块时按整块删除，避免纸面残留半个合并区。该能力只影响打印草稿和当前纸面 DOM，不回写 BOM、材料、采购、库存或业务附件。
- `色卡` 从 BOM 管理选中版本带值打开，服务板房工程部打印后线下贴布料 / 物料样本，交给仓库对色发料；色卡分块来自 BOM 明细 / 材料快照或当前草稿，不写材料主数据、采购或库存事实。打印中心默认样例的页脚保留 `制卡 / 日期 / 审核 / 复核`，其中 `审核 / 复核` 默认显示 `审核人 / 复核人`；从业务记录带值打开时，业务草稿明确为空的签核字段仍按空白处理，不用样例值伪造业务事实。
- `作业指导书` 是工程 / 生产 / 品质共用的作业资料打印件，当前保留双入口：BOM 管理选中版本时带产品、版本和工程备注上下文，服务内部工程资料准备；委外订单选中源单时带委外单号、加工厂、加工项目、委外数量和回货日期上下文，服务打印给加工厂执行。正文不再固定拆成 `裁床 / 刺绣 / 印花 / 车缝 / 备注` 字段，而是统一为可变正文行。正文行可切换为标题行、编号行或文本行，默认使用一致行高；编号行按最近的标题行或文本行分组从 `1` 重新编号，匹配源 Excel 中每个小模块独立编号的口径；只有编号行可维护行内图片。右上角产品图和编号行图片都只作为当前打印草稿图片槽，打印不表示委外、生产、质检或库存事实完成；生产进度、来料质检和加工环节页当前不直接生成作业指导书草稿。

## 销售订单受理字段链路

销售订单受理是当前本地试用模拟优先闭环，但它不是当前正式打印模板。销售订单页已经接入客户配置 `sales_orders.default` 字段策略，列表和导出共用同一组可见列；打印侧仍不生成销售订单打印模板，也不把客户专属字段硬编码进 Product Core。

| 字段 / 业务含义 | 当前唯一真源 | 页面采集 / 回显 | 列表 / 导出 | 打印模板状态 | 缺值 / 残值口径 |
| --- | --- | --- | --- | --- | --- |
| 销售订单号 | `sales_orders.order_no` | 销售订单表单与详情使用订单号，不展示内部 ID | `订单号` 列，导出同列 | 未接销售订单打印 | 缺失时按页面空态显示，不从打印样例补造 |
| 客户 | `customer_id` + `customer_snapshot` | 新建 / 编辑从客户主数据带出快照，切换客户时重带联系人和客户默认值 | `客户` 列读取快照；缺快照但有关联时显示“客户已关联” | 未接销售订单打印 | 不从模板默认客户名兜底，历史缺快照只显示既定关联提示 |
| 客户订单号 / 来源号 | `sales_orders.customer_order_no`，字段策略键为 `source_no` | 表单维护客户订单号 | `客户订单号` 列受 `sales_orders.default.source_no` 控制，导出跟随可见列 | 未接销售订单打印 | 字段被策略隐藏时列表和导出同时隐藏，不在打印模板绕过 |
| 业务员 / 跟单人 | `sales_orders.sales_owner` | 表单维护文本值 | `业务员 / 跟单人` 列，导出同列 | 未接销售订单打印 | 为空显示空态，不从登录角色或客户包推断 |
| 联系人 | `contact_snapshot` | 选择客户后从联系人主数据带出；无联系人时清空联系人字段 | `联系人` 列读取联系人快照 | 未接销售订单打印 | 切换客户会清空或替换旧联系人，避免残留上一客户联系人 |
| 付款条件 | `payment_method`、`payment_term_days` 与客户默认配置 | 表单维护并可从客户默认值带出 | `付款条件` 列和导出使用同一格式化函数 | 未接销售订单打印 | 没有真源时显示空态，不从客户模板硬填 |
| 签约日期 | `sales_orders.order_date` | 表单维护 | `签约日期` 列，导出格式化日期 | 未接销售订单打印 | 非法日期由页面 / 后端日期合同拦截，不在打印层修正 |
| 计划交付日期 | `sales_orders.planned_delivery_date`，字段策略键为 `expected_ship_date` | 表单维护 | `计划交付日期` 列受 `sales_orders.default.expected_ship_date` 控制，导出跟随可见列 | 未接销售订单打印 | 字段被策略隐藏时列表和导出同时隐藏 |
| 生命周期状态 | `sales_orders.lifecycle_status` | 生命周期按钮调用后端 usecase / ProcessRuntime，前端不本地伪造事实状态 | `状态` 列和导出使用同一状态文案 | 未接销售订单打印 | submit 进入销售订单受理流程，不等于库存、出货、质检或财务事实过账 |
| 明细行 | `sales_order_items` 及其产品 / 数量 / 金额快照 | 表单明细区采集和回显 | 当前列表主表不导出明细；`sales_order_items.default` 尚未发布为 active field policy | 未接销售订单打印 | 明细字段未完成页面 / 字段策略 / 打印全链路前，不发布客户配置字段策略 |

后续若要新增销售订单确认单、出货前确认单或其他销售类打印模板，必须先定义领域模型到打印草稿的 mapper / view model，并补齐表单采集、保存、列表 / 详情 / 导出 / 打印、`fieldRequirements`、`moduleKeys`、服务端 PDF 模块门禁和历史缺值回补测试；不能直接在模板里写死永绅字段、客户抬头、签章或旧资料截图。
