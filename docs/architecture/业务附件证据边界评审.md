# 业务附件证据边界评审 / Business Attachment Evidence Boundary Review

## 结论

业务附件是 Reporting / Audit / Integration 层的证据能力，不是 MasterData、Source Document、Fact 或 Workflow 的状态真源。

当前实现新增统一 `business_attachments` 表和 `attachment` JSON-RPC 域，把文件挂到既有业务对象：

| owner_type | 对应业务对象 | 当前接入页面 |
| --- | --- | --- |
| `sales_order` | 销售订单 | 销售订单编辑弹窗 |
| `purchase_order` | 采购订单 | 采购订单编辑弹窗 |
| `outsourcing_order` | 委外 / 加工合同源单 | 加工合同编辑弹窗 |
| `purchase_receipt` | 采购入库单 | 入库草稿 / 加行弹窗 |
| `quality_inspection` | 来料质检单 | 质检创建 / 判定弹窗 |
| `shipment` | 出货单 | 出货单编辑弹窗、收窄出货事实页 |
| `finance_fact` | 财务事实 | 应收、应付、发票、对账收窄页面 |
| `production_fact` | 生产事实 | 生产进度收窄页面 |
| `outsourcing_fact` | 委外事实 | 委外事实收窄页面 |
| `product_sku` | SKU 主数据 | 产品 / SKU 页面 |
| `bom_header` | BOM 版本 | BOM 管理页面 |
| `workflow_task` | Workflow 协同任务 | Workflow V1 页面、岗位任务端详情 |

## 页面布局口径

- 销售订单、采购订单、委外订单、出货单、BOM、SKU 等带主表字段和明细 items 的编辑弹窗中，附件作为主对象证据字段处理，默认放在备注、交付、合同资料或凭证语义附近，并位于订单行、BOM 明细、出货明细等 item 区之前。
- 新建未保存记录允许先选择附件并暂存在当前前端表单内，保存业务记录成功拿到 owner_id 后再自动上传绑定；取消或保存失败不得生成悬空附件。已有记录无附件时显示紧凑空态，不在弹窗末尾撑出独立大区块，避免明细增多后用户必须滚到最后才能上传。
- 页面级选中记录附件，例如收窄事实页、Workflow 桌面页和岗位任务端详情，可保留独立区块，但仍必须绑定当前已保存 owner，并保持“证据能力、不改业务事实”的语义。

## 真源边界

- 附件只能证明“该业务记录存在一份外部资料或现场证据”。
- 附件上传、下载、删除和轻量预览不改变订单生命周期、BOM 状态、质检判定、出货状态、库存流水、财务事实、Workflow 任务状态或业务状态投影。
- 出货单 `SHIPPED`、库存 `inventory_txns`、财务 `finance_facts`、质检判定、Workflow task status 等仍由各自 usecase 负责。
- `workflow_task` 附件不等于任务完成；岗位任务端仍必须通过 Workflow API 执行处理、阻塞、退回或完成动作。
- `finance_fact` 附件不替代税控、发票查验、纳税申报、总账凭证、付款审批、收付款核销或多账簿。

## API 与权限

`attachment` JSON-RPC 当前提供：

- `list_attachments`
- `upload_attachment`
- `download_attachment`
- `delete_attachment`

权限不另造一套页面旁路；服务端按 `owner_type` 复用所属业务对象的读写权限。例如销售订单附件读取需要 `sales_order.read`，上传 / 删除需要销售订单创建或更新类权限；Workflow 任务附件读取需要 `workflow.task.read`，上传 / 删除需要任务创建、更新、完成或驳回类权限。

前端预览只复用 `download_attachment` 读取内容：PNG、JPG / JPEG、WEBP、GIF 和 PDF 可在当前页面轻量预览；HEIC / HEIF、Word、Excel、CSV、文本、ZIP、邮件证据和 WPS 文件继续下载后查看。预览不新增 `/files` 静态外链、不绕过 owner 权限，也不把附件内容写回业务对象。

当前允许上传的证据格式：

| 分组 | 扩展名 / 类型 | 预览策略 |
| --- | --- | --- |
| 图片证据 | `.png`、`.jpg`、`.jpeg`、`.webp`、`.gif` | 当前页面轻量预览 |
| 手机图片证据 | `.heic`、`.heif` | 允许上传 / 下载；不承诺浏览器内预览或在线转换 |
| PDF | `.pdf` | 当前页面轻量预览 |
| Office 文档 | `.doc`、`.docx`、`.xls`、`.xlsx` | 下载后查看 |
| 数据 / 文本 | `.csv`、`.txt` | 下载后查看 |
| 资料包 | `.zip` | 下载后查看 |
| 邮件证据 | `.eml`、`.msg` | 下载后查看 |
| WPS 文件 | `.wps`、`.et`、`.dps` | 下载后查看；前端按扩展名归一到受控 MIME，不放开任意 `application/octet-stream` |

服务端还做这些守卫：

- 必须是后台管理员登录态。
- `owner_type` 必须在白名单内。
- `owner_id` 必须对应真实业务记录。
- 单个附件最大 50MB；这是解码后的文件内容上限，JSON-RPC base64 请求体会更大，当前仍不把附件层升级为对象存储或大文件平台。
- 仅允许上表列出的证据格式，且文件扩展名必须与受控 MIME 合同匹配；不接受任意二进制、脚本、HTML、SVG 或可执行文件。
- JSON-RPC 日志会脱敏 `content_base64`、`base64`、`content`、`file_content` 等大字段或敏感字段。

## 当前不包含

- 不实现对象存储、CDN、缩略图、OCR、病毒扫描、电子签章或全文检索。
- 不实现 Office 在线转换、CSV / 文本在线渲染、客户外链共享或跨权限临时下载链接；当前只允许图片 / PDF 轻量预览。
- 不实现版本化附件、附件审批流、附件归档保管期限、附件水印或客户外链共享。
- 不把附件拆成每个业务表的字段，也不把附件 ID 写回业务事实表。
- 不做客户真实文件导入或历史 backfill。

## 验收口径

代码层至少核对：

1. `business_attachments` migration 和 Ent generated code 存在。
2. `attachment` JSON-RPC 的 list/upload/download/delete 可通过服务端测试或真实 RPC 调用验证。
3. P0/P1 页面只把附件挂到已保存业务记录；新建未保存时只能保留前端待上传附件，保存失败或取消不得生成悬空附件。
4. 上传、下载、删除或预览附件后，所属业务对象状态、事实状态和 Workflow 任务状态不发生变化。
5. 超过 50MB、非法 MIME、无权限或不存在 owner 的请求被拒绝。
