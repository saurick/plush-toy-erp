# 材料 / 产品 / 库存 schema 早期评审记录 / Material Product Inventory Schema Review

> 当前状态：本文是早期 schema 评审记录，已经被后续 Ent schema、Atlas migration、InventoryUsecase、Purchase/Quality/operational fact usecase 和 `business_records` 删除闭环替代。当前实现真源以 `docs/当前真源与交接顺序.md`、领域 usecase、Ent schema 和测试为准。

## 当前结论

- 旧 `business_records / business_record_items / business_record_events` 表族已由 `20260612112337` migration 删除。
- 本文历史上关于“通用表保留 legacy/archive 查询、打印取值候选、debug fixture、source snapshot 或旧数据回补来源”的口径已过期。
- 当前库存事实真源是 `inventory_txns`；`inventory_balances` 是当前余额 / 查询加速表；`inventory_lots` 是批次追溯和批次可用性状态真源。
- 当前主数据和工程资料底座包括 `units`、`materials`、`products`、`warehouses`、`bom_headers`、`bom_items`。
- 当前采购和质检最小事实包括 `purchase_receipts`、`purchase_returns`、`purchase_receipt_adjustments`、`quality_inspections`。
- 当前业务事实最小实现包括 `production_facts`、`outsourcing_facts`、`shipments`、`shipment_items`、`stock_reservations`、`finance_facts`。

## 仍有效的设计边界

| 边界 | 当前口径 |
| --- | --- |
| 库存流水 | 已生效库存流水不允许物理修改或删除；错误通过 `REVERSAL` 冲正 |
| 库存余额 | 余额是查询加速 / 当前结果，必须能由事实流水校验 |
| 数量类型 | 库存、BOM、采购、退货和调整数量 / 金额使用 decimal/numeric，不使用 float |
| Workflow / Fact | 协同任务完成不等于库存、质检、出货或财务事实落账 |
| Import evidence | 删除前 JSONL evidence 只能作为人工审计线索，不是运行时 source 或自动回补来源 |
| 旧表恢复 | 不恢复 `business_records` 作为历史快照、兼容查询、debug fixture 或通用事实表 |
