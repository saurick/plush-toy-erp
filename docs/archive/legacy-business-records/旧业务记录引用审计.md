# 业务记录引用审计 / business_records Reference Audit

本文记录旧 `business_records / business_record_items / business_record_events` 删除后的引用边界。当前实现真源仍以 `docs/当前真源与交接顺序.md`、Ent schema、Atlas migration、代码和测试为准。

## 当前结论

- 旧表族已由 Atlas migration `20260612112337` 删除。
- 旧通用记录 usecase、repo、Ent schema、generated code、前端 API client、前端表单/打印/source prefill helper 和旧通用记录测试已删除。
- 普通 `business` JSON-RPC 域只保留 `dashboard_stats`，旧记录方法已退出运行时。
- debug seed / cleanup 不再写旧表族。
- 采购事实对象和 schema 已移除 `business_record_id` 兼容字段；公开采购 API 仍拒绝输入 `business_record_id`，这是防恢复守卫，不是兼容来源。
- 前端 `BusinessModulePage`、旧通用业务模块路由和旧入口退出页已删除；移动端任务详情不再请求旧记录 source。

## 后续审计口径

新增功能或后续迁移中，出现下列情况必须先做边界评审：

- 重新新增 `business_records` 或语义相近的通用事实表。
- 让 Workflow payload 或旧 JSONL evidence 成为库存、出货、质检、财务或应收应付事实来源。
- 为历史查询复活普通 `business` 旧记录 API。
- 在前端恢复旧通用业务表单、打印、删除、恢复或任务派生链路。
