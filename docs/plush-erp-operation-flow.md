## 为什么不能照搬 trade-erp

`trade-erp` 的正式主线是报价、外销、出运、结汇。  
当前毛绒工厂原件展示的真实主线是：

`客户 / 款式立项 -> 材料 BOM -> 加工合同 / 委外下单 -> 生产排单 / 跟进 / 延期 / 返工 / 异常 -> 仓库收发 / 成品入库 / 待出货 -> 对账 / 结算 / 应收应付`

如果继续沿用 `trade-erp` 的外贸表和页面，会直接把编号体系、单据关系、字段语义和角色入口都带偏。

## 当前正式主线

### 1. 客户 / 款式立项

真源依据：

- 生产订单总表截图给出：`客户`、`订单编号`、`客户订单号`、`产品编号`、`产品名称`、`颜色`、`出货日期`、`跟单业务人员`
- 两份材料分析 Excel 给出：`款式编号（26029# / 26204#）`、`产品名称`、`订单编号`、`数量`
- 加工合同 PDF 给出：`产品订单编号`、`产品编号`、`产品名称`

当前结论：

- 当前至少存在四层编码：`客户 / 订单编号`、`产品订单编号`、`款式编号`、`产品编号 / SKU`
- 这四层不能再混成一个字段
- 跟单是这条链路的统一入口，老板负责审核包装材料打单表与生产资料

### 2. 材料 BOM / 主料 / 辅材 / 包材

真源依据：

- `26029#夜樱烬色才料明细表2026-1-19.xlsx`
- `26204#抱抱猴子材料明细表2026-4-10.xlsx`
- `辅材、包材 成慧怡.xlsx`

当前结论：

- 主料 BOM 真源来自 `材料分析明细表`
- 采购汇总是从 BOM 派生出来的，不是反向真源
- 辅材 / 包材来自独立采购汇总表，不能混入主料明细
- 色卡和作业指导书属于附件 / SOP 资料层

### 3. 加工合同 / 委外下单

真源依据：

- `9.3加工合同-子淳.pdf`
- `加工 成慧怡.xlsx`

当前结论：

- 加工合同 PDF 是正式单据快照，至少包含合同头、合同行、结算条款、回货日期和附件图片
- 委外加工汇总表是导入 / 历史台账来源，字段包括委外加工订单号、加工项目、厂家名称、工序类别、单价、数量、金额
- 加工商资料表是 partner 主档候选来源，但包含敏感信息和脏格式，当前只先收口文档设计

### 4. 生产排单 / 跟进 / 延期 / 返工 / 异常

真源依据：

- 正式汇报版 PDF 第 4 页给出老板视角流程图
- 第 7 页给出 `PMC 看板` 和 `生产经理看板`
- 生产订单总表截图给出 `订单数量`、`损头版`、`生产数量`、`未出货数`

当前结论：

- PMC 负责全流程跟单、齐套跟进、交期跟进
- 生产经理负责决定车缝 / 手工 / 内外发走向
- 返工和异常必须独立成日志，不能继续沉在备注
- 当前只做排单、进度回填、延期原因、返工、异常中心，不做扩展硬件链路 / 工时采集

### 5. 仓库收发 / 成品入库 / 待出货

真源依据：

- 正式汇报版 PDF 第 4 页主线与包装材料支线
- 生产订单总表截图里的 `出货日期`、`未出货数`

当前结论：

- 主料 / 辅料到仓后要经过 `仓库 + 品质 IQC`
- 包装材料有独立支线：下单 -> 到仓 -> 质量检查 -> 包装环节领用
- 车缝 / 手工完成后成品回仓并检验，再入成品仓
- 发货前需要 `业务确认 + 仓库出货单 + 财务放行`

### 6. 对账 / 结算 / 应收应付

真源依据：

- 加工合同 PDF 的结算方式条款
- 加工汇总 Excel / 辅包材采购 Excel 的金额列

当前结论：

- 当前已能明确加工费、辅材 / 包材费用和待付款提醒的入口
- 还没有正式结算单 / 对账单样本，暂不硬建完整账务模型
- 财务移动端先做待对账、待付款、异常费用和结算提醒

## 当前打印模板出口

- `C类辅料采购合同`：采购 / 辅材 / 包材下单、供应商确认与留档
- `B类委外加工合同`：委外加工下单、回签、财务对账留档
- `材料分析汇总表`：主料采购汇总与内部核对
- `加工分析汇总表`：工序分析与补加工核对
- `生产订单总表`：老板 / 跟单 / 生产经理共用的经营快照报表

## 角色分工收口

| 角色 | 桌面后台重点 | 移动端重点 |
| --- | --- | --- |
| 老板 / 管理层 | 待审核、延期预警、异常订单、待结算 | 交期风险、异常、未完成数量、本周重点 |
| 业务跟单 | 客户 / 款式、资料缺口、包装材料打单、出货确认 | 缺资料、催料、催合同、交期预警 |
| 采购 | 主料需求、辅包材采购、加工合同、回签 | 缺料、到料、单价确认、回签 |
| PMC / 生产经理 | 齐套、排单、延期、返工、异常 | 今日排产、进度回填、延期原因、返工 |
| 仓库 | 到仓、IQC、成品入库、待出货 | 收货、备料、入库、异常件处理 |
| 财务 | 对账、待付款、放行、异常费用 | 待对账、待付款、异常费用、结算提醒 |

## 本轮明确 deferred

以下能力在原件或汇报 PDF 里出现过，但本轮统一只保留为 deferred，不假装已支持：

- 扩展硬件链路
- PDA
- 条码枪
- 图片识别
- 现场质检
- 离线同步

## 当前不改的路径

- 不把 `trade-erp` 的报价 / 外销 / 出运 / 结汇主线搬进来
- 不新增第二套后端服务
- 不把桌面后台拆成多个桌面站点
- 不在字段关系还没稳定前把流程状态强行绑定到正式数据库 schema
