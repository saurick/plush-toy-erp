# 毛绒 ERP 主流程草案

## 当前正式主线

`客户 / 款式 -> BOM / 材料 -> 加工合同 -> 生产排单 -> 仓库收发 -> 对账结算`

这条链路是当前仓库的正式业务主线，用于替换 `trade-erp` 的报价、外销、出运、结汇那条外贸链路。

## 分阶段口径

### 1. 客户与款式立项

- 业务跟单维护客户、款式编号、版本、交期和打样状态。
- 这里是后续材料、合同和排单的共同起点。

### 2. BOM / 材料拆解

- 维护主料、辅料、包材、工艺和损耗口径。
- 资料到齐后再做 Excel 导入，不在不完整资料上强行补字段映射。

### 3. 加工合同 / 委外下单

- 收口合同模板、加工单位、单价、付款节点和附件。
- 这轮先做入口与帮助文档，不做正式打印模板。

### 4. 生产排单 / 进度跟催

- 厂长与生产角色看排产、延期、返工和完工节点。
- 手机端先做进度确认与异常上报。

### 5. 仓库收发 / 成品入库

- 仓库角色看待收货、待入库、备货和异常登记。
- 本轮明确不做扫码入库。

### 6. 对账 / 结算 / 经营复盘

- 财务角色看加工费、辅材包材、应收应付和异常费用。
- 管理层看延期、待结算和高风险订单。

## 边界说明

- 先收口流程和资料入口，再接真实合同和 Excel。
- 不把拍照扫码、打印模板和自动识别混入当前主路径。
- 后续新增正式业务实体时，继续以这份流程文档为入口校验。
