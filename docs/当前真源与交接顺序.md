补充：菜单投影结论以 `adminProfileSync` 与 `ERPLayout` 当前代码为准。`adminProfileSync` 只负责把 effective session 归一到 admin profile，并通过 `filterNavigationSectionsByAdminProfile` / `shouldRedirectFromCurrentNavigation` 返回前端菜单过滤和当前 URL 是否应跳转的 helper 判定；`customer_config.get_effective_session` 拉取、cached effective session 复用、`effective_session_sync_failed` 空投影挂载，以及实际 `navigate(..., { replace: true })` fallback 跳转都由 `ERPLayout` 负责。前端菜单投影固定两层：第一层是 RBAC 菜单路径，普通账号必须命中 `allowedMenuPaths`；第二层是 `effective_session.pages` 页面 key，普通账号在 `pages` 是数组时必须命中页面 key，空数组会收窄为无可见页面，不退回 RBAC-only。super admin 是产品核心 / 客户系统全功能审阅和配置账号，在前端菜单路径、active pages、active actions 和 field policy 可见性上都不被 `effective_session` 再收窄；页面范围仍只来自导航定义和 React 路由，不把未登记 URL 升级成正式入口。`pages` 缺失或不是数组时，正式普通账号不回退 RBAC-only；通过 `attachEffectiveSessionToAdminProfile` 挂载的 effective session 即使输入缺少 pages，也会被归一为空数组。`getAdminProfileSyncErrorAction` 的 `hasCachedProfile` 只决定同步失败错误的动作分类；`ERPLayout` 在客户配置同步失败时仍只复用 `adminProfileRef.current.effective_session`，普通 `me` profile 缓存不等于已经存在客户配置投影缓存。helper 本身不登记页面、不校验当前 URL 是否属于正式菜单；菜单项过滤的 page key 只来自 `ERPLayout` 传入的导航定义，隐藏 URL 判定使用 `resolveCurrentNavigationEntry`：已登记 exact / prefix 路径才返回 page key，未命中菜单定义时只返回工作台显示 fallback，`pageKey / menuPath` 保持为空。`printTemplateDefaults` 也必须从 active revision 的 `get_effective_session` 投影进入正式前端；当前已接入的消费方是采购订单页 `material-purchase-contract` 和委外订单页 `processing-contract` 打印入口，只读取买方 / 委托方默认字段，不覆盖供应商 / 加工方业务快照，不启用销售订单打印模板。

隐藏 URL 跳转也是 helper 判定而不是授权来源：直接打开已登记菜单路径但 RBAC 未授权、已登记页面被 active revision 隐藏，或 pages 判定不属于上述诊断例外时，`shouldRedirectFromCurrentNavigation` 只返回是否需要跳转；`ERPLayout` 只有在已过滤后的 `visibleSections[0].items[0].path` 存在时才 `replace` 到第一个可见入口。没有可见 fallback 时只显示“当前账号暂无可见后台入口”并阻止业务 `Outlet`，不会跳到隐藏页、RBAC-only 页面或默认全量后台。当前 URL 的 RBAC 判断来自 `resolveMenuPermissionKey(location.pathname)` 解析出的 `currentMenuPath`，pages 判断来自 `resolveCurrentNavigationEntry` 对未过滤菜单定义的解析结果：已登记 exact / prefix 路径才返回 page key；未命中菜单定义时只返回工作台显示 fallback，`pageKey / menuPath` 为空。这个 fallback 只服务页头展示，不把原始 URL 升级为菜单入口、授权入口或业务页面准入；是否渲染业务内容仍取决于 React 路由、已过滤菜单是否为空、当前页面实际路由和对应后端权限。

`ERPLayout` 在 `get_effective_session` 同步失败时只复用 `adminProfileRef.current.effective_session` 这个客户配置投影缓存；普通管理员 `me` profile 缓存不等于客户配置投影缓存。已有正常 cached effective session 时继续复用正常投影；缓存本身已经是 sync-failed 空投影时才继续复用该空投影；没有客户配置投影缓存时才挂载新的 `effective_session_sync_failed` 空投影。active revision 正常返回空页面清单不是 sync failure，而是按空 active pages 投影处理。`web/src/erp/utils/adminProfileSync.test.mjs` 覆盖正式普通账号 sync failure 不退回 RBAC-only、本地开发直达 URL pages 诊断不放开普通账号菜单、super admin 产品核心看全、当前页面被 active pages 隐藏时普通账号需要跳转，以及 super admin 不受 field policy 隐藏列收窄；`web/src/erp/utils/currentNavigationEntry.test.mjs` 覆盖已登记路由保留 page key、未登记 URL 只使用显示 fallback 且不授予 page key；`scripts/qa/formal-frontend-customer-config-boundary.test.mjs` 只静态锁住 `ERPLayout` 仍存在空入口提示和 sync-failed helper anchor。这些测试只锁住前端 helper / 页面壳边界，不替代后端 RBAC、active revision、目标环境 smoke 或 release evidence。
