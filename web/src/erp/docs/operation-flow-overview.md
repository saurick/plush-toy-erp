# ERP 流程图总览

> 目的：给新同事、实施和验收人员一个先读入口，用最短路径理解当前毛绒 ERP 的业务主线、角色边界和验收入口。

## 当前主线

当前系统已经按 `business_records + workflow_tasks + workflow_task_events + workflow_business_states` 接入 6 条真实业务闭环：

这 6 条是 v1 主干闭环，不是所有业务全覆盖。它们用于先把订单、采购、委外、生产、仓库、品质和财务的主路跑通；库存余额、付款流水、完整财务专表、PDA、条码枪、图片识别和更细行业专表仍按 `当前明确不做` 的边界继续 deferred 或后续评审。

| 闭环                                     | 当前状态  | 默认责任角色     |
| ---------------------------------------- | --------- | ---------------- |
| 订单提交 -> 老板审批 -> 工程资料任务     | 已接入 v1 | 业务、老板、工程 |
| 采购到货 -> IQC -> 入库                  | 已接入 v1 | 采购、品质、仓库 |
| 委外发料 -> 回货 -> 检验 -> 入库         | 已接入 v1 | 生产、品质、仓库 |
| 成品完工 -> 成品抽检 -> 成品入库 -> 出货 | 已接入 v1 | 生产、品质、仓库 |
| 出货 -> 应收登记 -> 开票登记             | 已接入 v1 | 仓库、财务       |
| 采购/委外 -> 应付登记 -> 对账            | 已接入 v1 | 仓库、财务       |

## 流程速览

| 分段               | 先看什么                                 | 业务含义                                           |
| ------------------ | ---------------------------------------- | -------------------------------------------------- |
| 基础资料与业务立项 | 客户、订单编号、产品订单编号、款式、交期 | 先把“这是什么订单”说清楚，再进入老板审批和工程资料 |
| 材料与委外准备     | 主料、辅包材、加工合同、供应商回签       | 先确认采购和委外准备是否齐套，再让 PMC 推排产      |
| 仓库与品质联动     | 到货、IQC、委外回货、成品抽检、入库      | 仓库负责实物流转，品质负责检验结论                 |
| 生产与出货         | 排产、进度、返工、成品入库、待出货       | 生产完成不等于财务完成，出货后还要接应收和开票     |
| 结算与移动端协同   | 应收、开票、应付、对账、任务处理         | 财务收口金额和放行，手机端只处理任务和反馈         |

## 查看完整版流程文档

如果只想知道自己今天要做什么，先回帮助中心首页按角色查；如果要校对字段、状态、打印或验收，继续看 `工作流主任务树 v1`、`仓库与品质 v1`、`财务 v1` 和 `当前明确不做`。

## 推荐阅读顺序

1. 先看 `工作流主任务树 v1`，确认当前 6 条闭环和不做项。
2. 再看 `角色权限矩阵 v1`，确认每个角色能看什么、能处理什么。
3. 再看 `通知 / 预警 / 催办 / 升级 v1`，确认预警和关注项怎么出现。
4. 再看 `仓库与品质 v1`、`财务 v1`，确认入库、检验、出货、应收和开票边界。
5. 最后看 `验收结果总览`，确认当前命令、盲区和下一步排查入口。

## 当前边界

- 当前不拆行业专表，暂不新增 `production_order`、`shipment_order`、`inventory_txn`、`ar_receivable`、`ar_invoice`、`ap_payable`、`ap_settlement` 等 Ent schema。
- 当前不做真实库存余额和库存流水专表。
- 当前不做完整总账、凭证、会计科目和纳税申报。
- 当前不新增独立 notification 表和外部推送。
- 当前没有完整业务 E2E 造数 runner；验收主要依赖单元测试、`style:l1`、移动端 smoke 和只读链路调试。

## 排查入口

| 问题                         | 入口                                                                              |
| ---------------------------- | --------------------------------------------------------------------------------- |
| 想看完整业务闭环             | `工作流主任务树 v1`                                                               |
| 想看角色是否能处理任务       | `角色权限矩阵 v1`、`手机端角色流程`                                               |
| 想看任务为什么移动端不可见   | `协同任务调试`，优先检查 `owner_role_key`、`task_group`、终态和移动端任务查询策略 |
| 想看字段残值、缺值、打印取值 | `字段联动覆盖`                                                                    |
| 想看本轮验收命令和盲区       | `验收结果总览`、`运行记录`、`专项报告`                                            |
