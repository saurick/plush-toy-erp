# 任务 / 业务状态字典

> 适用对象：老板、PMC、生产经理、采购、仓库、品质、财务、系统管理员
> 当前用途：统一任务状态、业务状态和推进阶段，供桌面工作台、手机端任务页、帮助中心和后端保存链路继续对线。
> 当前约束：这是当前仓库的正式状态口径；workflow 和通用业务记录 v1 已通过 Ent + Atlas 落地，细分业务专表仍以后续迁移为准。

## 1. 先把三层状态分开

1. `任务状态` 只回答某个协同动作现在能不能做、谁在做、为什么卡住。
2. `业务状态` 只回答某个订单、批次或单据所在的主链阶段。
3. `推进阶段` 只回答当前功能建设做到哪一层，用于产品 / 实施 / 研发协同，不写进正式业务单据。

## 2. 任务状态字典

| 状态         | 含义                       | 谁最常用                  | 典型触发条件                                 | 下一步                                       |
| ------------ | -------------------------- | ------------------------- | -------------------------------------------- | -------------------------------------------- |
| `pending`    | 任务已创建，但前置条件未齐 | 业务、PMC、采购           | 立项已发起，但资料、审批、回签或到料未满足   | 等上游补齐后进入 `ready`                     |
| `ready`      | 可以开始执行               | PMC、生产经理、仓库、品质 | 单据、资料、齐套或上游确认条件已满足         | 责任人接手后进入 `processing`                |
| `processing` | 已有人接手，正在推进       | 全角色                    | 已开始采购、排单、检验、返工、对账或放行处理 | 完成则进 `done`，卡住则进 `blocked`          |
| `blocked`    | 被外部条件卡住             | PMC、老板                 | 缺料、缺资料、未回签、未放行、数量差异       | 等阻塞原因解除，再回 `ready` 或 `processing` |
| `done`       | 当前任务完成               | 全角色                    | 当前任务的完成条件已达到                     | 可推动下游任务创建或完成                     |
| `rejected`   | 被退回                     | 老板、品质、财务          | 审批未通过、检验退回、财务未放行             | 回上一责任人修正后再走主链                   |
| `cancelled`  | 任务取消                   | 业务、PMC                 | 订单取消、方案切换、需求撤销                 | 仅保留历史，不再继续流转                     |
| `closed`     | 已归档关闭                 | 管理层、系统管理员        | 主链闭环、历史沉淀完毕                       | 进入历史追溯阶段                             |

## 3. 业务状态字典

| 状态                    | 含义            | 典型主责角色     | 典型进入条件                       | 典型离开条件            |
| ----------------------- | --------------- | ---------------- | ---------------------------------- | ----------------------- |
| `project_pending`       | 立项待确认      | 业务             | 客户 / 款式立项刚创建              | 资料和审批前置满足      |
| `project_approved`      | 立项已放行      | 老板、业务       | 老板或管理层放行                   | 开始进入资料准备        |
| `engineering_preparing` | 资料准备中      | 业务、采购       | BOM、色卡、作业指导书开始补齐      | 达到可采购 / 可生产条件 |
| `material_preparing`    | 齐套准备中      | 采购、PMC        | 主料、辅包材、回签或回货仍在推进   | 满足可排条件            |
| `production_ready`      | 待排产          | PMC、生产经理    | 齐套条件满足                       | 给出排单决策            |
| `production_processing` | 生产中          | 生产经理、PMC    | 已进入裁切、车缝、手工、组装或外发 | 完工待检或出现异常      |
| `qc_pending`            | 待检验          | 品质、仓库       | 到仓待检、过程待检、返工待复检     | 放行或退回              |
| `warehouse_processing`  | 待入库 / 待出货 | 仓库、业务       | 成品回仓、待出货准备               | 财务放行并进入出库      |
| `shipping_released`     | 已放行待出库    | 业务、财务、仓库 | 业务确认 + 财务放行完成            | 形成出库事实            |
| `shipped`               | 已出货          | 仓库、业务       | 出库事实形成                       | 进入对账 / 结算         |
| `reconciling`           | 对账中          | 财务             | 出货和金额快照已具备               | 对账闭环                |
| `settled`               | 已结算          | 财务、老板       | 对账完成且结算确认                 | 进入归档                |
| `blocked`               | 业务阻塞        | PMC、老板        | 主链被缺料、延期、未放行或异常卡住 | 阻塞解除后回主链        |
| `cancelled`             | 业务取消        | 业务、管理层     | 订单或批次整体取消                 | 保留历史快照            |
| `closed`                | 业务归档        | 财务、系统管理员 | 主链和结算都闭环                   | 转历史查询              |

## 4. 推进阶段字典

| 阶段                 | 当前含义           | 主要使用方       | 当前应该产出什么                                                 |
| -------------------- | ------------------ | ---------------- | ---------------------------------------------------------------- |
| `source_locked`      | 真源已收口         | 产品、实施、研发 | 明确唯一真源字段、单据和角色边界                                 |
| `page_defined`       | 页面已收口         | 前端、产品       | 明确菜单、角色入口、帮助中心和工作台卡片                         |
| `status_defined`     | 状态已统一         | 前后端、产品     | 统一任务状态、业务状态、阻塞原因                                 |
| `schema_v1_ready`    | Schema v1 已落地   | 后端、架构       | workflow 和通用业务记录表已通过 Ent + Atlas 生成迁移             |
| `api_v1_ready`       | API v1 已接通      | 后端、前端       | workflow 与 business JSON-RPC 已支持当前表格、弹窗和任务池主路径 |
| `save_link_v1_ready` | 保存链路 v1 已接通 | 前后端、测试     | 桌面业务页可保存通用业务记录、写状态快照并创建协同任务           |

## 5. 当前推荐使用方式

1. 页面上优先展示 `任务状态`，因为它最贴近“现在谁该处理”。
2. 订单、批次、单据详情优先展示 `业务状态`，因为它更适合表达主链阶段；桌面业务页当前支持按状态字典流转业务状态，并同步写入业务状态快照。
3. `推进阶段` 只出现在帮助中心、changes 记录或研发协作文档，不应写回业务记录。
4. `blocked` 和 `cancelled` 在桌面业务页流转时必须填写原因；原因会写入业务记录 payload、业务状态快照和关联未终止任务的阻塞 / 取消说明，避免状态变化后任务池仍停留在旧状态。
5. `due_soon`、`overdue` 建议继续作为聚合层或前端派生态，不先当数据库真字段。

## 6. 当前明确不做的事

1. 不让每个页面自己发明一套状态名。
2. 不把 `任务状态` 当成 `业务状态` 的替代品。
3. 不把 `推进阶段` 写回客户订单、BOM、合同、入库或对账记录。
4. 不在当前阶段引入复杂状态机引擎或自动派单引擎。
