# 模板打印与字段口径

## 当前目标

本轮固定模板工作流的主路径只保留两套正式合同模板：

- `采购合同`
- `加工合同`

当前实现边界：

- 已做：业务页选中记录带值打开打印窗口、统一打印中心默认样例入口、左右双栏编辑壳页、PDF 预览 / 下载、浏览器打印、模板口径文档、帮助中心入口
- 当前编辑窗口已补到窗口级 `state` 壳页恢复：从业务页或打印中心打开都会先经过 `/print-window-shell.html`，再进入当前模板工作台；工作台稳定后会持续回写整窗 HTML 快照，刷新时壳页优先直接恢复当前窗口；同模板多窗口不再默认共用一份草稿。
- 当前在线 PDF 预览统一在独立预览窗口中打开，不再在工作台内分叉成页内弹层和新标签两套交互
- 当前 PDF 预览 / 下载已收口到统一链路：前端先冻结当前右侧纸面主区域 DOM 快照，再走后端 `/templates/render-pdf` 统一生成 PDF，不再保留客户端各自导出引擎
- 当前合同打印分页继续按当前 A4 纸面口径收口：浏览器打印、在线 PDF 预览和下载 PDF 都以同一套 A4 盒模型输出，明细行、条款块和签字区不会在分页处被直接截断；单页继续保持 `0` 页边距，若内容跨到第 `2` 页及后续页，会统一切到 `@page margin: 5mm 0 5mm`，避免正文贴页顶开始排
- 未做：PDF 坐标回填、Excel 母版回写、打印结果自动留档或反向回写业务记录、图片识别 / 扩展硬件链路 / 盖章自动识别

## 当前已落模板

| 模板     | 真源文件                                                             | 当前用途                                  | 字段性质                           |
| -------- | -------------------------------------------------------------------- | ----------------------------------------- | ---------------------------------- |
| 采购合同 | `模板-材料与加工合同.xlsx` 的 `C类辅料合同`                          | 辅材 / 包材采购下单、供应商确认、留档打印 | 合同头和明细都是业务快照           |
| 加工合同 | `模板-材料与加工合同.xlsx` 的 `B类加工合同` + `9.3加工合同-子淳.pdf` | 委外加工下单、回签、财务对账留档          | 合同头、合同行、条款都属于合同快照 |

## 模板分类规则

当前打印中心只保留合同类模板。

规则：

- 合同编号、下单日期、回货日期、对方名称、联系人、电话、地址都按打印当下保存为快照
- 合同条款是正式业务口径的一部分，不能只写在帮助中心里，不出现在模板里
- 合同金额、单价、数量后续即使主档调整，也不能反写历史合同打印件

## 首批固定字段口径

### 采购合同

关键字段：

- `采购订单号`
- `产品订单编号`
- `产品编号`
- `产品名称`
- `材料品名`
- `厂商料号`
- `规格`
- `单位`
- `单价`
- `采购数量`
- `采购金额`
- `备注`

注意：

- `供应商名称 / 联系人 / 联系电话 / 供应商地址` 当前来自 `材料厂商编号` sheet
- `订货人 / 联系电话` 当前来自 `原辅料采购汇总表`
- `厂商料号` 列在真实采购样本里经常承载供应商简称；打印页先保留原列名，帮助中心再提醒导入清洗
- `采购金额` 默认按 `采购数量 × 单价` 带值；如果合同快照已有确认金额，工作台允许直接手工改写金额列
- 工作台里的 `采购金额` 编辑框会显示 `¥` 前缀，并把输入收口成纯数字金额；右侧合同快照和 PDF 仍保持纯数字，不把货币符号写进真值
- 采购合同纸面严格按实拍样本收口：只有中间采购明细区保留表格，其余合同头、条款和签字区都按普通排版输出
- 工作台上插 / 下插明细行时会新增真正空白行，不再复制当前行的订单号、产品号、单位或其他字段
- 工作台当前支持“选择单元格 / 合并选区 / 拆分当前”；合并后仅保留左上角内容，删除命中纵向 merge 时会按整块删行
- 签字区默认留给纸质手写签名，不带电子印章

### 加工合同

关键字段：

- `合同编号`
- `加工方名称`
- `委托单位`
- `下单日期`
- `回货日期`
- `产品订单编号`
- `产品编号`
- `产品名称`
- `工序名称`
- `加工厂商`
- `工序类别`
- `单价`
- `委托加工数量`
- `委托加工金额`

注意：

- `9.3加工合同-子淳.pdf` 证明这类单据不只是 Excel 台账，而是正式合同快照
- 条款区里的延期处罚、返工责任、次月对账都是财务和生产流程节点真源
- 工作台顶部会显示 `当前明细行数 / 300`，加工合同明细当前最多支持 300 行
- 右侧加工合同纸面只渲染真实明细行，不再额外补底部占位空白行；如果合同需要空白行，需通过工作台显式插入
- 工作台上插 / 下插明细行时会新增真正空白行，不再预填合同号、产品号或其他相邻行字段
- 工作台当前支持“选择单元格 / 合并选区 / 拆分当前”；金额列允许直接改写，也可以参与合并展示
- 纸样 / 图样附件当前通过工作台左侧上传条独立上传，并同步进入右侧页底附件位，随 PDF / 打印一起输出
- 工作台壳层、顶部按钮、左右双栏编辑布局，以及 `DOM 快照 -> 后端 render-pdf -> 独立预览窗口 / 下载` 输出链路都已收口到当前固定打印模板主路径

## 当前为什么只保留这两套

1. 合同模板已经拿到明确真源文件和实拍 / PDF 对照样本，字段口径相对稳定
2. 材料 / 加工汇总和生产报表目前更适合作为资料真源或分析页参考，不再继续占用正式打印主路径
3. 先把正式合同的编辑、PDF 与打印链路做稳，比同时维护多套静态预览更可控
