# 模板打印与字段口径

## 当前目标

本轮继续参照 `trade-erp` 的固定模板工作流，但主路径只保留两套正式合同模板：

- `采购合同`
- `加工合同`

当前实现边界：

- 已做：统一打印中心、左右双栏编辑壳页、PDF 预览 / 下载、浏览器打印、模板口径文档、帮助中心入口
- 未做：PDF 坐标回填、Excel 母版回写、业务保存链路自动带值、图片识别 / 扩展硬件链路 / 盖章自动识别

## 当前已落模板

| 模板     | 真源文件                                                             | 当前用途                                  | 字段性质                           |
| -------- | -------------------------------------------------------------------- | ----------------------------------------- | ---------------------------------- |
| 采购合同 | `模板-材料与加工合同.xlsx` 的 `C类辅料合同`                          | 辅材 / 包材采购下单、供应商确认、留档打印 | 合同头和明细都是业务快照           |
| 加工合同 | `模板-材料与加工合同.xlsx` 的 `B类加工合同` + `9.3加工合同-子淳.pdf` | 委外加工下单、回签、财务对账留档          | 合同头、合同行、条款都属于合同快照 |

## 模板分类规则

当前打印中心只保留合同类模板。

规则：

- 合同编号、下单日期、回货日期、对方名称、联系人、电话、地址都按打印当下保存为快照
- 合同条款是正式业务口径的一部分，不能只写在帮助中心里，不出现在模板里
- 合同金额、单价、数量后续即使主档调整，也不能反写历史合同打印件

## 首批固定字段口径

### 采购合同

关键字段：

- `采购订单号`
- `产品订单编号`
- `产品编号`
- `产品名称`
- `材料品名`
- `厂商料号`
- `规格`
- `单位`
- `单价`
- `采购数量`
- `采购金额`
- `备注`

注意：

- `供应商名称 / 联系人 / 联系电话 / 供应商地址` 当前来自 `材料厂商编号` sheet
- `订货人 / 联系电话` 当前来自 `原辅料采购汇总表`
- `厂商料号` 列在真实采购样本里经常承载供应商简称；打印页先保留原列名，帮助中心再提醒导入清洗
- 采购合同纸面严格按实拍样本收口：只有中间采购明细区保留表格，其余合同头、条款和签字区都按普通排版输出
- 签字区默认留给纸质手写签名，不带电子印章

### 加工合同

关键字段：

- `合同编号`
- `加工方名称`
- `委托单位`
- `下单日期`
- `回货日期`
- `产品订单编号`
- `产品编号`
- `产品名称`
- `工序名称`
- `加工厂商`
- `工序类别`
- `单价`
- `委托加工数量`
- `委托加工金额`

注意：

- `9.3加工合同-子淳.pdf` 证明这类单据不只是 Excel 台账，而是正式合同快照
- 条款区里的延期处罚、返工责任、次月对账都是财务和生产流程节点真源
- 纸样 / 图样附件当前通过工作台单独上传，不进入右侧合同模板或 PDF
- 工作台壳层、顶部按钮和左右双栏编辑布局已对齐 `trade-erp` 打印窗口

## 当前为什么只保留这两套

1. 合同模板已经拿到明确真源文件和实拍 / PDF 对照样本，字段口径相对稳定
2. 材料 / 加工汇总和生产报表目前更适合作为资料真源或分析页参考，不再继续占用正式打印主路径
3. 先把正式合同的编辑、PDF 与打印链路做稳，比同时维护多套静态预览更可控

## 下一步

1. 接真实业务带值前，先继续验证合同头、合同行、条款和签字区的字段真源
2. 再决定是否把合同字段抽成正式后端打印 DTO
3. 最后才评估 PDF 生成、Excel 母版回写和打印留档
