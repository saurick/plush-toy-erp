# 任务 / 单据映射表

> 适用对象：老板、业务、PMC、生产经理、采购、仓库、品质、财务、系统管理员
> 当前用途：把“任务从哪来、由谁处理、由哪张单据或哪个节点驱动完成”收口到一页，供继续做工作台卡片、手机端动作和后端状态迁移边界时对线。
> 当前约束：任务是协同层，不是业务真源；`workflow_tasks` 已作为协同表落地，但本页不等于自动派单规则或万能任务引擎。

## 1. 先把任务理解正确

1. 当前业务真源仍是立项、BOM、采购、加工合同、入库、排单、进度、异常、出货、对账这些业务单据与节点。
2. 任务只负责回答四件事：`谁该做`、`什么时候做`、`现在卡在哪`、`做完后回给谁`。
3. 任务页、任务卡片、手机端快捷动作，都应由业务单据或业务节点派生，不单独发明第二套业务事实。
4. 当前可以先把任务用于工作台、手机端和提醒视图，但不要因此替代正式页面入口和单据详情页。
5. 当前数据库结构以 Ent schema 和 Atlas migration 为准；本页只定义任务来源、责任人和完成条件，不直接扩成自动派单规则。

## 2. 当前推荐任务状态口径

| 状态         | 当前含义                         | 典型触发条件                                     | 当前展示位置                 |
| ------------ | -------------------------------- | ------------------------------------------------ | ---------------------------- |
| `pending`    | 已创建，但前置条件还没齐         | 立项已发起，但审批、资料、到料或回签还没到位     | 桌面看板、手机端待跟进       |
| `ready`      | 前置条件已满足，可以开始执行     | 单据、资料、到料、齐套或上游确认已经满足         | 桌面看板、手机端待处理       |
| `processing` | 已有人接手，正在处理中           | 已进入排单、采购跟进、检验、返工或对账           | 桌面看板、详情页状态摘要     |
| `blocked`    | 当前被外部条件卡住，不能继续推进 | 缺资料、缺料、合同未回签、品质未放行、财务未放行 | PMC 视角、异常视角、催办视角 |
| `done`       | 当前任务的完成条件已经满足       | 对应业务单据或节点已达到完成口径                 | 看板历史、详情页时间轴       |
| `rejected`   | 审批或确认类动作被退回           | 老板未放行、品质退回、财务未放行                 | 审批链、异常链、退回链       |
| `cancelled`  | 任务因上游变化被取消             | 订单取消、委外转内做、需求撤销                   | 历史记录、时间轴             |
| `closed`     | 已完成并可归档                   | 主链闭环后统一关闭                               | 历史记录、归档视图           |

当前补充展示态：

- `due_soon`：即将超时，建议只做前端或聚合层计算，不先当正式库字段。
- `overdue`：已超时，建议只做前端或聚合层计算，不先当正式库字段。

## 3. 一级任务分组

| 任务组                    | 当前定位                               | 是否每单都必然出现       | 当前主要处理角色    |
| ------------------------- | -------------------------------------- | ------------------------ | ------------------- |
| T1 立项 / 审批            | 把客户、编号、交期和关键审批先收口     | 是                       | 业务、老板          |
| T2 资料 / BOM             | 把 BOM、色卡、作业指导书和资料缺口补齐 | 是                       | 业务、采购          |
| T3 材料 / 辅包材采购      | 主料、辅材、包材的下单、到料和缺料反馈 | 大多会出现               | 采购、PMC           |
| T4 加工合同 / 委外        | 外发合同、回签、回货和委外进度         | 仅在存在委外时出现       | 采购、PMC、财务     |
| T5 排单 / 生产执行        | 齐套判断、排单、完工、延期和进度反馈   | 仅在进入生产后出现       | PMC、生产经理       |
| T6 品质 / 返工            | IQC、过程检验、返工复检和异常闭环      | 仅在需要检验或返工时出现 | 品质、生产经理、PMC |
| T7 入库 / 待出货 / 出库   | 入库、待出货确认、财务放行和出库执行   | 仅在已可交付时出现       | 仓库、业务、财务    |
| T8 对账 / 结算 / 付款提醒 | 对账、待结算、待付款和异常费用提醒     | 仅在形成应结应付后出现   | 财务、老板          |

当前明确不采用的做法：

1. 不要求每个订单一创建就机械生成全部 8 个一级任务。
2. 不把“可能发生的任务”提前落成大量无意义的空任务。
3. 更稳的是按业务阶段和前置条件逐步派生。

## 5. 当前角色任务池建议

| 角色     | 当前更适合先看什么任务           | 当前不应承接什么                       |
| -------- | -------------------------------- | -------------------------------------- |
| 老板     | `T1-2`、高风险 `T5`、高风险 `T8` | 不直接处理 BOM、采购、仓库执行明细     |
| 业务     | `T1`、`T2`、`T7-2`               | 不直接做排单、对账、仓库出库执行       |
| PMC      | `T3`、`T4`、`T5`、异常态 `T6/T7` | 不替老板审批、不替财务放行             |
| 生产经理 | `T5-2`、`T5-3`、`T6-2`           | 不接客户前置资料和财务结算             |
| 采购     | `T2-2`、`T3`、`T4`               | 不接生产现场决策和财务放行             |
| 仓库     | `T4-2`、`T6-1`、`T7`             | 不独立决定品质结论和财务放行           |
| 品质     | `T6`、质量相关 `T7`              | 不维护客户、合同和财务主线             |
| 财务     | `T7-2`、`T8`                     | 不替仓库录入出库、不替采购维护合同执行 |

## 6. 手机端当前更适合做什么任务动作

| 角色     | 当前手机端推荐动作                 | 当前手机端不做          |
| -------- | ---------------------------------- | ----------------------- |
| 老板     | 审批、加急、风险确认               | 复杂录入、明细维护      |
| 业务     | 缺资料回传、客户确认、改期反馈     | 复杂 BOM 维护、合同编辑 |
| PMC      | 催办、分发异常、确认是否可排       | 全量分析、复杂排产维护  |
| 生产经理 | 今日排产、进度回填、异常上报       | 大批量主数据维护        |
| 采购     | 缺料 / 到料 / 回签反馈             | 复杂采购模板维护        |
| 仓库     | 收货、回仓、入库、待出货反馈       | 完整库存主档维护        |
| 品质     | IQC、复检、退回 / 放行反馈         | 独立主数据维护          |
| 财务     | 放行反馈、待付款提醒、异常费用反馈 | 全量对账明细维护        |

## 7. 当前明确不按这页直接做的事

1. 不把任务状态当成独立于业务单据之外的唯一事实。
2. 不因为有任务池，就撤掉原有业务页和单据详情入口。
3. 不直接按这页扩正式 API 全字段或自动派单引擎；要先回到 `Workflow / Schema 草案`、Ent schema 和当前真源文档继续校对。
4. 不把 `blocked`、`overdue`、`due_soon` 先硬编码成跨系统统一底层真源。
5. 不在当前阶段补一套“万能任务中心”替代真实业务主链。
