# 运行记录

> 目的：告诉研发、测试和验收同事当前最近应该跑什么命令、这些命令覆盖什么，以及哪些结果还没有被结构化沉淀。

## 1. 当前结论

- 当前还没有通用持久化运行记录页；字段联动专项已先生成 `public/qa/erp-field-linkage-coverage.latest.json` 摘要。
- 当前运行记录仍以本地命令输出、Playwright 截图、测试报告、字段联动 latest JSON 和终端结果为准。
- 这页先统一命令入口，后续再把其他专项接成结构化摘要。

## 2. 当前命令分层

| 命令                                    | 覆盖范围                                 | 建议时机                                   |
| --------------------------------------- | ---------------------------------------- | ------------------------------------------ |
| `pnpm lint`                             | 前端 ESLint 自动修复和基础语法           | 前端改动后                                 |
| `pnpm css`                              | 样式规则检查                             | CSS 或样式类改动后                         |
| `pnpm test`                             | 前端单元测试和配置测试                   | 前端逻辑、文档配置、权限配置改动后         |
| `node scripts/qa/erp-field-linkage.mjs` | 字段联动专项测试并刷新 latest 报告       | 改字段真源、保存转换、合同金额、打印快照后 |
| `pnpm style:l1`                         | 浏览器级页面、表单、菜单、打印和文档回归 | 页面、导航、布局、打印改动后               |
| `scripts/qa/fast.sh`                    | 高频前后端快速检查                       | 日常开发                                   |
| `scripts/qa/full.sh`                    | 更完整的提交前检查                       | 提交前或推送前                             |
| `scripts/qa/strict.sh`                  | 严格检查                                 | 发版前                                     |

## 3. 推荐记录哪些信息

每次验收至少保留：

- 执行命令。
- 命中分支和工作区状态。
- 运行环境和关键端口。
- 通过、失败或跳过结论。
- 若失败，保留第一处失败原因和截图或日志路径。
- 若未跑某项，说明未跑原因和剩余风险。
