# ERP 字段联动口径

> 目的：统一项目内对“字段真源、单据快照、派生结果、打印回显”这四类信息的理解，避免残值和缺值在页面、打印、导出、搜索或历史记录里漂移。
>
> 这页优先回答两个问题：
>
> - 问题 1：改上游真源后，后面已经出现的页面、合同、汇总表会不会一起变？
> - 问题 2：改下游执行记录后，会不会把前面的真源字段静默改回去？

## 1. 先看结论

- 当前正式口径统一按“真源字段单独维护 + 历史单据保留快照 + 汇总 / 打印按既定规则派生”理解。
- 一句话速记：`主数据不乱回写，历史快照不静默改，派生结果按规则重算。`
- 已经生成好的合同、汇总表、报表快照，不因为上游主档调整就批量静默重写。
- 同一字段若同时出现在表单带值、保存前处理、打印映射、导入映射和导出映射，优先收口到共享映射规则，不在每层各补一个条件分支。
- 涉及编号、金额、日期、附件这类跨模块字段时，必须同时防两类问题：
  - `残值`：来源已经变了，但旧值还留在下游页面、打印、导出或统计里。
  - `缺值`：真源其实已经有值，但快照、回显、打印或历史记录没带出来。

## 2. 先分清 5 类字段

| 类型              | 典型字段                                                             | 当前口径                                                 |
| ----------------- | -------------------------------------------------------------------- | -------------------------------------------------------- |
| 主档 / 主数据候选 | 客户、加工厂、辅包材供应商、款式编号、物料基础信息                   | 作为长期复用字段理解，但当前仍有部分编码体系未完全稳定   |
| 业务快照字段      | 订单编号、产品订单编号、产品编号、产品名称、颜色、回货日期、出货日期 | 出现在订单、合同、台账、报表里时优先按当时快照理解       |
| 派生 / 汇总字段   | 材料分析汇总表采购数量、加工金额、待出货数、待付款提醒               | 由上游真源或 Excel 公式派生，不反写覆盖原始真源          |
| 打印快照字段      | 合同头、合同行、条款、地址、联系人、图样附件                         | 进入正式模板后按打印当下保存，不跟随后续主档变更自动重刷 |
| 附件 / 资料层字段 | 色卡、作业指导书、合同附图、Excel 内嵌图片                           | 应作为独立附件或资料层管理，不塞进主表万能备注           |

## 3. 当前最关键的字段边界

- 客户 / 订单编号
- 产品订单编号 / 客户订单号
- 款式编号
- 产品编号 / SKU

这四层编号当前不能再混成一个字段。

补充理解：

- `款式编号` 当前最像 style 级主档编号，真源主要来自 `材料分析明细表`。
- `产品编号 / SKU` 当前主要来自加工合同 PDF、生产订单截图和业务快照。
- `产品订单编号 / 客户订单号 / 订单编号` 当前命名仍有混用，必须带着原件一起核，不允许拍脑袋归并。

## 4. 高频问题速查

- 改客户 / 款式立项里的编号：不会自动把历史加工合同、历史汇总表、历史打印件全部重写；以后新生成的记录应按新真源带值。
- 改材料 BOM 明细：不会反向把已打印的采购合同、加工合同、历史汇总表静默改掉；汇总派生层需要按共享映射规则重新取值。
- 改辅材 / 包材采购里的 `厂商料号`：不能默认它就是正式物料料号，因为真实样本里它常常承载供应商简称。
- 改加工合同联系人 / 地址 / 条款：这是合同快照，不应倒写供应商主档。
- 改生产、仓库、结算结果：会影响当前待处理、待出货、待付款等派生结果，但不会反写前面的原始订单 / BOM / 合同字段。
- 真源已经有值但页面没显示：先查该页读的是主档、业务快照还是打印快照，不要先补一个临时 fallback。

## 5. 改上游后，旧下游会怎样

### 5.1 核心业务字段

| 字段                | 当前真源                                 | 会出现在哪里                                    | 改上游后，旧下游会怎样                                       | 当前提醒                                 |
| ------------------- | ---------------------------------------- | ----------------------------------------------- | ------------------------------------------------------------ | ---------------------------------------- |
| 客户                | 生产订单总表截图、客户主档候选           | 客户/款式立项、加工合同、生产订单总表、结算对象 | 已生成的合同和报表继续保留历史快照；以后新记录才按新真源带值 | 不把客户简称、加工方简称、供应商简称混用 |
| 订单编号            | 生产订单总表截图、材料 Excel             | 立项、BOM 头、生产台账                          | 历史记录不静默改；新记录按当前真源带值                       | 仍需继续核与产品订单编号的层级关系       |
| 产品订单编号        | 材料 Excel、加工合同 PDF、加工汇总表     | 加工合同、辅包材采购、导入映射                  | 已生成合同行保留快照，不自动批量重写                         | 当前最容易和订单编号、客户订单号混用     |
| 款式编号            | `材料分析明细表` 中的 `26029# / 26204#`  | 产品页、BOM、后续 style 候选主档                | 已生成订单、合同中的产品编号不会因为 style 变更一起改        | `款式编号 != 产品编号 / SKU`             |
| 产品编号 / SKU      | 加工合同 PDF、生产订单截图               | 合同、生产单、采购 / 仓库快照                   | 历史合同和生产快照保留原值                                   | 不能回写覆盖款式编号                     |
| 产品名称 / 颜色     | 材料 Excel、合同 PDF、生产订单截图       | 订单、合同、BOM、仓库快照                       | 历史合同 / 报表保留快照；新记录按当前真源带值                | 当前仍存在整款名、颜色款名、部件名三层   |
| 主料物料字段        | `材料分析明细表`                         | BOM、材料分析汇总表、后续主料采购派生           | 汇总表需按共享规则重新派生，不反写 BOM 真源                  | `材料分析明细表` 才是 BOM 真源           |
| 辅材 / 包材字段     | `辅材、包材 成慧怡.xlsx`                 | 辅材采购、包装材料打单、结算快照                | 历史采购快照保留原值                                         | 不能回写主料 BOM                         |
| 加工方联系人 / 地址 | 加工合同 PDF、加工厂商资料               | 合同头、partner 主档候选                        | 历史合同保留打印快照，不自动覆盖主档                         | 合同快照和 partner 主档必须分层          |
| 图片 / 附件         | 合同附图、作业指导书、色卡、Excel 内嵌图 | 合同、资料页、后续 attachment                   | 旧合同与旧资料继续保留原附件引用                             | 不塞进万能备注字段                       |

### 5.2 日期与条款字段

| 字段            | 当前真源         | 当前口径                                           |
| --------------- | ---------------- | -------------------------------------------------- |
| 回货日期        | 加工合同 PDF     | 属于加工合同承诺回货日期，不等于客户出货日期       |
| 出货日期        | 生产订单总表截图 | 属于订单 / 发货快照，不反写加工合同回货日期        |
| 下单日期        | 合同或采购快照   | 属于单据生成时快照，后续不跟主档同步改写           |
| 结算方式 / 条款 | 加工合同 PDF     | 属于正式合同口径，不能只写在帮助中心里代替合同原文 |

## 6. 改下游后，会不会反写上游

| 下游动作                         | 上游原始字段会不会改            | 哪些结果会变化                           | 说明                       |
| -------------------------------- | ------------------------------- | ---------------------------------------- | -------------------------- |
| 改辅材 / 包材采购单价、数量      | 不会回写主料 BOM 或订单原始字段 | 当前采购金额、采购汇总、待付款提醒会变化 | 采购快照只改采购层         |
| 改加工合同数量、单价、工序       | 不会回写订单、款式或物料真源    | 合同金额、结算快照、回签留档会变化       | 合同行是合同自己的快照层   |
| 改生产排单 / 返工 / 异常日志     | 不会回写立项或 BOM 原值         | 当前进度、延期、异常统计会变化           | 进度结果属于执行层         |
| 改仓库收发 / 成品入库 / 出货结果 | 不会回写订单原始数量            | 当前待出货、库存相关派生结果会变化       | 仓库结果是执行层快照       |
| 改对账 / 付款状态                | 不会回写合同原始金额或条款      | 待付款、异常费用、财务提醒会变化         | 财务状态属于结算层派生结果 |

## 7. 残值与缺值怎么防

### 7.1 先问 4 个问题

1. 当前唯一真源字段是谁？
2. 库里或页面里是不是还留着快照字段？
3. 打印、导出、搜索、统计是不是读了派生字段？
4. 历史旧数据缺值时，是否有明确回补路径？

### 7.2 本项目里的高风险场景

| 场景                | 容易出什么问题                             | 当前处理原则                                   |
| ------------------- | ------------------------------------------ | ---------------------------------------------- |
| 订单编号体系切换    | 旧订单编号残留在合同、导入映射、搜索条件里 | 先收口共享字段映射，再决定哪些快照需要同步重建 |
| 供应商 / 加工商切换 | 旧联系人、旧地址继续留在打印件或详情页     | 主档切换后要明确清空还是保留历史合同快照       |
| BOM 明细变更        | 汇总表或采购派生层继续显示旧值             | 派生层重新取值，不在多个页面各补一个条件       |
| 附件替换            | 页面显示了新附件，但历史合同仍指向旧附件   | 合同附件按历史快照保留，资料页按当前真源展示   |
| 旧数据缺字段        | 页面空白后被临时 fallback 成错误值         | 优先定义回补规则，不伪造值                     |

## 8. 导入映射口径

- `材料分析明细表` -> BOM 真源
- `材料分析汇总表-修改` -> 采购汇总派生
- `委外加工汇总表` -> 加工合同行导入候选
- `辅材、包材` -> 独立辅材 / 包材采购导入源
- `9.3加工合同-子淳.pdf` -> 合同头、合同行、条款、附件图样

## 9. 页面入口建议

| 场景                       | 优先入口                            | 说明                                                 |
| -------------------------- | ----------------------------------- | ---------------------------------------------------- |
| 想确认字段会不会互相影响   | `ERP 字段联动口径`                  | 先看“改上游后旧下游会怎样”和“改下游后会不会反写上游” |
| 想确认金额、数量、日期口径 | `ERP 计算口径`                      | 这类问题优先按计算语义理解，不要混到字段联动里       |
| 想确认正式流程上下文       | `ERP 流程图总览`                    | 先看阶段、责任角色和资料来源                         |
| 想确认模板字段为何这样显示 | `模板打印中心` + `ERP 字段联动口径` | 先分清打印快照和主档真源                             |
