# ERP 操作教程（入口与使用方式）

> 适用对象：老板、跨岗协作人员、新同事、带教人员  
> 当前用途：这一页只解释“总后台、角色后台、手机端、帮助中心怎么理解”，不替代具体角色流程页。

## 1. 先把系统理解成“一套总后台 + 多个角色切片”

| 入口     | 当前应该怎么理解                           | 主要承载什么                                 |
| -------- | ------------------------------------------ | -------------------------------------------- |
| 总后台   | 老板视角的总后台，也是当前唯一完整后台入口 | 全链路总览、帮助中心、打印、异常和关键业务页 |
| 角色后台 | 从总后台剪裁出来的角色视图                 | 只保留该角色真正需要的页面、卡片和待办       |
| 手机端   | 任务端，不是复杂录入端                     | 任务分配、任务处理、状态回填、异常反馈       |

## 2. 当前帮助中心怎么读

建议阅读顺序：

1. `ERP 流程图总览`
2. `角色协同链路`
3. `桌面端角色流程`
4. `手机端角色流程`
5. `ERP 字段联动口径`
6. `ERP 计算口径`
7. `打印 / 合同 / 快照口径`
8. `异常 / 返工 / 延期处理`
9. `当前明确不做`

## 3. 遇到不同问题先看哪一页

| 你现在想确认什么               | 先看哪一页               | 为什么                             |
| ------------------------------ | ------------------------ | ---------------------------------- |
| 主流程到底怎么走               | `ERP 流程图总览`         | 先看主链路和支线，不先陷进字段细节 |
| 谁把任务交给谁                 | `角色协同链路`           | 直接看角色交接、触发、反馈和回退   |
| 桌面端 4 个角色分别干什么      | `桌面端角色流程`         | 单独看老板、业务、PMC、生产经理    |
| 手机端各角色做什么             | `手机端角色流程`         | 统一看任务分配、处理、反馈         |
| 一个字段改了会影响哪里         | `ERP 字段联动口径`       | 先区分真源、快照、派生             |
| 为什么数量 / 金额 / 日期对不上 | `ERP 计算口径`           | 先核字段语义和计算口径             |
| 合同、打印、历史快照怎么看     | `打印 / 合同 / 快照口径` | 合同头、合同行和条款都按快照处理   |
| 异常谁处理、怎么关闭           | `异常 / 返工 / 延期处理` | 看谁发起、谁处理、退回哪一层       |
| 某能力为什么还没做             | `当前明确不做`           | 先看 deferred 边界，避免脑补       |

## 4. 角色后台和帮助中心的关系

1. 帮助中心是当前正式业务口径入口，角色后台不再各写一套独立说明。
2. 老板总后台可以看到完整帮助中心目录。
3. 角色后台默认只展示与本角色直接相关的帮助中心切片，不复制文案。
4. 页面内的“查看说明”统一跳回帮助中心对应页面或对应锚点。

## 5. 手机端的默认定位

1. 手机端先按“任务分配 -> 任务处理 -> 处理反馈”理解。
2. 手机端优先做催办、确认、回填、异常上报，不抢跑复杂主数据维护。
3. 手机端不是打印端，不在手机端编辑正式模板。
4. 手机端不是大表录入端，不在手机端维护完整 BOM、合同、采购模板和复杂财务表。

## 6. 仓库和工序主链先怎么理解

1. 当前仓库至少按 `主料仓`、`其他仓`、`成品仓` 三层理解。
2. 所有主料、辅料、包装材料、半成品、成品、返工成品都必须先入仓或从仓库领出，不走“跳过仓库”的口头链路。
3. 外发给其他商家代工的部分也先走仓库出库，再走外发回仓。
4. 当前工序主链先按这条线理解：

`主料仓 -> 激光 / 机器加工 / 裁切 -> 车缝领裁片 -> 其他仓 -> 手工领主料/辅料 -> 缝合 / 组装 / 包装 -> 成品仓`

5. 返工成品也继续回仓，再按返工任务重新领出，不单独绕开仓库处理。

## 7. 当前这批角色流程页怎么用

| 页面             | 当前作用                          | 校对重点                               |
| ---------------- | --------------------------------- | -------------------------------------- |
| `桌面端角色流程` | 用于和老板校对桌面端 4 类角色分工 | 看主链路是否缺角色、子链路是否挂错人   |
| `手机端角色流程` | 用于和老板校对手机端角色切片      | 看任务分配、任务处理、处理反馈是否对线 |
| `角色协同链路`   | 用于校对角色交接关系              | 看触发点、交付物、回退链是否真实       |

## 8. 当前明确要避免的误解

1. 不把 `trade-erp` 的报价、外销、出运、结汇语义直接搬过来套当前毛绒工厂流程。
2. 不把角色后台理解成第二套独立系统；它只是总后台的剪裁视图。
3. 不把手机端理解成“什么都能做”的全功能端；当前先收口任务协同。
4. 不把帮助中心当聊天补丁；页面、角色、字段和打印口径都以帮助中心为正式入口。
