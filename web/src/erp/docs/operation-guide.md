# ERP 操作教程（入口与使用方式）

> 适用对象：老板、跨岗协作人员、新同事、带教人员  
> 当前用途：这一页只解释“总后台、角色后台、手机端、帮助中心怎么理解”，不替代具体角色流程页。

## 1. 先把系统理解成“一套总后台 + 多个角色切片”

| 入口     | 当前应该怎么理解                           | 主要承载什么                                 |
| -------- | ------------------------------------------ | -------------------------------------------- |
| 总后台   | 老板视角的总后台，也是当前唯一完整后台入口 | 全链路总览、帮助中心、打印、异常和关键业务页 |
| 角色后台 | 从总后台剪裁出来的角色视图                 | 只保留该角色真正需要的页面、卡片和待办       |
| 手机端   | 任务端，不是复杂录入端                     | 任务分配、任务处理、状态回填、异常反馈       |

## 2. 当前业务页已经能写什么

1. 首次进入默认到 `任务看板`，先看待处理、处理中、阻塞、退回、超时和即将到期任务；要看各业务模块记录数、状态分布、部门待处理和风险预警，再进入 `业务看板`。
2. 图里的销售链路、采购 / 仓储、生产环节和财务环节菜单已作为主业务菜单保留，报价单仍处于待确认状态，不进入当前菜单和路由。
3. 每个已启用业务页都是表格 + 弹窗录入形态，保存到 `business_records`，通用明细保存到 `business_record_items`；明细金额为空且有数量 / 单价时按行派生，表头数量 / 金额为空时按明细合计回写。
4. 业务页统一按“顶部说明卡片 -> 筛选卡片 -> 列表工具栏 -> 选中操作条 -> 表格 -> 协同任务池”展示；筛选卡片只放关键字、业务状态、模块日期字段和排序筛选，日期类型与起止日期在同一个日期范围控件内完成。
5. 列表工具栏只保留总记录、当前结果、已选数量、金额 / 数量合计以及导出、列顺序、回收站和新建记录；“新建记录”是主操作，导出、列顺序和回收站是次操作。
6. 未选中记录时不会展示一整排禁用业务按钮；选中记录后才显示编辑、打印模板、关联表格、创建协同任务、流转、删除和批量删除等操作。
7. 工具栏“列顺序”或表头菜单可调整列位置，列顺序会跟随当前管理员账号保存，浏览器本地只作为同步失败时的兜底，导出当前结果会沿用当前筛选结果和列顺序。
8. 选中单条记录后可点“关联表格”跳转上下游业务页；跳转会把当前单据号或来源单号带入目标页搜索框，并优先按 `document_no` / `source_no` 白名单过滤，避免备注或其他字段同号造成误命中。
9. 业务页保存和“流转业务状态”都会同步写入 `workflow_business_states`，用于记录单据来源、业务状态、主责角色和状态快照；状态流转会带完整业务记录字段保存，避免只传状态导致其他字段被清空。
10. 流转到 `业务阻塞` 或 `业务取消` 时必须填写原因；原因会写回业务记录 payload、业务状态快照和关联未终止任务，任务池会同步显示阻塞 / 取消说明。
11. 业务页可从选中记录创建 `workflow_tasks`，任务状态变化写入 `workflow_task_events`；手机端按角色读取这些任务并回填处理、阻塞和完成状态。
12. `business_records` 是当前首版通用业务记录真源；客户、BOM、采购、仓库、生产、财务等更细的专表仍按真实样本逐步拆，不用任务表替代业务单据。

| 使用者        | 建议先看                                             | 什么时候再往下看                                                               |

| 普通业务用户  | 帮助中心首页的“新手先看”“按角色找工作”“按业务主线查” | 遇到异常、打印字段、手机端任务不可见时，再看 FAQ 或相关正式文档                |

## 4. 遇到不同问题先看哪一页

| 你现在想确认什么               | 先看哪一页                   | 为什么                                               |
| ------------------------------ | ---------------------------- | ---------------------------------------------------- |
| 主流程到底怎么走               | `ERP 流程图总览`             | 先看主链路和支线，不先陷进字段细节                   |
| 谁把任务交给谁                 | `角色协同链路`               | 直接看角色交接、触发、反馈和回退                     |
| 角色能看哪些页面、单据怎么串   | `角色权限 / 页面 / 单据矩阵` | 一页先收口角色边界、页面入口和主链                   |
| 任务从哪来、由什么完成         | `任务 / 单据映射表`          | 先分清任务是协同层，单据才是真源                     |
| 任务和业务状态应该怎么统一     | `任务 / 业务状态字典`        | 先统一状态，再继续做页面和保存链路                   |
| SQL / schema 应该怎么看        | `Workflow / Schema 草案`     | 看已落地 Ent / Atlas 表、JSON-RPC 接口和后续拆表边界 |
| 桌面端 4 个角色分别干什么      | `桌面端角色流程`             | 单独看老板、业务、PMC、生产经理                      |
| 手机端各角色做什么             | `手机端角色流程`             | 统一看任务分配、处理、反馈                           |
| 一个字段改了会影响哪里         | `ERP 字段联动口径`           | 先区分真源、快照、派生                               |
| 为什么数量 / 金额 / 日期对不上 | `ERP 计算口径`               | 先核字段语义和计算口径                               |
| 合同、打印、历史快照怎么看     | `打印 / 合同 / 快照口径`     | 合同头、合同行和条款都按快照处理                     |
| 异常谁处理、怎么关闭           | `异常 / 返工 / 延期处理`     | 看谁发起、谁处理、退回哪一层                         |
| 某能力为什么还没做             | `当前明确不做`               | 先看 deferred 边界，避免脑补                         |

## 5. 角色后台和帮助中心的关系

1. 帮助中心是当前正式业务口径入口，角色后台不再各写一套独立说明。
2. 老板总后台可以看到完整帮助中心目录。
3. 角色后台默认只展示与本角色直接相关的帮助中心切片，不复制文案。
4. 页面内的“查看说明”统一跳回帮助中心对应页面或对应锚点。

## 6. 手机端的默认定位

1. 手机端先按“任务分配 -> 任务处理 -> 处理反馈”理解。
2. 手机端只保留任务页和退出登录；不展示说明页、角色文案、端口说明、技术字段或帮助文案。
3. 手机端任务页读取真实 `workflow_tasks`，同时展示任务、预警、通知和进度；预警由阻塞、退回、业务阻塞、临近 / 已超时和高优先级任务派生，通知由最近更新任务派生，进度由任务状态汇总派生。
4. 手机端当前支持处理、阻塞、完成；阻塞前必须填写原因，处理或完成会清空当前任务阻塞说明。
5. 手机端复用管理员登录态，登录页提供密码登录和短信登录；账号必须先在 `权限管理` 勾选对应移动端角色权限才能进入任务页，短信登录还要求手机号已绑定到管理员账号。
6. 手机端不是打印端，不在手机端编辑正式模板。
7. 手机端不是大表录入端，不在手机端维护完整 BOM、合同、采购模板和复杂财务表。

## 7. 仓库和工序主链先怎么理解

1. 当前仓库至少按 `主料仓`、`其他仓`、`成品仓` 三层理解。
2. 所有主料、辅料、包装材料、半成品、成品、返工成品都必须先入仓或从仓库领出，不走“跳过仓库”的口头链路。
3. 外发给其他商家代工的部分也先走仓库出库，再走外发回仓。
4. 当前工序主链先按这条线理解：

`主料仓 -> 激光 / 机器加工 / 裁切 -> 车缝领裁片 -> 其他仓 -> 手工领主料/辅料 -> 缝合 / 组装 / 包装 -> 成品仓`

5. 返工成品也继续回仓，再按返工任务重新领出，不单独绕开仓库处理。

## 8. 当前这批角色流程页怎么用

| 页面                         | 当前作用                             | 校对重点                               |
| ---------------------------- | ------------------------------------ | -------------------------------------- |
| `角色权限 / 页面 / 单据矩阵` | 用于先定角色边界、页面范围和主链单据 | 看角色剪裁、正式入口和单据链是否对线   |
| `任务 / 单据映射表`          | 用于先定任务来源、责任人和完成条件   | 看任务是否从单据派生、是否多造一层状态 |
| `任务 / 业务状态字典`        | 用于统一任务状态、业务状态和推进阶段 | 看状态名、阻塞口径和阶段定义是否一致   |
| `Workflow / Schema 草案`     | 用于起草任务协同层和业务状态层字段   | 看表结构草案、SQL 样例和落地顺序       |
| `桌面端角色流程`             | 用于和老板校对桌面端 4 类角色分工    | 看主链路是否缺角色、子链路是否挂错人   |
| `手机端角色流程`             | 用于和老板校对手机端角色切片         | 看任务分配、任务处理、处理反馈是否对线 |
| `角色协同链路`               | 用于校对角色交接关系                 | 看触发点、交付物、回退链是否真实       |

## 9. 当前明确要避免的误解

1. 不把报价、外销、出运、结汇等旧外贸语义直接搬过来套当前毛绒工厂流程。
2. 不把角色后台理解成第二套独立系统；它只是总后台的剪裁视图。
3. 不把手机端理解成“什么都能做”的全功能端；当前先收口任务协同。
4. 不把帮助中心当聊天补丁；页面、角色、字段和打印口径都以帮助中心为正式入口。
