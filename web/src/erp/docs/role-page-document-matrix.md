# 角色权限 / 页面 / 单据矩阵

> 适用对象：老板、业务、PMC、生产经理、采购、仓库、品质、财务、系统管理员
> 当前用途：把“角色该看什么、当前有哪些正式页面、主链路单据怎么串”收口到一份总表，供后续继续做菜单剪裁、默认页和细分业务表时对线。
> 当前约束：这里写的是当前仓库的正式业务口径，不是未来完整版 ERP 蓝图；通用业务记录和 workflow 已落地，细分业务专表和完整后端角色鉴权仍以后续主实现为准。

## 1. 使用方式

1. 先用这页确认角色边界、页面范围和单据主链是否挂对。
2. 再打开 `任务 / 单据映射表`，确认任务是由哪张单据或哪个节点派生出来的。
3. 再打开 `任务 / 业务状态字典` 和 `Workflow / Schema 草案`，统一状态、通用业务记录表和后续拆表边界。
4. 再回到 `桌面端角色流程`、`手机端角色流程` 和 `角色协同链路` 看具体交接细节。
5. 若要继续拆细分业务专表，先按这里的页面和单据顺序推进，不要绕过当前通用业务记录层。

## 2. 当前角色权限矩阵

### 2.1 使用说明

1. 当前 `权限管理` 正式控制后台菜单可见范围、管理员绑定手机号和角色移动端登录权限；不等于后端业务接口已经按角色做完完整硬拦截。
2. 桌面端当前只有 `老板 / 业务 / PMC / 生产经理` 四类角色先收口为独立角色视图；采购、仓库、品质、财务仍以总后台剪裁和手机端任务为主。
3. 手机端统一按“任务分配 -> 任务处理 -> 处理反馈”理解，不扩展成复杂主数据维护端。

## 5. 当前推进顺序

1. 先按本页确认角色边界，不让桌面和手机端继续混角色。
2. 再按页面清单做菜单剪裁和默认页，不额外扩展第二套桌面站点。
3. 再按单据流转矩阵推进细分业务表，一次只拆一条主链，不同时铺复杂状态机和所有专表。
