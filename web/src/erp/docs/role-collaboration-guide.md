# 角色协同链路

> 适用对象：老板、业务、PMC、生产经理、采购、仓库、品质、财务  
> 当前用途：这一页专门回答“谁把任务交给谁、在哪个节点交接、异常时退回给谁”。

## 1. 角色协同总览

| 链路                          | 发起角色    | 接收角色            | 交接内容                                      | 链路类型     |
| ----------------------------- | ----------- | ------------------- | --------------------------------------------- | ------------ |
| 业务 / 跟单 -> 老板           | 业务 / 跟单 | 老板                | 立项资料、包装材料打单、交期风险              | 主链前置审批 |
| 老板 -> 采购 / PMC            | 老板        | 采购、PMC           | 审批放行、资料齐套要求、优先级                | 主链放行     |
| 采购 -> PMC                   | 采购        | PMC                 | 主料、辅包材、加工合同、回签、缺料 / 到料结果 | 主链协同     |
| PMC -> 生产经理               | PMC         | 生产经理            | 齐套状态、待排订单、延期风险、外发条件        | 主链协同     |
| 生产经理 -> 品质 / 仓库       | 生产经理    | 品质、仓库          | 完工送检、返工结果、入库准备、仓间流转        | 主链协同     |
| 品质 -> PMC / 生产经理 / 仓库 | 品质        | PMC、生产经理、仓库 | IQC、过程检验、返工结论、放行结果             | 子链反馈     |
| 仓库 -> 业务 / 财务           | 仓库        | 业务、财务          | 到仓、仓间流转、成品入库、待出货、出货单信息  | 主链协同     |
| 财务 -> 业务 / 老板 / 仓库    | 财务        | 业务、老板、仓库    | 对账、付款提醒、发货放行 / 未放行反馈         | 主链收口     |

## 2. 任务、业务、角色如何绑定

| 名词     | 通俗解释                                           | 当前系统里怎么看                                                       |
| -------- | -------------------------------------------------- | ---------------------------------------------------------------------- |
| 业务单据 | 这是什么事，例如订单、采购、委外、入库、出货、对账 | 业务页里的记录，当前先落到 `business_records`                          |
| 任务     | 现在轮到谁处理这件事                               | 手机端和协同任务池里的待办，当前落到 `workflow_tasks`                  |
| 角色     | 这件事进入哪个角色池                               | 老板、业务、PMC、采购、生产、仓库、品质、财务等角色任务池              |
| 事件     | 谁在什么时候做了什么                               | 处理、阻塞、完成、退回、状态流转等记录，当前落到任务事件和业务状态快照 |

理解顺序是：先看业务单据确认“这是什么事”，再看任务确认“现在谁要处理”，再看角色确认“进哪个角色池”，最后看事件确认“谁什么时候做了什么”。不要把任务当成业务真源，也不要让某个角色替另一个角色直接点完成。

## 3. 主链路上的角色交接

### 3.1 业务 / 跟单 -> 老板

1. 业务 / 跟单先收客户、订单编号、产品订单编号、款式编号、交期和包装材料前置信息。
2. 老板确认订单优先级、包装材料打单表和生产资料是否允许进入后续环节。
3. 如果资料不齐，任务退回业务 / 跟单继续补齐，不直接推进采购或排产。

### 3.2 老板 -> 采购 / PMC

1. 老板放行后，采购进入主料、辅包材、加工合同准备。
2. 同时 PMC 开始跟齐套、跟资料、跟交期，不等所有单据完全落地后才介入。
3. 如果老板要求调整交期、包装材料或优先级，采购和 PMC 都要同步收到变更。

### 3.3 采购 -> PMC

1. 采购向 PMC 反馈主料需求、辅包材下单、加工合同回签、缺料和到料状态。
2. PMC 根据这些结果判断能否排产、是否需要拆批、是否需要改外发节奏。
3. 如果缺料或合同未回签，任务不直接推给生产经理，而是停留在齐套 / 催办层。

### 3.4 PMC -> 生产经理

1. PMC 把齐套状态、待排订单、外发条件、延期风险交给生产经理。
2. 生产经理决定机器加工 / 裁切、车缝、手工、本厂或外发走向，以及返工优先级。
3. 若生产经理判断不能开线，要回退给 PMC 继续追料、改批次或调整交期说明。

### 3.5 生产经理 -> 品质 / 仓库

1. 生产经理在完工、返工完成或阶段性可送检时，把任务交给品质，并同步仓库准备仓间流转。
2. 当前生产实物流转先按 `主料仓 -> 机器加工 / 裁切 -> 车缝 -> 其他仓 -> 手工 / 组装 / 包装 -> 成品仓` 理解。
3. 通过检验后，仓库接收入库、备货、待出货准备；如果品质不通过，任务回退给生产经理；如果数量或包装不齐，任务同时反馈 PMC。

### 3.6 仓库 -> 业务 / 财务

1. 仓库把到仓、IQC 通过、仓间流转、成品入库、待出货、出货单等状态回传给业务和财务。
2. 业务据此做客户确认和发货准备，财务据此做放行或未放行反馈。
3. 如果仓库发现缺件、异常件、未齐套，不能直接完成发货，要回退到对应上游角色。
4. 返工成品也先回仓，再按返工任务重新领出，不单独绕开仓库。

### 3.7 财务 -> 业务 / 老板 / 仓库

1. 财务根据合同条款、加工金额、辅包材金额和放行条件做对账与提醒。
2. 财务把已放行 / 未放行结果回给业务和仓库，决定发货是否继续。
3. 重大异常费用、结算风险和延期风险同时反馈老板。

## 4. 子链路上的角色交接

### 4.1 包装材料打单审批链

1. 业务 / 跟单发起包装材料打单准备。
2. 老板审批是否放行。
3. 放行后采购和 PMC 同步接单；未放行则退回业务补资料或改方案。

### 4.2 主辅料到仓 + IQC 链

1. 采购或供应商交付主辅料到仓。
2. 仓库发起收货，按主料仓、其他仓、成品仓分层挂账，品质执行 IQC。
3. IQC 结果反馈采购、PMC、仓库；不通过时回退采购处理供应商或补料问题。

### 4.3 主料仓 -> 裁切 -> 车缝 -> 其他仓 -> 手工 -> 成品仓

1. 主料先从主料仓领出，进入激光、机器加工和裁切。
2. 车缝从主料仓领取已加工 / 已裁切的裁片进行缝纫。
3. 车缝后的半成品先回其他仓。
4. 手工再从仓库领取主料、辅料和半成品，完成缝合、组装和包装。
5. 手工完成的成品统一回成品仓；返工成品也同样回仓。

### 4.4 包装材料到仓 + 质量检查 + 领用链

1. 包装材料到仓后由仓库和业务关注是否齐套。
2. 品质或业务确认包装质量是否符合要求。
3. 合格后进入包装领用；不合格则回退给采购或业务重新协调。

### 4.5 延期 / 返工 / 异常链

1. PMC、生产经理、品质、仓库、业务都可以发起异常。
2. 生产相关异常优先回到 PMC / 生产经理。
3. 质量异常优先回到品质 + 生产经理。
4. 发货或放行异常优先回到业务 + 仓库 + 财务。
5. 重大异常同步抄送老板。

## 5. 手机端和桌面端怎么衔接

1. 手机端负责把任务接住、处理掉、回填结果，不单独定义第二套流程。
2. 桌面端负责看全链路、审批、决策、例外处理和正式记录入口。
3. 手机端任何“已处理 / 已确认 / 异常上报”最终都要回到桌面端主链节点继续推进。

## 6. 当前校对重点

1. 老板是否只保留桌面总后台，不再给采购、仓库、财务单独做桌面角色后台。
2. 手机端角色是否确认包含老板、业务、PMC、生产经理、采购、仓库、品质、财务。
3. 包装材料审批、IQC、返工异常、发货放行这四类支线是否都挂对了角色。
4. 主料仓、其他仓、成品仓三层仓库划分，以及“裁切 -> 车缝 -> 手工 -> 成品仓”的工序链是否准确。
5. 哪些交接必须先有审批或先有反馈，哪些可以并行。
