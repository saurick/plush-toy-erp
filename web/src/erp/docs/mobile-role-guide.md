# 手机端角色流程

> 校对说明：本页按目标协同口径先整理 8 类手机端角色，供你和老板确认“哪些角色应该有手机端、各自只做什么任务”。  
> 当前约束：手机端主要承接任务分配、任务处理、处理反馈，不扩展为复杂主数据维护端。

当前补充口径：

- 仓库至少分 `主料仓`、`其他仓`、`成品仓`。
- 所有材料、半成品、成品、返工成品、外发件都必须先走仓库或从仓库领出。
- 当前工序链按 `主料仓 -> 机器加工 / 裁切 -> 车缝 -> 其他仓 -> 手工 / 组装 / 包装 -> 成品仓` 理解。

## 1. 手机端角色总览

| 角色     | 手机端定位         | 主链路任务                             | 典型子链路任务                 |
| -------- | ------------------ | -------------------------------------- | ------------------------------ |
| 老板     | 审批与风险查看     | 待审批、交期风险、重大异常             | 审批反馈、优先级调整           |
| 业务     | 前置协调与客户反馈 | 缺资料、交期确认、发货前确认           | 催办、变更反馈、客户异常       |
| PMC      | 任务中枢           | 齐套推进、排产推进、延期跟进           | 催料、催合同、异常分发         |
| 生产经理 | 现场决策           | 今日排产、返工处理、完工反馈           | 外发切换、异常回报             |
| 采购     | 供应链反馈         | 缺料、到料、回签、单价确认             | 催供应商、补料反馈             |
| 仓库     | 收发执行           | 收货、备料、仓间流转、入库、待出货处理 | 缺件上报、返工回仓、未放行提醒 |
| 品质     | 检验反馈           | IQC、过程异常、返工复检                | 不合格退回、关闭异常           |
| 财务     | 放行与结算提醒     | 对账提醒、付款提醒、放行反馈           | 异常费用反馈、未放行提醒       |

## 2. 老板

### 手机端定位

老板手机端只看待审核、交期风险、重大异常和待结算摘要。

### 主链路任务

1. 接收业务发起的审批任务。
2. 查看 PMC 汇总的交期风险和延期预警。
3. 查看财务汇总的待结算和异常费用摘要。

### 子链路任务

1. 对包装材料打单和关键放行事项给出审批反馈。
2. 对重大异常给出优先级调整或处理意见。

### 触发来源

业务、PMC、财务的待审批或高风险任务。

### 反馈去向

回给业务、PMC、财务，决定主链路是否继续推进。

### 手机端不做什么

1. 不维护 BOM、合同或采购明细。
2. 不在手机端做复杂经营分析录入。

## 3. 业务

### 手机端定位

业务手机端负责把客户前置动作变成可推进任务，并及时回传客户侧反馈。

### 主链路任务

1. 确认缺资料、交期和客户变更。
2. 确认发货前客户侧信息和业务侧放行条件。

### 子链路任务

1. 催资料、催合同、催料。
2. 上报客户改期、改数量、改包装要求。
3. 对接待出货前的最终确认。

### 触发来源

立项页、待出货页、老板审批链、财务未放行提醒。

### 反馈去向

回给老板、PMC、仓库、财务，推动排产或发货。

### 手机端不做什么

1. 不维护完整 BOM。
2. 不编辑正式合同模板。

## 4. PMC

### 手机端定位

PMC 手机端是跟进端，核心是看今天该催什么、哪里卡住了、谁还没反馈。

### 主链路任务

1. 跟齐套、跟排产、跟交期。
2. 确认某单是否可推给生产经理。
3. 确认成品入库和待出货节奏。

### 子链路任务

1. 催采购反馈缺料、到料、回签。
2. 催生产经理反馈排产、返工、延期原因。
3. 把异常分发给采购、品质、仓库或业务。

### 触发来源

采购反馈、生产反馈、仓库反馈、异常页任务。

### 反馈去向

回给老板、生产经理、业务或仓库，保持主链路不停滞。

### 手机端不做什么

1. 不维护大批量主数据。
2. 不替代桌面端做完整异常分析。

## 5. 生产经理

### 手机端定位

生产经理手机端是现场决策端，重点是今天怎么排、出了什么问题、是否能交接。

### 主链路任务

1. 确认今日排产。
2. 回填裁切、车缝、手工、返工、延期结果。
3. 确认可送检、可回其他仓、可入成品仓的批次。

### 子链路任务

1. 上报外发切换、裁切完成、车缝完成、手工切换、返工优先级。
2. 上报品质异常、现场异常和延期原因。

### 触发来源

PMC 下发的待排订单、品质退回、仓库催交任务。

### 反馈去向

回给 PMC、品质、仓库，推动送检和入库。

### 手机端不做什么

1. 不维护客户前置资料。
2. 不在手机端处理采购或财务明细。

## 6. 采购

### 手机端定位

采购手机端只处理供应链状态，不做复杂模板维护。

### 主链路任务

1. 反馈缺料、到料、加工合同回签、单价确认。
2. 确认主料、辅包材、加工合同这三类任务的当前状态，以及是否已可入主料仓。

### 子链路任务

1. 催供应商回复。
2. 上报补料、延迟、回签异常。

### 触发来源

业务催办、PMC 催办、仓库 IQC 反馈、供应商结果。

### 反馈去向

回给 PMC、业务、仓库、财务，决定排产与对账。

### 手机端不做什么

1. 不编辑复杂采购模板。
2. 不在手机端维护供应商主档全量信息。

## 7. 仓库

### 手机端定位

仓库手机端负责把到仓、备料、仓间流转、入库、待出货这些执行动作快速反馈出来。

### 主链路任务

1. 确认主辅料和包装材料收货并进入对应仓位。
2. 确认主料仓领料、其他仓回仓、成品仓入库和待出货准备。
3. 确认出货前仓库侧完成情况。

### 子链路任务

1. 上报缺件、异常件、未齐套、返工回仓。
2. 提醒业务确认和财务放行。

### 触发来源

采购到料、裁切 / 车缝 / 手工完成、品质检验结果、待出货任务。

### 反馈去向

回给业务、PMC、财务和品质，决定是否继续发货。

### 手机端不做什么

1. 不在手机端维护完整库存主档。
2. 不直接决定财务放行。

## 8. 品质

### 手机端定位

品质手机端只做检验反馈和异常闭环，不扩展为独立数据维护端。

### 主链路任务

1. 确认 IQC 结果和仓库收货后是否可入正式仓位。
2. 确认过程品质异常和返工复检结果。
3. 确认可入其他仓、可入成品仓、可出货的质量状态。

### 子链路任务

1. 退回不合格批次。
2. 关闭已复检通过的异常。
3. 上报反复出现的质量问题。

### 触发来源

仓库收货、生产送检、返工完成、待出货检查。

### 反馈去向

回给 PMC、生产经理、仓库，决定返工或放行。

### 手机端不做什么

1. 不建立客户订单或合同。
2. 不维护财务、采购或主档明细。

## 9. 财务

### 手机端定位

财务手机端只看待对账、待付款、异常费用和放行反馈。

### 主链路任务

1. 接收加工费、辅包材金额和待结算提醒。
2. 对发货节点给出已放行 / 未放行反馈。

### 子链路任务

1. 上报异常费用。
2. 提醒待付款和对账周期。
3. 对未满足放行条件的单据给出回退反馈。

### 触发来源

加工合同、辅包材采购、仓库待出货、老板关注项。

### 反馈去向

回给业务、仓库、老板，决定发货和后续结算处理。

### 手机端不做什么

1. 不在手机端做复杂利润报表。
2. 不在手机端维护完整合同模板。
