# 当前明确不做

> 当前用途：把 deferred 能力、未落地边界和仍在校对中的角色拆分单独写清楚，避免后续继续脑补。

## 1. 当前已明确 deferred 的能力

1. 扩展硬件链路
2. PDA
3. 条码枪
4. 图片识别
5. 离线同步
6. 复杂利润报表
7. PDF 坐标回填
8. Excel 母版回写
9. 手机端模板编辑

## 2. 当前帮助中心里的角色页怎么理解

1. `桌面端角色流程` 先按目标口径只保留老板、业务、PMC、生产经理四类桌面端角色。
2. `手机端角色流程` 先按目标口径整理老板、业务、PMC、生产经理、采购、仓库、品质、财务八类角色。
3. 这批角色页当前主要用于你和老板校对流程，不等于所有角色后台都已完全落地。

## 3. 帮助中心首页不会展示所有开发细节

1. 帮助中心首页默认面向不懂编程的业务角色，优先展示新手入口、角色工作表、业务主线和常见问题。
2. `Workflow / Schema 草案`、`任务 / 业务状态字典`、`日志 / 审计 / Trace v1`、字段联动口径、计算口径和验收调试资料属于高级文档，不作为普通业务用户首页主入口平铺。
3. 所有正式 Markdown 文档仍然保留并继续注册在文档中心；这里只调整首页展示层级，不删除文档，也不改变 `/erp/docs/:key` 的访问方式。
4. 开发与验收模块继续承接验收结果总览、业务链路调试、协同任务调试、字段联动覆盖、运行记录和专项报告，不和普通帮助中心入口合并。

## 4. 当前不应该误判成已完成的事情

1. 不应把手机端理解成已经具备完整主数据维护能力。
2. 不应把角色后台理解成各自独立的一整套系统。
3. 不应把通用业务记录理解成客户、BOM、采购、库存、生产和财务专表都已经拆完。
4. 不应把打印模板理解成已经自动留档、反向回写业务数据或具备后端打印 DTO。
5. 不应把菜单权限裁剪理解成后端已经具备完整业务级角色鉴权。

## 5. 当前仍待继续确认的边界

| 主题             | 当前状态                                            | 为什么还没完全定死                                   |
| ---------------- | --------------------------------------------------- | ---------------------------------------------------- |
| 手机端角色拆分   | 已先按 8 个角色写校对稿                             | 还要等你和老板确认是否全部成立                       |
| 角色后台剪裁方式 | 已明确“总后台 + 角色切片”方向，并已接入菜单权限预设 | 后端业务级角色鉴权仍要后续补齐                       |
| 财务桌面端       | 当前不作为独立桌面角色后台继续展开                  | 要避免和老板总后台职责重叠                           |
| 品质手机端       | 当前已单列为目标角色                                | 还要确认是否独立端口或挂在其他角色切片下             |
| 细分业务专表     | 当前先用通用业务记录承接                            | 需要按真实样本继续拆 BOM、采购、合同、出入库和财务表 |

## 6. 这页存在的意义

1. 防止新需求讨论时默认把 deferred 能力当成已经上线。
2. 防止后续 AI 或维护者把“校对稿角色拆分”误当成“全部功能已经实现”。
3. 给帮助中心、产品设计和功能开发提供同一条边界线。
