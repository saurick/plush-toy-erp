# 业务链路调试

> 目的：把毛绒 ERP 当前怎样排查一条业务链路说清楚。`/erp/qa/business-chain-debug` 已提供只读查询页，但不提供一键重建测试场景，也不复用旧外贸链路的调试样本。

## 1. 当前结论

- 当前业务保存层真源是 `business_records`、`business_record_items` 和 `workflow_business_states`。
- 当前协同任务真源是 `workflow_tasks` 和 `workflow_task_events`。
- 当前业务页已经能保存通用单据、明细行、业务状态和协同任务。
- 业务链路调试页只读取当前服务返回的业务记录、业务状态快照和任务，适合按单据号、来源单号、客户、供应商、物料、模块 key 或任务名称排查。
- 当前不新增造数按钮、清库按钮或旧外贸 `tddbg_*` 样本。

## 2. 排查一条链路先看什么

| 排查问题               | 先看入口                       | 判断重点                                                |
| ---------------------- | ------------------------------ | ------------------------------------------------------- |
| 单据有没有保存成功     | 对应业务页列表                 | `business_records` 是否出现记录，表头字段是否为空或残留 |
| 明细金额是否正确       | 对应业务页弹窗和列表金额       | 明细金额为空时是否按数量和单价派生，手工金额是否被保留  |
| 状态有没有同步         | 业务页状态选择和任务看板钻取   | `workflow_business_states` 是否记录最新业务状态         |
| 任务有没有派发         | 业务页协同任务池和手机端任务页 | `workflow_tasks` 是否按角色、来源和状态展示             |
| 阻塞或取消原因是否丢失 | 业务页状态弹窗、任务池、手机端 | 原因是否同步到业务记录 payload、业务状态和未终止任务    |

## 3. 只读调试页怎么用

1. 打开 `/erp/qa/business-chain-debug`。
2. 输入单据号、来源单号、客户、供应商、物料、模块 key 或任务名称。
3. 先看“匹配业务记录”确认表头字段、明细数量、金额、业务状态快照和主责角色。
4. 再看“匹配协同任务”确认任务状态、业务状态、主责角色和阻塞原因。
5. 如果查询词是模块 key，例如 `project-orders` 或 `processing-contracts`，页面会回查该模块当前记录，便于先从模块层排查。

## 4. 当前调试顺序

1. 先从左侧菜单进入目标业务页。
2. 新建或编辑一条最小记录，确认表头字段、明细行和金额显示。
3. 流转一次业务状态，确认成功提示、状态标签和任务看板变化。
4. 创建协同任务，确认任务池出现对应任务。
5. 到对应手机端角色任务页确认任务可读、可处理、可阻塞、可完成。
6. 如果页面表现异常，再回到后端 Ent 表和 JSON-RPC 日志核对。

## 5. 当前不做什么

- 不提供“清空业务 SQL 数据”按钮。
- 不提供“一键重建调试样本”按钮。
- 不用报价、外销、出运、结汇、水单等旧外贸链路当作毛绒 ERP 样本。
- 不把临时调试样本混进正式业务记录。

## 6. 后续升级条件

后续如果要把这页升级成真正工具页，需要先具备：

- 毛绒业务自己的固定场景 key。
- 可重复创建和清理的测试数据前缀。
- 明确不会误删真实业务数据的安全边界。
- 对应浏览器回归和后端测试。
