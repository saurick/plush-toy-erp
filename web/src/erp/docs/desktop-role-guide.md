# 桌面端角色流程

> 校对说明：本页按当前资料先整理桌面端 4 类角色流程，供你和老板先校对分工是否对线。  
> 当前约束：桌面端只保留老板、业务、PMC、生产经理四类角色；其他角色后台默认从总后台剪裁，不再单独做完整桌面角色首页。

当前补充口径：

- 仓库至少按 `主料仓`、`其他仓`、`成品仓` 三层理解。
- 所有材料、半成品、成品、返工成品和外发件都必须先走仓库节点。
- 当前生产链按 `主料仓 -> 机器加工 / 裁切 -> 车缝 -> 其他仓 -> 手工 / 组装 / 包装 -> 成品仓` 理解。

## 1. 桌面端角色总览

| 角色     | 桌面端定位               | 主链路关注点                                        | 典型子链路                               |
| -------- | ------------------------ | --------------------------------------------------- | ---------------------------------------- |
| 老板     | 总后台拥有者与关键审批人 | 立项放行、包装材料审批、交期风险、异常、待结算      | 审批、优先级调整、重大异常决策           |
| 业务     | 客户与订单前置负责人     | 客户 / 款式立项、资料齐套、交期、发货前确认         | 缺资料催办、包装材料前置沟通、客户变更   |
| PMC      | 全链路跟进与齐套中枢     | 齐套、排产准备、延期预警、跨角色推进                | 催料、催合同、异常分发、节奏协调         |
| 生产经理 | 生产执行与资源决策人     | 排单、裁切 / 车缝 / 手工 / 外发决策、返工、完工交接 | 返工优先级、外发切换、仓间流转、完工送检 |

## 2. 老板

### 角色定位

老板在桌面端看全局经营风险、关键审批和异常，不下沉做复杂录入。

### 主链路

1. 审核业务提交的立项资料和包装材料打单需求。
2. 决定是否放行采购准备、生产资料准备和主线推进。
3. 关注 PMC 和生产经理反馈的交期风险、延期和返工。
4. 关注仓库、财务反馈的待出货、待放行、待结算结果。

### 子链路

1. 审批包装材料打单表和关键生产资料。
2. 处理重大延期、重大异常订单和优先级调整。
3. 查看待结算、异常费用和需要管理层拍板的事项。

### 协作关系

| 对接对象 | 老板主要收到什么                     | 老板主要回给什么       |
| -------- | ------------------------------------ | ---------------------- |
| 业务     | 立项资料、包装材料前置需求、交期风险 | 是否放行、是否改优先级 |
| PMC      | 齐套风险、延期预警、异常概览         | 资源优先级、处理要求   |
| 财务     | 待结算、异常费用、放行风险           | 放行意见、风险确认     |

### 不负责的范围

1. 不在桌面端维护完整 BOM。
2. 不在桌面端录入采购合同或加工合同明细。
3. 不在桌面端逐条回填生产进度或仓库收发。

## 3. 业务

### 角色定位

业务是客户 / 订单前置负责人，也是资料齐套和发货前确认的发起人。

### 主链路

1. 发起客户 / 款式立项，收口客户、订单编号、产品订单编号、款式编号、产品编号和交期。
2. 确认资料是否齐套，并把缺资料任务回推到对应责任人。
3. 在发货前完成业务确认，确保客户侧和内部侧信息一致。
4. 接收仓库和财务反馈，推进最终发货放行。

### 子链路

1. 发起包装材料打单前置准备，并提交给老板审批。
2. 催资料、催合同、催到料、催回签。
3. 处理客户改期、改款、改数量等前置变化。
4. 对接待出货、未放行和客户侧异常反馈。

### 协作关系

| 对接对象 | 业务主要收到什么             | 业务主要回给什么         |
| -------- | ---------------------------- | ------------------------ |
| 老板     | 是否放行、是否调整优先级     | 立项资料、审批申请       |
| 采购     | 缺料、到料、合同、回签反馈   | 下游交期压力、资料催办   |
| 仓库     | 成品入库、待出货、出货单状态 | 客户确认、发货安排       |
| 财务     | 放行 / 未放行结果            | 客户侧发货信息和确认结果 |

### 不负责的范围

1. 不在桌面端维护完整采购台账。
2. 不决定车缝 / 手工 / 外发排单。
3. 不在桌面端处理财务对账细节。

## 4. PMC

### 角色定位

PMC 是桌面端的全链路中枢，负责把采购、生产、仓库、异常和交期连接起来。

### 主链路

1. 根据立项、采购和合同结果确认订单是否齐套。
2. 将可执行订单推入排产准备，并把未齐套订单继续挂回催办链。
3. 跟踪生产进度、延期风险、返工和异常。
4. 与仓库协同确认主料仓出库、其他仓半成品回仓、成品仓回仓、待出货和关键交期节点。

### 子链路

1. 催料、催合同、催回签。
2. 发现延期、返工、异常后分发到生产经理、品质、采购或业务。
3. 汇总多角色反馈，形成老板可读的交期风险和异常视图。

### 协作关系

| 对接对象 | PMC 主要收到什么                                         | PMC 主要回给什么             |
| -------- | -------------------------------------------------------- | ---------------------------- |
| 采购     | 缺料、到料、回签、加工合同状态                           | 排产紧急度、催办优先级       |
| 生产经理 | 排单能力、返工情况、异常说明                             | 待排订单、齐套结果、交期要求 |
| 仓库     | 主料仓出入库、其他仓半成品、成品仓入库、待出货、缺件反馈 | 发货节奏、仓间优先级信息     |
| 老板     | 交期风险、重大异常                                       | 处理建议和资源请求           |

### 不负责的范围

1. 不直接做客户前置沟通。
2. 不直接审批合同条款或财务放行。
3. 不代替生产经理做最终生产方式决策。

## 5. 生产经理

### 角色定位

生产经理负责生产执行决策，把可生产订单变成可交付结果。

### 主链路

1. 接收 PMC 的待排订单、齐套状态和交期要求。
2. 决定机器加工 / 裁切、车缝、手工、本厂或外发走向。
3. 跟踪返工、异常、完工、送检和仓间流转节奏。
4. 把完工结果交给品质和仓库，完成从其他仓到成品仓的入库前交接。

### 子链路

1. 处理返工优先级和生产异常。
2. 决定某批次是否切换外发、拆批执行或调整仓间流转节奏。
3. 针对品质异常给出返工或重做安排。

### 协作关系

| 对接对象 | 生产经理主要收到什么                                   | 生产经理主要回给什么             |
| -------- | ------------------------------------------------------ | -------------------------------- |
| PMC      | 待排订单、齐套结果、交期压力                           | 排单决定、延期说明、返工状态     |
| 品质     | 检验结论、返工要求                                     | 完工送检、返工结果               |
| 仓库     | 主料仓领料、其他仓回仓、成品仓入库准备、包装和出货节奏 | 完工批次、仓间流转节点、交付时点 |

### 不负责的范围

1. 不负责客户 / 款式立项。
2. 不负责合同回签和采购对账。
3. 不负责财务放行和待付款处理。
