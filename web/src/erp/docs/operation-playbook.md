# 毛绒 ERP 首批流程草案

## 人话版主线

`客户 / 款式 -> BOM / 材料 -> 加工合同 -> 生产排单 -> 仓库收发 -> 对账结算`

这条主线是本项目当前正式口径。后续合同模板、Excel 导入和移动端动作都按这条路径落位。

## 1. 客户与款式立项

- 业务跟单维护客户、款式编号、版本、交期和打样状态。
- 这里是后续材料、合同和排单的共同来源入口。
- 本轮先做工作台和文档，不补客户消息或审批流。

## 2. BOM / 材料拆解

- 把主料、辅料、包材和工艺拆成结构化资料层。
- 当前已预留入口，等待更多 Excel 后再接正式导入。
- 不允许在资料不全时直接猜字段映射或打印口径。

## 3. 加工合同 / 委外下单

- 先收口合同模板、加工单位、单价、付款节点和附件位。
- 当前只初始化合同入口和帮助文档，不落正式打印引擎。
- 资料到齐后再统一做字段真源、保存链路和打印模板。

## 4. 生产排单 / 进度跟催

- 厂长和生产角色关注排产、延期原因、返工与完工节点。
- 手机端先做进度确认、异常上报和待办提醒。
- 拍照、扫码和工时采集延后。

## 5. 仓库收发 / 成品入库

- 仓库角色先看待收货、待入库、备货和异常登记。
- 当前不做扫码入库，只初始化收发工作台与移动端点检卡片。
- 后续如需 PDA 或条码，再单独补资料和边界说明。

## 6. 对账 / 结算 / 经营复盘

- 财务角色先看加工费、辅材包材、应收应付和异常费用节点。
- 管理层看延期、待结算和高风险订单。
- 本轮先做摘要和提醒，不做利润细算或票据识别。
