# 毛绒 ERP 系统初始化说明

## 本轮目标

- 基于 `trade-erp` 的信息架构初始化本项目自己的 ERP 壳层。
- 先收口后台路由、角色工作台、帮助中心、流程文档、移动端入口和资料准备清单。
- 把复杂任务落到 `docs/changes/plush-erp-bootstrap-init.md`，便于后续继续接力。

## 本轮已初始化

- 后台主路由：管理员登录后默认进入 `/erp/dashboard`。
- 初始化看板：统一查看模块状态、角色入口、资料准备与端口边界。
- 流程总览：先用毛绒工厂的业务主线替换 trade 的外贸业务链。
- 帮助中心与文档中心：把“先看什么、当前能做什么、后续资料接哪里”做成正式入口。
- 移动端工作台：同仓库内提供各角色的手机端初始化页面，不单独拆第二个项目。

## 本轮明确不做

- 拍照扫码、PDA、条码枪、图片识别。
- 正式 Excel 导入、合同模板打印和 PDF 坐标填充。
- 完整业务实体、权限矩阵、审批流、经营利润口径。

## 为什么不直接照搬 trade-erp

- `trade-erp` 的主路径是报价、外销、出运和结汇，不适合直接作为毛绒工厂业务真源。
- 当前任务是“先初始化框架”，不是把外贸 ERP 的所有表和字段原样复制过来。
- 先把毛绒 ERP 的信息架构、角色工作台和帮助中心立住，后续合同、Excel 与打印模板才能接在正确层。

## 当前端口口径

- 前端开发端口：`5175`
- 后端 HTTP 端口：`8200`
- 后端 gRPC 端口：`9200`
- PostgreSQL 宿主机映射端口：`5435`

## 后续接入顺序

1. 先补更多合同与 Excel 原件。
2. 再收口字段真源、导入映射与保存链路。
3. 最后再做打印模板、扫码或图片识别。
